# Claude Code 官方来源判定规则

## 目的

定义什么内容可以进入 `refs/`，什么内容不能进入 `refs/`。

## 可视为官方来源的内容

满足以下任一条件，可作为官方来源候选：

1. 域名明确属于 Anthropic 官方控制范围
   - 例如：`anthropic.com`、`docs.anthropic.com`、`console.anthropic.com`
2. 页面内容明确以官方产品文档、官方公告、官方控制台说明出现
3. 页面可被其他官方入口反向引用或导航到达

## 不应直接视为官方来源的内容

以下内容即使有参考价值，也不应放入 `refs/` 作为官方证明：

- 第三方教程
- 博客转载
- 社区帖子
- 聚合站镜像
- AI 二次总结稿
- 个人笔记

## 灰区处理

以下情况需要单独标注，不直接视为“已验证官方事实”：

- 模型既有知识回忆出的 URL
- 无法实时访问核验的历史官方路径
- 登录后可见、公开入口不可直达的页面
- 官方域名下的临时、归档、跳转页面

## 状态建议

- `unverified-live`：基于既有知识判断很可能是官方，但未实时核验
- `verified`：已核验为可达官方来源
- `redirected`：原始链接跳转到其他官方页面
- `archived`：历史官方页面，现已归档或不再主导航展示
- `blocked`：访问受限，暂无法确认
