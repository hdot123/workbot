# official-upstream

用于存放各项目的原始官方文档原料，并作为后续整理、吸收、promotion 的上游证据层入口。

## 目录目标

本目录不直接承载最终知识成品。
它的职责是：

- 保存官方原始材料
- 保存覆盖范围与抓取索引
- 保存官方来源证明
- 保存人工核对线索

## 固定四层

每个项目目录下固定保留以下四层：

- `raw/`
  - 原始抓取结果、原始 markdown、导出的页面
- `manifest/`
  - 页面清单、站点地图、抓取索引
- `refs/`
  - 官方链接证明
  - 官方入口页
  - 官方页面 URL 清单
  - 页面到来源的映射
- `notes/`
  - 人工整理备注、页间关系、待核点

## 使用原则

- `raw/` 看原文
- `manifest/` 看覆盖范围
- `refs/` 看官方来源证明
- `notes/` 看人工整理线索

## refs/ 约束

`refs/` 只放上游官方来源证明。

允许放入：

- 官方入口页
- 官方页面 URL 清单
- 页面到来源的映射
- 来源可达性与访问方式说明

不允许放入：

- 分析结论
- 提炼后的真理
- 最终整理稿
- promotion 之后的下游结论
