# OpenCode promotion 前检查清单

## 目的

在把上游原料吸收到下游知识层之前，先检查证据链是否足够完整。

## 证据链检查

- [ ] 该结论是否能追溯到 `refs/` 中的官方来源
- [ ] 该页面是否已在 `page-source-map` 中建立映射
- [ ] 对应原始内容是否已落在 `raw/`
- [ ] 页面是否已进入 `manifest/` 页面索引
- [ ] 是否区分了“官方原文”与“人工判断”

## 时效性检查

- [ ] 页面是否已实时核验，还是仅来自模型既有知识
- [ ] 若未核验，是否明确标记为候选而非已验证事实
- [ ] 若页面高频变化，是否已标 freshness 风险

## 语义检查

- [ ] 该结论是否直接来自页面原文，而不是由多层推断得来
- [ ] 是否误把示例、提示、边栏说明提升成了稳定规则
- [ ] 是否存在和其他页面冲突但尚未解决的地方

## promotion 禁止条件

出现以下任一情况，不应直接 promotion：

- 来源不是官方
- 页面尚未可追溯
- 原始内容未落盘
- 只有模型记忆，没有页面证据
- 已发现冲突但未处理
