# OpenCode 当前待核点

## 与来源相关

- 当前 OpenCode 的现行官方主入口到底是 `opencode.ai/`、`opencode.ai/docs`，还是仓库 README
- `https://github.com/sst/opencode` 是否应视为主官方事实源，还是仅作代码与发布旁证
- `https://github.com/sst` 仅用于归属旁证，还是也承担官方导航作用

## 与页面结构相关

- 安装、配置、权限、工作流、排障是否为独立页面，还是同页分节
- 是否存在公开文档之外、登录后才可见的页面
- release notes 更可能在官网、文档站还是 GitHub Releases

## 与后续 promotion 相关

- 哪些页面能作为“根事实页面”优先进入 promotion
- 哪些内容属于仓库实现细节，不应直接提升为稳定知识
- 哪些页面变化频率高，需要单独标 freshness

## 当前结论边界

在未联网核验前：

- 可以用模型既有知识提供高概率官方入口种子
- 不能把这些 URL 当作最终已验证事实
- 不能把候选路径直接当作最终页面地图
