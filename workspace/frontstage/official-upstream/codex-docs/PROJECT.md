# Codex 文档原料项目说明

## 目标

本目录用于沉淀 Codex 相关的原始官方文档原料，并把后续的抓取、索引、人工核对流程固定下来。

## 目录约定

- `raw/`：原始抓取结果、原始 markdown、导出的页面
- `manifest/`：页面清单、站点地图、抓取索引
- `refs/`：官方链接证明、官方入口页、页面到来源映射
- `notes/`：人工整理备注、页间关系、待核点

## 建议工作流

1. 先把官方站点页面原样落到 `raw/`
2. 在 `manifest/` 里登记页面来源、抓取时间、文件落点、状态
3. 在 `refs/` 里记录来源入口、官方 URL、来源映射与核验状态
4. 在 `notes/` 里记录交叉引用、冲突点、待核点
5. 不在这里直接写最终整理稿；这里保留“上游原料层”职责

## 状态字段建议

建议统一使用以下状态：

- `todo`：待抓取
- `seeded`：已有入口种子
- `fetched`：已抓取
- `parsed`：已初步整理
- `reviewed`：已人工核对
- `blocked`：存在访问或内容问题

## 命名建议

建议原始文件名尽量包含：

- 日期
- 站点或栏目
- 页面标题 slug

示例：

- `2026-04-12-codex-root.md`
- `2026-04-12-api-overview.html`

## 备注

本目录不假设任何单一抓取工具。
可由浏览器导出、API 抓取、爬虫落盘或人工复制整理后统一归档。
