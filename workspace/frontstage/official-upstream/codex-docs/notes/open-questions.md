# Codex 当前待核点

## 与来源相关

- 当前 Codex 的现行官方主入口到底是 `platform.openai.com/docs`、独立产品页，还是被收敛进其他文档栏目
- `https://openai.com/index/openai-codex/` 是否只应视为历史/研究入口，而不是现行产品文档入口
- `platform.openai.com/docs/guides` 下是否存在明确以 Codex 命名的现行指南页

## 与页面结构相关

- Codex 的 CLI、workflow、permissions、configuration 是否为独立页面，还是同页分节
- Codex 相关说明是否更多散落在 API / Agents / Responses 等其他文档栏目中
- 是否存在登录后才可见、公开入口不可直接到达的 Codex 相关页面

## 与后续 promotion 相关

- 哪些页面能作为“根事实页面”优先进入 promotion
- 哪些内容只是历史背景，不应提升为现行稳定知识
- 哪些页面变化频率高，需要单独标 freshness

## 当前结论边界

在未联网核验前：

- 可以用模型既有知识提供高概率官方入口种子
- 不能把这些 URL 当作最终已验证事实
- 不能把候选路径直接当作最终页面地图
