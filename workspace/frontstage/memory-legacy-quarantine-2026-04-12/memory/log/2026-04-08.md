## 概述

今天把 `pm-bot` 在 `cmux` 交互态中的 Crawl4AI 调用路径问题做了收口，并把记录层级重新对齐。

这件事的核心不是 `crawl4ai` 服务异常，而是交互启动链给了模型过宽的 Bash fallback 和自证空间，导致它会先把 `mcp__crawl4ai__md` 误走成 `Bash(mcp__crawl4ai__md ...)`。

---

## 今日结论

- `crawl4ai` 服务本身正常，问题在 `pm-bot` 的交互启动链
- 只改身份 md 不够，必须把约束前置到 runtime bootstrap
- 对 `pm-bot` 生效的收口方式是 `--bare`、`--strict-mcp-config`、单独注入 `crawl4ai` MCP 配置、`--append-system-prompt`、`--disallowed-tools 'Bash(*)'`
- 修复后，交互态已经能稳定直接走原生 `crawl4ai - md (MCP)`，并返回 `页面标题：Example Domain`
