### 执行过程

1. **需求分析**
   - 目标：统一使用 ClawRouter 作为协议适配层

2. **架构设计**
   - 创建新的 `clawrouter` provider
   - 迁移所有模型到 ClawRouter
   - 保留 ollama (本地模型)
   - 删除旧的 zai/bailian providers

3. **配置实现**
   - baseUrl: `http://192.168.88.27:3000/v1`
   - apiKey: `${CLAWROUTER_API_KEY}` (任意值)
   - api: `openai-responses`
   - models: 16 个（智谱 + 百炼 + Voyage）

4. **环境变量**
   - 添加 CLAWROUTER_API_KEY 到 .env
   - 保留旧环境变量（兼容性）

### 技术洞察

- **协议适配**: ClawRouter 使用 OpenAI Responses API，自动适配 Anthropic/OpenAI 协议
- **API Key**: ClawRouter 不验证 API Key，任意值即可（保持配置一致性）
- **本地优先**: ClawRouter 在内网（192.168.88.27），延迟更低

### 关键决策

- **保留 ollama**: 本地模型不走 ClawRouter，直接连接
- **删除旧 providers**: 避免配置混乱，统一使用 ClawRouter
- **环境变量隔离**: CLAWROUTER_API_KEY 独立，不与其他 key 混用
