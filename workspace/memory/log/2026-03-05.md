### [04:01] 发现 OpenClaw Gateway 版本显示 bug
- **问题**: Gateway 服务版本显示 "dev" 而非实际版本号
- **环境**: macOS 26.2 (arm64), Homebrew 安装, CLI 2026.2.26
- **根因**: `resolveRuntimeServiceVersion()` 函数未找到版本环境变量，回退到 "dev"
- **影响**: 仅显示问题，无功能影响
- **临时方案**: `OPENCLAW_VERSION=2026.2.26 openclaw gateway start`
- **状态**: 📋 待提交给官方

<details>
<summary>详细技术分析</summary>

版本解析链：
1. `OPENCLAW_VERSION` 环境变量 - 未设置
2. `OPENCLAW_SERVICE_VERSION` 环境变量 - 未设置
3. `npm_package_version` 环境变量 - 未设置
4. 回退到 `"dev"` ← 问题根源

验证命令：
```bash
openclaw gateway call system-presence
# 显示 "version": "dev" 而非 "2026.2.26"
```

相关代码位置：
- `/opt/homebrew/Cellar/openclaw-cli/2026.2.26/libexec/lib/node_modules/openclaw/dist/env-vars-*.js`
- `/opt/homebrew/Cellar/openclaw-cli/2026.2.26/libexec/lib/node_modules/openclaw/dist/gateway-cli-*.js`
</details>
