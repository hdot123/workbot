- 改进目标：支持多 pane 自动发现与过滤，不再硬编码单一目标

### 新增目标选择模式

- **单 pane 模式**：`--target <spec>` 指定单个目标
- **自动发现模式**：`--auto` 发现所有 pane
- **过滤发现模式**：`--session`、`--window`、`--title`、`--path`、`--cmd` 组合过滤

### 新增参数

- `--auto`：自动发现所有 pane
- `--session <name>`：按 session 名称过滤（支持 glob *）
- `--window <idx>`：按 window 索引过滤

- `--cmd <pattern>`：按运行命令过滤（glob 匹配）

- 多个过滤器为 AND 逻辑
- `--auto` 可与过滤器组合使用
- 默认行为改为常驻模式（`--no-exit-on-alert`）

- 每个 pane 独立追踪状态（无变化计数、时间戳等）
- 单个 pane 目标缺失不会导致整体退出（除非 `--exit-on-alert`）
- 输出中 `[target]` 前缀标识来源
- 事件日志中每条包含 `target` 字段

## tmux-lookplus 改进完成（第一阶段）

- 改进时间：2026-03-26
- 改动范围：`scripts/tmux-look.sh`、`docs/tmux-lookplus.md`
- 改进目标：降低噪音、收口模式、强化结构化输出

### 新增参数

- `--no-change-threshold <n>`：设置 no_change 事件抑制阈值，默认 3 次连续无变化才记录

### 新增结构化输出

- `TMUX_LOOK_START`：启动时输出，包含配置信息
- `TMUX_LOOK_STILL_WAITING`：长时间无变化时输出，现已在主循环中实际触发
- `TMUX_LOOK_EXIT reason=alert`：告警退出时输出
- `TMUX_LOOK_DECISION`：命中需要指挥官决策的消息时输出，并附带上下文摘要

### 决策消息抓取策略

- 第一次命中需要指挥官决策的消息时，直接抓取完整上下文
- 后续同类消息重复命中时，再自动收窄为摘要输出
- 通过 `--decision-full-lines` 控制首次全量抓取上限
- 通过 `--decision-excerpt-lines` 控制后续摘要行数

### 新增事件类型

- `session_start` - 会话启动
- `no_change_suppressed` - 被跳过的无变化事件
- `still_waiting` - 长时间无变化
- `decision_needed` - 命中需要指挥官选择或确认的消息

### no_change 噪音降低机制

- 连续无变化次数 < threshold：不记录事件
- 连续无变化次数 >= threshold：记录一次 no_change，然后重置计数器
- 内容变化时：如有被抑制的 no_change，记录 `no_change_suppressed`

### 两种运行模式收口

- **常驻模式**：`--no-exit-on-alert`（默认），告警后继续观察，适合长期监控
- **告警即退出模式**：`--exit-on-alert`，命中告警立刻 exit 2，适合 Codex 即时接管

### 只读保证

- 脚本仅使用 `capture-pane` 和 `display-message` 读取状态
- 不使用 `send-keys` 或任何写入命令
- 不修改 pane 状态，不发送按键

## tmux-look 参数

### 目标选择

- `--target <spec>`：单个目标，可用 `tbot:0.0` 或 `%0` 这种格式
- `--auto`：自动发现所有 pane
- `--session <name>`：按 session 名称过滤（支持 glob *）
- `--window <idx>`：按 window 索引过滤
- `--title <pattern>`：按 window 标题过滤（glob 匹配）
- `--path <pattern>`：按 pane 当前路径过滤（glob 匹配）
- `--cmd <pattern>`：按运行命令过滤（glob 匹配）

### 其他选项

- `--interval`：可选，轮询间隔秒数，默认 `15`
- `--lines`：可选，每次抓取的历史行数，默认 `80`
- `--event-log`：可选，自定义事件日志文件路径，默认写入 `workspace/artifacts/tmux-look/events.jsonl`
- `--no-notify`：可选，关闭 macOS 本地通知
- `--patterns`：可选，自定义异常关键词，多个关键词用逗号分隔
- `--no-exit-on-alert`：可选，告警后不退出（默认行为）
- `--exit-on-alert`：可选，告警后立即退出
- `--show-no-change`：可选，每轮无变化时也打印状态
- `--no-change-threshold`：可选，连续无变化多少次才记录事件，默认 3
- `--decision-full-lines`：可选，首次命中决策消息时最多抓取多少行，默认 500
- `--decision-excerpt-lines`：可选，后续重复命中同类决策消息时摘要保留多少行，默认 20
- `--help`：显示帮助信息

## tmux-look 当前行为

- 启动时输出 `TMUX_LOOK_START` 结构化行，并记录 `session_start` 事件
- 若目标 pane 不存在，会记录 `target_missing` 事件（多 pane 模式下不立即退出）
- 若 pane 的当前命令不再是 `node`，会记录 `command_changed` 并发提醒
- 若命中异常，会输出 `TMUX_LOOK_ALERT ...` 结构化行
- 若目标不存在，会输出 `TMUX_LOOK_EXIT reason=target_missing target=<target>`
- 若告警退出，会输出 `TMUX_LOOK_EXIT reason=alert kind=<kind> target=<target>`
- 若本轮内容与上轮完全一致，累计无变化计数，达到阈值才记录 `no_change`
- 若内容有变化且命中异常关键词，会记录 `pattern_match` 并发提醒
- 若内容命中“请确认执行哪个任务 / 选项 A / 选项 B / Do you want to proceed”等决策消息，首次会抓完整上下文，后续再收窄为摘要
- 若内容有变化但未命中异常关键词，会记录 `content_changed`
- 长时间无变化时，输出 `TMUX_LOOK_STILL_WAITING` 并记录 `still_waiting`
- 多 pane 模式下，每个 pane 独立追踪状态，输出带 `[target]` 前缀

## tmux 指挥链踩坑复盘（2026-03-26）

### 事件背景

今天在通过 tmux 指挥 `dev-bot` 执行 AEdu 第一阶段任务时，出现了多次“看到了提示但没有及时处理”“消息挂在输入框里未真正发送”“把任务来源文档暴露给执行 bot”“把 Claude 审批框当成普通 shell 输入处理”等问题。

### 核心教训

1. `tmux-look` 的职责是提醒，不是替代处理。
2. 一旦收到 `TMUX_LOOK_ALERT` 或明显的 `content_changed`，指挥官必须立即去对应 pane 处理，不要先在外部解释脚本状态。
3. 在 Claude 聊天界面里发消息，不能只停在“粘贴内容”，必须确认消息已经真正进入聊天流，而不是还挂在输入框。
4. `This command requires approval` 这种界面是 Claude 内部审批 UI，不是普通 shell 提示；处理时要按 Claude 交互方式推进，并在每一步后立即复查是否真的生效。
5. `dev-bot` 不应自己维护任务表或 CE 状态。`dev-bot` 只负责改代码、跑最小验证、回报结果；任务状态由指挥官维护。
6. `claude` 进程启动归人工准备环节，指挥官默认从“pane 已经处于 Claude 聊天界面”开始接手；后续只负责丢身份、发任务、确认发送、观察和切下一条。
7. OCR 任务必须以资料前提为边界。没有资料前提时，不得把 OCR 深化任务混进主线开发。

### 固化规则

- 门铃一响就处理：收到 `tmux-look` 提醒后，先去 dev 窗口处理，再回来汇报。
- 任务表给指挥官看，任务指令给 bot。
- Claude 审批框必须“操作 + 复查”成对出现，不能只发按键不验收结果。
- 先锁边界，再发任务，避免 `dev-bot` 回头检查任务来源文档。

### 当前收口结论

这次问题不在 `tmux-look` 脚本本身，而在指挥官没有把提醒立刻转成动作。后续应严格执行“提醒 -> 窗口处理 -> 复查 -> 再汇报”的闭环。

### 审批框处理纠偏

- 今天现场验证后确认：Claude 审批框不应按普通 shell 交互处理。
- 旧版“先试 `Enter / 1 / 2 / Tab`”口径已被现场证伪。
- 当前已验证可用路径应改为：先用**方向键**移动到目标选项，再按 `Enter`，然后立即复查确认框是否消失。
