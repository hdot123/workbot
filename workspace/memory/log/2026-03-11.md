### 技术洞察

- **双协议实现**: ClawRouter 的核心能力是协议适配层，所有 chat 模型天然支持双协议
- **embeddings 本质**: 向量模型输出的是向量（数字数组），不是文本，只支持 OpenAI embeddings API
- **元数据设计**: `/v1/models` 的职责是"表达清楚能力"，而不是"暴露内部路由实现"

### 关键决策

- **不进行 v2.6.0 大重构**: 路由名称包含协议是合理的，不需要改
- **模型 ID 命名风格**: 保持现有 `glm-5-bailian` 格式，不改为 `bailian/glm-5`
- **只补强元数据**: 改进 `/v1/models` 的表达能力，不改路由架构
