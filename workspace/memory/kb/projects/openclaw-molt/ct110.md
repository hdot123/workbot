## 常用命令

### 快速恢复/部署（5 分钟）

```bash
# SSH 登录
ssh root@192.168.88.29

# 1. 停止旧进程
pkill mihomo

# 2. 清理 iptables
iptables -t nat -F PREROUTING
iptables -t nat -F POSTROUTING
iptables -F FORWARD

# 3. 启动 Mihomo
nohup /usr/local/bin/mihomo -d /root/.config/mihomo > /tmp/mihomo.log 2>&1 &

# 4. 配置 iptables 透明代理
iptables -t nat -A PREROUTING -d 192.168.88.29/32 -j RETURN
iptables -t nat -A PREROUTING -p tcp --dport 53 -j RETURN
iptables -t nat -A PREROUTING -p udp --dport 53 -j RETURN
iptables -t nat -A PREROUTING -d 10.0.0.0/8 -j RETURN
iptables -t nat -A PREROUTING -d 172.16.0.0/12 -j RETURN
iptables -t nat -A PREROUTING -d 127.0.0.0/8 -j RETURN
iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-ports 7892
iptables -t nat -A PREROUTING -p tcp --dport 443 -j REDIRECT --to-ports 7892
iptables -A FORWARD -i eth0 -o eth0 -j ACCEPT
iptables -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -t nat -A POSTROUTING -j MASQUERADE

# 5. 验证
ps aux | grep mihomo | grep -v grep
netstat -tlnp | grep -E '7890|7892|9090'
iptables -t nat -L PREROUTING -n -v
```

## 故障排查

### 常见问题

1. **节点加载不全**
   - 原因: 使用了第三方转换服务（丢失 vless/tuic）
   - 解决: 直接使用原始订阅链接

2. **HTTPS 代理 502**
   - wget 测试 HTTPS 会 502，但浏览器正常
   - 建议用浏览器测试

3. **Gemini 无法访问**
   - 检查是否走了 HK 组（香港不支持 Gemini）
   - 应该走 Gemini 组（JP/SG/US）

4. **透明代理不工作**
   - 检查设备网关是否设置为 `192.168.88.29`
   - 确认 iptables 规则：`iptables -t nat -L PREROUTING -n -v`
   - 确认 REDIRECT 计数器在增加
   - 检查 Mihomo 日志：`tail /tmp/mihomo.log`

5. **一键恢复脚本**
```bash
pkill mihomo
iptables -t nat -F PREROUTING && iptables -t nat -F POSTROUTING && iptables -F FORWARD
nohup /usr/local/bin/mihomo -d /root/.config/mihomo > /tmp/mihomo.log 2>&1 &
iptables -t nat -A PREROUTING -d 192.168.88.29/32 -j RETURN
iptables -t nat -A PREROUTING -p tcp --dport 53 -j RETURN && iptables -t nat -A PREROUTING -p udp --dport 53 -j RETURN
iptables -t nat -A PREROUTING -d 10.0.0.0/8 -j RETURN && iptables -t nat -A PREROUTING -d 172.16.0.0/12 -j RETURN && iptables -t nat -A PREROUTING -d 127.0.0.0/8 -j RETURN
iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-ports 7892
iptables -t nat -A PREROUTING -p tcp --dport 443 -j REDIRECT --to-ports 7892
iptables -A FORWARD -i eth0 -o eth0 -j ACCEPT && iptables -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -t nat -A POSTROUTING -j MASQUERADE
```
