# PM Bot Global Binding and Legacy Fence

> This active lesson replaces the missing active reference formerly pointed to by `pm-bot-crawl4ai-runtime-path.md`.

## Active Truth

- `pm-bot` is a global bot body bound into `workbot`.
- `workbot` activates `pm-bot` through project binding, not by inventing a new local ontology.
- `pm-bot` role body is 产品分析 / 模仿产品 / 整理需求 / 采集网站内容 / benchmarking / imitation analysis.
- If `clarification` appears, it only means product-side requirement organization, not main-thread task adjudication.
- `pm-bot` does not own task breakdown, scope convergence, acceptance framing, dispatch, closure, or adjudication.
- `pm-bot` tools boundary is separate from role body.
- Crawl4AI is not `pm-bot` current canonical owner truth.
- `pm-bot` control-chain truth is `dispatch_owner = codex` and `shared core + pm-specific special cases`.
- External `main-thread` remains outside the project bot set.
- `cmux-browser` remains a board surface, not a bot.
