对，但“**claude 是全局的**”只说明两件事：

一是这个可执行文件在你的 PATH 里，任何目录都能运行。  
二是你刚才 `codex mcp add` 默认加成了全局配置。

它**不等于** Claude Code 启动时会自动绑定到某个项目目录。`claude mcp serve` 官方示例只有 `command: "claude", args: ["mcp","serve"]`，没有任何“自动跟随当前仓库”的语义；它只是把 Claude Code 作为一个 MCP server 暴露出来。

真正决定它能看哪个目录的，是**启动时的工作目录**，以及你有没有显式加额外目录。Anthropic 官方文档说明

，Claude Code 默认访问启动它时所在目录的文件；如果要扩展范围，用 `--add-dir <path>`。
所以你现在这条：

```bash
codex mcp add claude-code -- claude mcp serve
```

含义是“无论你在哪个项目里用 Codex，都去启动一个全局可找到的 `claude` 命令”。  
因为配置里没有指定 `cwd`，所以 `codex mcp list` 才会显示 `Cwd -`。这也正是你现在看到的状态。

结论很简单：

- **全局安装**：只解决“能不能找到 `claude` 命令”
    
- **全局 MCP 配置**：只解决“这个 server 在所有项目里都可用”
    
- **项目目录**：仍然要你自己指定，或者在启动时进入该目录
    

对你来说，最稳有两种做法。

第一种，继续用全局 server，但每次让它进项目目录启动：

```bash
codex mcp remove claude-code
codex mcp add claude-code -- sh -lc 'cd ~/workbot && claude mcp serve'
```

第二种，不改 `cwd`，直接给 Claude 补访问目录：

```bash
codex mcp remove claude-code
codex mcp add claude-code -- claude --add-dir ~/workbot mcp serve
```

如果你主要就是服务 `~/workbot` 这个仓库，我更推荐第一种；语义更直接，Claude 的默认工作根也会更清楚。  
如果你准备让同一个 `claude-code` server 服务多个目录，再考虑 `--add-dir`。([Claude API Docs](https://docs.anthropic.com/fr/docs/claude-code/mcp?utm_source=chatgpt.com "Connecter Claude Code aux outils via MCP - Anthropic"))

你现在不用纠结“全局”这件事本身。关键点只有一句：

**全局 ≠ 自动指向当前项目。**
