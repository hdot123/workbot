> 最后更新：2026-03-21  
> 维护人：实施与试点运营负责人

> 用途：支撑 GRAPH 首轮骨架开发准入复核  
> 变更方式：后续修改需通过新一轮准入评审

---

## 1. 文档目的

本文档用于冻结 GRAPH 首轮试点的执行范围与责任边界，回答以下问题：

1. GRAPH 做到哪一步为止
2. 哪些结果不能直接暴露给下游
3. 哪些问题必须停在图谱层，不得提前推给 OBS / SIM
4. GRAPH 与 TWIN、OBS、SIM、人工复核之间如何划边界

---

## 2. 试点冻结范围

本轮试点范围冻结如下：

| 维度 | 冻结值 |
|------|--------|
| 地区 | 安徽省 |
| 学段 | 高中 |
| 年级 | 高一 |
| 学科 | 物理 |
| 教材版本 | `PHY_PEP_G1_V1` |
| 输入源 | 家长文本、老师反馈文本、单页扫描/OCR |
| 模块范围 | GRAPH 首轮骨架开发 |
| 允许目标 | 单学生图谱骨架、最小检索闭环、版本回滚 |
| 不在试点内 | 多地区、多学科、多教材版本、扩展输入源、全量联调 |

---

## 3. 试点内允许能力

本轮试点只允许以下能力进入开发与验收：

1. 单学生图谱骨架建设。
2. Event / Fact / Relation / Snapshot / Retrieval Unit 的最小写入闭环。
3. 对象级版本链和基础回滚。
4. GraphRAG 最小检索单元生成。
5. 审计留痕与追溯链维护。

---

## 4. 试点内禁止能力

以下能力本轮禁止提前塞入 GRAPH：

1. 面向家长或老师的完整解释文案输出。
2. 干预建议、模拟结论、反事实推演。
3. 群体聚合图谱、学校级全景图。
4. 多地区、多教材版本并存图谱。
5. 原始长文本直接图谱化。
6. 将 OCR 低置信候选直接固化为长期强关系。
7. 以 GRAPH 替代 OBS 的展示职责。
8. 以 GRAPH 替代 SIM 的推演职责。

---

## 5. GRAPH 与 TWIN 的边界

### GRAPH 负责

- 承接 TWIN 输出的结构化对象
- 沉淀长期记忆与关系结构
- 提供回滚、追溯和最小检索基础

### TWIN 负责

- 形成当前态
- 维护状态变化与风险质量摘要
- 决定哪些对象可以进入 GRAPH

### 边界结论

TWIN 是当前态核心，GRAPH 是长期记忆核心。  
GRAPH 不替代 TWIN 做当前态判定，TWIN 也不替代 GRAPH 做长期结构存储。

---

## 6. GRAPH 与 OBS 的边界

### GRAPH 负责

- 提供结构化检索基础
- 提供可追溯对象和关系链

### OBS 负责

- 角色化过滤
- 报告生成
- 展示文案与页面呈现

### 边界结论

GRAPH 输出“可用结构”，OBS 负责“可读表达”。  
GRAPH 不直接向家长端、老师端输出完整可读解释。

---

## 7. GRAPH 与 SIM 的边界

### GRAPH 负责

- 提供长期记忆、历史轨迹、结构关系和检索基础

### SIM 负责

- 干预策略生成
- 风险预测与反事实推演
- 建议输出与结果校准

### 边界结论

GRAPH 是推演输入基础，不是推演执行器。  
本轮试点不允许把 SIM 的职责提前塞进 GRAPH。

---

## 8. GRAPH 与人工复核的边界

### GRAPH 负责

- 保留低置信事件的最小可追溯记录
- 支持回滚和修正后重写

### 人工复核负责

- 裁定冲突引用
- 修正 OCR 误映射
- 决定是否允许回写有效对象

### 边界结论

GRAPH 不替代人工裁定高不确定性输入。  
人工复核完成后，GRAPH 才根据复核结果固化长期有效对象。

---

## 9. 责任边界表

| 角色 / 模块 | 负责 | 不负责 |
|-------------|------|--------|
| KB | 知识底座主数据和引用有效性 | 学生当前态与图谱长期写入裁定 |
| INGEST | 原始输入解析、标准事件生成 | 图谱长期存储 |
| TWIN | 当前态、状态变化、风险与质量摘要 | 长期图谱主存储 |
| GRAPH | 长期记忆、关系结构、版本回滚、最小检索闭环 | 前台展示、推演建议 |
| OBS | 展示、报告、角色化表达 | 图谱对象主写入 |
| SIM | 推演与建议输出 | 图谱基础存储与回滚 |
| 人工复核 | 裁定冲突和低置信对象 | 常规自动写入 |

---

## 10. 问题升级路径

出现问题时，按以下路径升级：

1. 知识引用无效或教材映射冲突：升级到 KB 负责人。
2. 标准事件不稳定或解析错误：升级到 INGEST 负责人。
3. 当前态与图谱结果不一致：升级到 TWIN 负责人。
4. 图谱对象误写、版本链异常、回滚失败：升级到 GRAPH 负责人。
5. 展示口径不一致：升级到 OBS 负责人。
6. 推演输入越界或推演结果冒入图谱：升级到 SIM 负责人。
7. OCR 低置信、关系冲突、对象归属争议：进入人工复核。

---

## 11. 本轮试点不做事项

本轮明确不做：

1. 多学生群体对比图谱。
2. 多学校并行试点图谱。
3. 多地区并行适配。
4. 多学科统一长期记忆。
5. 报告批量生产联调。
6. 干预模拟与建议落图。
7. 全量历史事件重建。
8. 原始全文检索替代结构检索。
