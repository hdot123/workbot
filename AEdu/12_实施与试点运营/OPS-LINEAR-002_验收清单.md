# Linear Webhook / n8n / Gateway-Admin 验收清单

> 文档编号：OPS-LINEAR-002  
> 版本：V1.0  
> 创建日期：2026-05-03  
> 维护人：bailian-worker  

---

## 一、方案背景

本项目将 Linear 从结构化计划工具升级为**自动化项目运营系统**。当前阶段优先落地两项基础设施：

1. **Webhook 消息处理管道**：接收 Linear Webhook 事件 → gateway-admin → n8n 工作流 → 事件入库/面板展示
2. **n8n 服务监控**：确保 n8n 实例持续可用，异常时告警

本文档为验收标准清单，按 P0 / P1 / P2 三级划分。

---

## 二、P0 验收标准（阻断级，必须通过）

### 2.1 DNS 与 HTTPS

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P0-DNS-01 | 域名解析生效 | `dig` / `nslookup` 查询配置的域名 | 返回正确 A/CNAME 记录，TTL ≤ 300s |
| P0-HTTPS-01 | HTTPS 证书有效 | `curl -I https://<域名>/health` | HTTP 200，证书未过期，链完整 |
| P0-HTTPS-02 | HTTP 自动跳转 HTTPS | `curl -I http://<域名>/health` | 返回 301/302 跳转至 HTTPS 地址 |
| P0-HTTPS-03 | TLS 版本 ≥ 1.2 | `openssl s_client -connect <域名>:443 -tls1_1` | TLS 1.1 及以下版本应被拒绝 |

### 2.2 n8n 服务健康

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P0-N8N-01 | n8n health endpoint 可达 | `curl https://<n8n域名>/healthz` | HTTP 200，响应体含 `"status": "ok"` |
| P0-N8N-02 | n8n 进程持续运行 | `systemctl status n8n` 或 Docker `docker ps` | 进程状态 active/running，uptime > 0 |
| P0-N8N-03 | n8n 数据库连接正常 | 在 n8n 中执行简单 workflow（如 Cron → No-op） | 执行成功，无数据库连接错误 |

### 2.3 Linear Test Event 端到端

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P0-LIN-01 | Linear Webhook 注册成功 | 在 Linear Settings → Webhooks 中创建并保存 | Webhook 状态为 active，Last ping 有时间戳 |
| P0-LIN-02 | gateway-admin 能接收 Linear 测试事件 | 在 Linear UI 触发 Test Event（如创建/更新 Issue） | gateway-admin 日志显示收到 POST 请求，HTTP 200 响应 |
| P0-LIN-03 | gateway-admin → n8n 转发成功 | 查看 n8n Webhook Node 执行记录 | n8n workflow 被触发，execution 状态为 success |
| P0-LIN-04 | 端到端延迟 < 5 秒 | 从 Linear 触发事件到 n8n 执行完成的时间 | 端到端时间 ≤ 5s |

### 2.4 事件入库

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P0-EVT-01 | Linear 事件落库成功 | 触发 Issue Created 事件后查询数据库 | 数据库中存在对应记录，字段完整（id, type, payload, created_at） |
| P0-EVT-02 | 事件 schema 符合约定 | 检查入库记录与预定义 schema 对比 | 所有必填字段存在，字段类型正确 |
| P0-EVT-03 | 事件去重 | 连续发送两次相同的 Linear 事件（相同 eventId） | 数据库只保留一条记录，或标记为 duplicate |

### 2.5 错误告警

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P0-ERR-01 | n8n 宕机告警 | 停止 n8n 服务，等待监控检查周期 | 告警渠道（如飞书/钉钉/邮件）收到告警通知，延迟 ≤ 监控周期 × 2 |
| P0-ERR-02 | Webhook 接收失败告警 | 模拟 gateway-admin 返回 500 | 告警系统记录错误并发出通知 |
| P0-ERR-03 | 告警恢复通知 | 恢复 n8n 服务 | 收到"服务已恢复"通知 |

### 2.6 安全拒绝

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P0-SEC-01 | 非法来源 IP 拒绝 | 从非 Linear IP 段（或伪造 IP）向 gateway-admin 发送请求 | HTTP 403，请求未转发至 n8n，日志记录拒绝原因 |
| P0-SEC-02 | 缺少签名/Token 的请求拒绝 | 发送不带 Linear Webhook 签名头的请求 | HTTP 401/403 |
| P0-SEC-03 | 签名验证失败拒绝 | 发送带错误签名的请求（篡改 body） | HTTP 403，签名不匹配 |

### 2.7 幂等

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P0-IDEM-01 | 重复事件不产生重复入库 | 同一 Linear eventId 连续发送 3 次 | 数据库仅 1 条有效记录，其余标记或丢弃 |
| P0-IDEM-02 | n8n workflow 重试不产生副作用 | 模拟 n8n 执行失败后自动重试 | 重试成功后，数据状态与首次成功一致 |

## 三、P1 验收标准（重要，上线前应完成）

### 3.1 DNS 与 HTTPS（P1）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P1-DNS-01 | DNS 故障自动切换（如有多 A 记录） | 停掉一个 IP，验证域名仍可解析到存活 IP | 解析切换时间 ≤ DNS TTL |
| P1-HTTPS-01 | 证书自动续期验证 | 检查 certbot / acme 配置和 cron | 证书到期前 30 天自动续期，无告警 |

### 3.2 n8n 服务健康（P1）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P1-N8N-01 | n8n 资源使用正常 | `docker stats` / `top` 检查 CPU/内存 | CPU < 80%，内存无持续增长（无泄漏） |
| P1-N8N-02 | 日志轮转 | 检查日志文件大小和轮转配置 | 单日志文件 ≤ 100MB，历史日志自动压缩/清理 |

### 3.3 事件面板（P1）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P1-PNL-01 | 面板展示最近事件列表 | 访问事件面板 URL | 显示最近 N 条事件，含类型、时间、状态列 |
| P1-PNL-02 | 事件过滤 | 按事件类型/时间范围过滤 | 筛选结果正确 |
| P1-PNL-03 | 事件详情 | 点击事件查看 payload 详情 | 完整展示 Linear webhook payload 的 JSON |
| P1-PNL-04 | 面板数据延迟 | 触发新事件后刷新面板 | 新事件 ≤ 30s 出现在面板上 |

### 3.4 错误告警（P1）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P1-ERR-01 | 告警分级 | 查看告警配置 | P0 级别即时通知（≤1min），P1 级别 5min 内通知 |
| P1-ERR-02 | 告警静默期 | 连续触发同一告警 | 静默期内不重复通知（如 15min 内同一告警只通知 1 次） |
| P1-ERR-03 | 数据库入库失败告警 | 模拟数据库不可用 | 收到入库失败告警 |

### 3.5 安全（P1）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P1-SEC-01 | 请求体大小限制 | 发送 > 配置上限的 payload（如 10MB） | HTTP 413，未触发 n8n workflow |
| P1-SEC-02 | 速率限制 | 短时间内发送大量请求（如 100 req/min） | 超出阈值后返回 HTTP 429 |
| P1-SEC-03 | 敏感字段脱敏 | 检查入库/面板中的 payload | Linear API key 等敏感信息不在日志或面板中明文暴露 |

## 四、P2 验收标准（加分项，后续迭代）

### 4.1 DNS 与 HTTPS（P2）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P2-DNS-01 | CDN 缓存命中率 | 查看 CDN 监控面板 | 静态资源缓存命中率 ≥ 90%（如有 CDN） |
| P2-HTTPS-01 | HSTS 头存在 | `curl -I https://<域名>` 检查响应头 | 含 `Strict-Transport-Security` 头 |

|------|--------|----------|----------|

| P2-N8N-02 | Workflow 版本控制 | 导出 workflow 配置并与 Git 对比 | workflow 配置可版本化，支持回滚 |

### 4.3 事件面板（P2）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P2-PNL-01 | 面板支持事件统计图表 | 查看面板统计页 | 显示事件量趋势、类型分布等图表 |
| P2-PNL-02 | 面板支持事件搜索 | 按关键词/issue title 搜索 | 搜索结果命中相关事件 |
| P2-PNL-03 | 面板 RBAC 权限控制 | 用不同角色登录面板 | 不同角色看到的数据/操作权限不同 |

### 4.4 错误告警（P2）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P2-ERR-01 | 告警聚合与周报 | 检查告警统计功能 | 每周生成告警汇总报告，含趋势分析 |
| P2-ERR-02 | 自愈能力 | n8n 异常退出后 | 自动重启，恢复时间 ≤ 2min |

### 4.5 安全（P2）

| 编号 | 验收项 | 验收方法 | 预期结果 |
|------|--------|----------|----------|
| P2-SEC-01 | 审计日志 | 查看所有 gateway-admin 操作 | 完整的请求/响应审计日志，保留 ≥ 90 天 |
| P2-SEC-02 | Webhook 密钥轮换 | 更换 Linear Webhook signing secret | 新密钥立即生效，旧密钥在宽限期后失效 |

## 五、验收通过判定规则

| 等级 | 通过标准 |
|------|----------|
| **P0** | 全部通过。任何一项 P0 不通过 → 不允许上线 |
| **P1** | ≥ 80% 通过。剩余项须有明确的修复计划和时间表 |
| **P2** | 不阻塞上线，按后续迭代优先级排入 backlog |
