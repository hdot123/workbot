## 1. 文档目的

本文档用于定义 SIM 模块（推演与决策层）在首轮试点中的范围与责任边界，确保试点工作在可控范围内开展，避免范围蔓延和职责模糊。

## 3. SIM 责任边界

### 3.1 SIM 负责的事项

| 责任项 | 说明 |
|--------|------|
| 干预动作模拟 | 评估不同干预动作的潜在效果 |
| 风险趋势预测 | 预测学生学习风险变化方向 |
| 动作比较排序 | 对候选干预动作进行优先级排序 |
| 影子推演执行 | 后台持续模拟不同干预路径 |
| 建议分级控制 | 按风险等级控制建议开放范围 |
| 推演结果校准 | 记录推演结果与实际结果对比 |

### 3.2 SIM 不负责的事项

| 非责任项 | 说明 | 负责模块 |
|----------|------|----------|
| 学生状态计算 | 学生状态的计算和更新 | TWIN |
| 图谱检索与分析 | 知识图谱的检索和分析 | GRAPH |
| 干预动作定义 | 干预动作库的维护 | SIM-003 |
| 前台展示 | 推演结果的前台展示 | OBS |
| 高风险决策 | 高风险决策的最终判断 | 人工复核 |

### 3.3 与上游模块的边界

| 边界场景 | SIM 职责 | 上游职责 |
|----------|----------|----------|
| 学生状态错误 | 不负责校验，仅负责推演 | TWIN 负责状态正确性 |
| 图谱检索失败 | 负责降级推演 | GRAPH 负责检索稳定性 |
| 数据质量低 | 负责降低输出强度 | TWIN 负责数据质量评估 |
| 推演结果不可用 | 负责降级或拒绝 | 上游负责输入正确性 |

## 4. 影子推演试点规则

### 4.1 触发条件

| 条件 | 说明 |
|------|------|
| 风险信号持续 | 学生风险等级 ≥ medium 且持续 3 天以上 |
| 数据完整性达标 | data_completeness_score ≥ 0.70 |
| 老师触发 | 老师主动发起推演请求 |

### 4.2 执行规则

| 规则 | 说明 |
|------|------|
| 仅后台执行 | 影子推演结果仅后台可见，不直接外显 |
| 低风险动作优先 | MVP 阶段仅模拟低风险动作（学习内容类、节奏管理类） |
| 7/14 天窗口 | 仅支持短期和中期窗口推演 |
| 方向性输出 | 输出趋势和方向，不输出精确数值承诺 |

### 4.3 结果查看权限

| 角色 | 查看权限 |
|------|----------|
| 后台运营 | 完整推演结果 |
| 老师 | 筛选后的推演摘要 |
| 家长 | 仅低风险建议（经筛选后） |
| 学校管理员 | 本轮不开放 |

## 5. 干预建议开放规则

### 5.1 建议分层

| 层级 | 内容 | 家长端开放 | 老师端开放 |
|------|------|------------|------------|
| 观察提示层 | 基于当前状态的观察提示 | ✅ | ✅ |
| 低风险行动提示层 | 低风险、低成本行动建议 | ✅ | ✅ |
| 角色化受控建议层 | 需要角色配合的建议 | ❌ | ✅ |
| 高风险后台建议层 | 高风险或高敏感建议 | ❌ | ❌（仅后台） |

### 5.2 开放控制

| 风险等级 | 家长端 | 老师端 | 后台 |
|----------|--------|--------|------|
| low | 可开放 | 可开放 | 可查看 |
| medium | 经筛选后开放 | 可开放 | 可查看 |
| high | 不开放 | 经筛选后开放 | 需人工复核 |
| pending_confirmation | 不开放 | 可查看（附说明） | 可查看 |

## 6. 试点问题升级机制

### 6.1 问题分类

| 问题类型 | 升级路径 | 处理时效 |
|----------|----------|----------|
| 推演结果错误 | SIM 开发团队 | 1 个工作日 |
| 风险判断错误 | SIM 开发团队 + 业务团队 | 1 个工作日 |
| 上游数据问题 | TWIN/GRAPH 团队 | 2 个工作日 |
| 用户反馈问题 | 运营团队 + SIM 团队 | 3 个工作日 |
| 重大故障 | 全团队紧急响应 | 4 小时 |

### 6.2 升级条件

出现以下情况时，需立即升级处理：

- 推演结果严重错误（如高风险建议误判为低风险）
- 用户无法正常使用核心功能
- 大面积用户投诉
- 数据泄露或安全问题

## 7. 试点成功标准

### 7.1 功能指标

| 指标 | 目标值 |
|------|--------|
| 推演任务创建成功率 | ≥ 99% |
| 推演任务响应时间（P95） | < 2s |
| 风险预测创建成功率 | ≥ 99% |
| 风险预测响应时间（P95） | < 1s |
| 数据不足降级覆盖率 | 100% |
| 高风险控制覆盖率 | 100% |

### 7.2 校准指标

| 指标 | 目标值 |
|------|--------|
| 推演可校准率 | ≥ 80% |
| 推演方向一致率 | ≥ 70% |
| 风险判断一致率 | ≥ 75% |

### 7.3 用户满意度指标

| 指标 | 目标值 |
|------|--------|
| 老师端满意度 | ≥ 80% |
| 用户愿意继续使用比例 | ≥ 70% |

## 8. 试点阻断条件

出现以下任一情况时，试点必须暂停：

- 核心功能无法使用（推演/风险预测）
- 高风险建议误开放给家长端
- 大面积用户投诉（> 20% 用户投诉）
- 数据安全问题
- 上游模块不可用导致 SIM 无法工作

---

## 9. 试点范围扩展条件

满足以下条件后，可考虑扩展试点范围：

- 首轮试点成功标准达成
- 无 P0 阻断问题
- 用户满意度达标
- 运营团队同意扩展

扩展范围包括：

- 扩展到更多班级
- 扩展到更多学校
- 扩展功能范围（群体推演等）

## 10. 与其他模块的协作边界

### 10.1 与 TWIN 的关系

SIM 消费 TWIN 状态，但不修改 TWIN 状态。

### 10.2 与 GRAPH 的关系

SIM 消费 GRAPH 检索结果，用于时序记忆和图谱上下文。

### 10.3 与 OBS 的关系

SIM 向 OBS 提供经筛选的低风险建议，高风险建议不直接输出。

### 10.4 与 RULE 的关系

SIM 消费 RULE 的规则约束，用于约束推演边界。

---

## 11. 结论

SIM 首轮试点的范围、责任边界、影子推演规则、建议开放规则、升级机制和成功标准已定义。

试点工作应严格在冻结范围内开展，避免范围蔓延和职责模糊。

影子推演结果仅后台可见，家长端仅开放低风险内容，高风险建议必须人工复核。
