---

## 1. 文档目的

本文档用于定义 TWIN 在首轮骨架开发中的最小可验收集合，回答以下问题：

1. TWIN 这一轮到底验什么
2. 什么算过
3. 什么算不过
4. 什么算可降级通过

---

## 2. 验收对象清单

本轮最小验收对象包括：

1. 冻结范围与学生绑定门禁
2. 标准事件准入门禁
3. 事实层更新
4. 行为层更新
5. 演化层刷新
6. 风险层刷新
7. 数据质量层刷新
8. 降级处理路径
9. `review_needed` 路径
10. `reject` 路径
11. 生命周期状态流转
12. 审计与追溯

---

## 3. 验收维度定义

| 维度 | 定义 |
|------|------|
| 正确性 | TWIN 的处理是否符合输入契约和状态映射规则 |
| 一致性 | 输入、引用、状态结果之间是否保持一致 |
| 完整性 | 进入链路的对象是否具备最低必需信息 |
| 可追溯性 | 状态变化是否能回到事件、证据和版本 |
| 可降级性 | 输入不完整时是否能安全降级，而不是硬更新或直接崩溃 |

---

## 4. 模块级最小验收矩阵表

| 验收对象 | 输入条件 | 预期行为 | 预期结果 | 验收维度 | 判定方式 | 可降级通过 |
|----------|----------|----------|----------|----------|----------|------------|
| 冻结范围门禁 | 事件来自安徽 / 高中 / 高一 / 物理 / `PHY_PEP_G1_V1` | 允许进入 TWIN 门禁校验 | 范围内事件继续，范围外事件被拦截 | 正确性 / 一致性 | 抽样校验 3 类范围内事件和 1 类范围外事件 | 否 |
| 标准事件准入 | 事件具备必需字段 | 进入输入契约校验 | 合法事件才进入更新链 | 完整性 / 正确性 | 缺字段事件不得进入更新链 | 否 |
| 事实层更新 | 有合法结果或知识引用 | 更新事实快照 | `latest_homework_summary`、`current_focus_points` 等变化正确 | 正确性 / 一致性 | 对照输入事件与更新结果检查 | 是 |
| 行为层更新 | 事件带有明确行为线索 | 更新行为摘要 | 节奏、订正、反馈摘要正确变化 | 正确性 / 完整性 | 对照行为线索与摘要输出检查 | 是 |
| 演化层刷新 | 已有当前事件与历史窗口 | 计算趋势摘要 | 趋势方向与窗口事实一致 | 正确性 / 一致性 | 检查趋势是否与前后事件一致 | 否 |
| 风险层刷新 | 有足够事实支撑风险变化 | 更新风险标签或状态 | 不出现“无事实支撑的高风险提升” | 正确性 / 可追溯性 | 检查风险变化是否能追到事实证据 | 否 |
| 数据质量层刷新 | 事件进入处理链 | 更新质量分 | 质量分与完整度、时效性、置信度一致 | 正确性 / 可追溯性 | 抽查 `state_confidence_score` 与 `confidence_score` 对齐关系 | 是 |
| 降级路径 | 事件缺失部分知识引用，但仍可形成行为或章节级摘要 | 限定更新范围 | 允许 `degraded`，但不得伪造知识掌握结论 | 可降级性 / 正确性 | 抽查降级事件是否只更新允许层级 | 是 |
| `review_needed` 路径 | 低置信、冲突、歧义事件 | 不进入主状态链 | 生成复核标记与追踪引用，不直接改关键状态 | 正确性 / 可追溯性 | 检查 `review_ticket_ref` 与无关键状态漂移 | 否 |
| `reject` 路径 | 关键字段缺失、范围外、来源非法 | 直接拒绝 | 不产生状态改写 | 正确性 / 完整性 | 检查拒绝日志和无状态变化 | 否 |
| 生命周期流转 | 学生对象经历绑定、冷启动、激活、稳定、复核 | 合法流转 | 可在 `pending_binding / cold_start / active / stable / review_needed` 之间按规则变化 | 一致性 / 可追溯性 | 检查状态流转日志 | 否 |
| 审计与追溯 | 任何被消费事件 | 生成差异与追踪引用 | 至少具备 `last_event_ref / trace_id / raw_input_ref` | 可追溯性 / 完整性 | 抽查 3 个已消费事件 | 否 |

## 5. 最小通过标准

TWIN 首轮骨架开发视为通过最小验收，至少要满足：

1. 3 类冻结输入源都能被正确分流。
2. 正常事件能更新事实层与行为层。
3. 有历史窗口时能刷新基础演化层。
4. 风险层不会被单条低质事件误改。
5. 引用不足时能安全降级。
6. 低置信事件能进入 `review_needed`。
7. 被接受的事件都能追溯到来源和版本。

---

## 6. 可降级通过标准

以下情况允许“可降级通过”：

1. `knowledge_refs` 缺失，但 `chapter_refs` 合法，且事件只更新章节级事实或行为摘要。
2. 家长或老师文本无法稳定映射知识点，但能形成明确行为摘要。
3. `anchor_refs` 缺失，但知识引用已稳定命中。
4. 单次事件不足以刷新演化层或风险层，但事实层和行为层已正确更新。

可降级通过的前提是：

- 必须明确记录降级原因
- 必须限制更新范围
- 不得伪造完整成功状态

---

## 7. 不通过判定条件

出现以下任一情况，即判定不通过：

1. 范围外事件进入主状态更新链。
2. 缺少关键字段的事件仍被消费。
3. `review_needed` 事件直接改写知识掌握或高风险状态。
4. `hold` 事件越权改写事实层关键字段。
5. 只有 `anchor_refs` 的事件直接修改知识掌握状态。
6. 状态变化找不到 `last_event_ref`、`trace_id` 或 `raw_input_ref`。
7. TWIN 直接生成报告、建议或推演结论来冒充下游模块。
