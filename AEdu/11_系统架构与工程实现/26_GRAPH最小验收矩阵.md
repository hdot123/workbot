# GRAPH 最小验收矩阵

> 文档编号：ARCH-026  
> 版本：V1.0  
> 创建日期：2026-03-21  
> 最后更新：2026-03-21  
> 维护人：系统架构与工程实现负责人  
> 状态：已冻结（首轮骨架开发基线）  
> 用途：支撑 GRAPH 首轮骨架开发准入复核  
> 变更方式：后续修改需通过新一轮准入评审

---

## 1. 文档目的

本文档用于定义 GRAPH 在首轮骨架开发中的最小可验收集合，回答以下问题：

1. GRAPH 这一轮到底验什么
2. 什么算过
3. 什么算不过
4. 什么情况下允许降级通过
5. 什么情况下必须具备回滚能力

---

## 2. 验收对象清单

本轮最小验收对象包括：

1. 冻结范围门禁
2. Event 写入
3. Fact 写入与替换
4. Relation 写入与降级
5. Snapshot 固化
6. Retrieval Unit 刷新
7. 版本链与回滚
8. 审计与追溯
9. 最小检索闭环

---

## 3. 验收维度定义

| 维度 | 定义 |
|------|------|
| 正确性 | 写入、替换、失效和检索是否符合契约与映射规则 |
| 一致性 | Event、Fact、Relation、Snapshot 之间是否能对齐 |
| 完整性 | 进入主链的对象是否具备最低必需信息 |
| 可追溯性 | 任一结果是否能回到来源事件、快照和版本 |
| 可回滚性 | 关键对象写错时是否能恢复到上一稳定版本 |

---

## 4. 模块级最小验收矩阵表

| 验收对象 | 输入条件 | 预期行为 | 预期结果 | 验收维度 | 判定方式 |
|----------|----------|----------|----------|----------|----------|
| 冻结范围门禁 | 对象来自安徽 / 高中 / 高一 / 物理 / `PHY_PEP_G1_V1` | 允许进入 GRAPH 主链 | 范围内对象继续，范围外对象拦截 | 正确性 / 一致性 | 抽样校验范围内外各 1 组 |
| Event 写入 | 合法标准事件或合法状态更新事件 | append 写入时序记忆 | 事件可查询，且带 `trace_id` | 正确性 / 完整性 / 可追溯性 | 对照事件输入与图谱结果检查 |
| Fact 写入 | 当前态事实合法且可追溯 | create 或 supersede | 当前有效 Fact 正确，旧版本保留 | 正确性 / 一致性 / 可回滚性 | 检查事实版本链 |
| Relation 写入 | 结构关系稳定且引用合法 | create / upsert | 稳定关系正确建边 | 正确性 / 一致性 | 核对知识与学生关系 |
| Relation 降级 | 只有 Chapter 级或行为级信息 | 不强建 KnowledgePoint 强关系 | 保留 Event / Snapshot，限制强关系 | 正确性 / 完整性 | 检查降级对象无越权强关系 |
| Snapshot 固化 | 当前态已稳定、来源链完整 | 固化状态快照 | 快照可回溯到事实和事件 | 一致性 / 可追溯性 | 抽查快照引用链 |
| Retrieval Unit 刷新 | 主对象写入成功 | 生成最小检索单元 | 可召回当前态 + 结构 + 关键事件 | 正确性 / 完整性 | 执行最小检索样例 |
| 版本链与回滚 | 某关键 Fact 或 Relation 被误写 | 支持回滚到上一稳定版本 | 当前态恢复正确，旧链不断裂 | 可回滚性 / 可追溯性 | 演练 1 组对象级回滚 |
| 审计与追溯 | 任一成功写入或回滚操作 | 记录审计链 | 操作可追踪 | 可追溯性 / 完整性 | 抽查 3 条审计记录 |
| 最小检索闭环 | 单学生查询当前态解释 | 召回当前事实、关键关系、近窗口事件 | 检索上下文完整且低噪音 | 正确性 / 一致性 | 执行端到端样例比对 |

## 5. 最小通过标准

GRAPH 首轮骨架开发视为通过最小验收，至少要满足：

1. 合法 Event 能成功写入。
2. 当前关键 Fact 能正确 supersede。
3. 稳定 Relation 能正确落图。
4. Snapshot 能在关键时点成功固化。
5. Retrieval Unit 至少能支撑单学生最小检索闭环。
6. 关键对象误写时可以对象级回滚。
7. 成功对象都具备追溯链。

---

## 6. 可降级通过标准

以下情况允许“可降级通过”：

1. `knowledge_refs` 缺失，但 `chapter_refs` 合法，GRAPH 只保留 Event / Snapshot 和 Chapter 级挂载。
2. Retrieval Unit 刷新失败，但 Event / Fact / Relation / Snapshot 主链已成功。
3. 仅增强关系写入失败，不影响当前主事实与核心结构关系。

可降级通过的前提是：

- 必须记录降级原因
- 必须明确哪些能力未就绪
- 不得伪造“结构完整”和“检索完整”结论

---

## 7. 不通过判定条件

出现以下任一情况，即判定不通过：

1. 范围外对象进入主写入链。
2. 缺少 `student_id / trace_id / raw_input_ref` 的对象仍被落图。
3. 低置信事件直接建强知识关系。
4. `review_needed` 中间状态被直接固化成长期有效 Fact。
5. 对象回滚后版本链断裂。
6. 图谱主链成功，但检索结果混入大量不相关旧事件。
7. GRAPH 直接生成前台解释或推演结论来冒充 OBS / SIM。
