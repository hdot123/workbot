## 1. 文档目的

本文档用于定义风险预测服务的服务设计，包括模块职责、输入输出、预测逻辑边界、输出契约、回退机制、日志追踪要求。

---

## 2. 模块职责

### 2.1 核心职责

| 职责 | 说明 |
|------|------|
| 学习风险识别 | 识别学生学习稳定性、知识掌握、进度偏离、节奏中断等风险 |
| 风险等级评估 | 输出四级风险等级（low/medium/high/pending_confirmation） |
| 风险方向判断 | 判断风险变化方向（up/stable/down/uncertain） |
| 风险因子分析 | 识别主要风险因子，支持解释 |

### 2.2 不负责

| 非职责 | 负责模块 |
|--------|----------|
| 干预动作模拟 | 推演引擎服务（ARCH-032） |
| 干预建议生成 | SIM-007 干预建议开放条件 |
| 前台展示 | OBS 模块 |
| 人格/心理诊断 | 不在系统范围内 |

## 3. 输入输出

| 输入项 | 类型 | 来源 | 必需 | 说明 |

| student_id | string | 调用方 | 是 | 学生标识 |
| current_state_ref | string | TWIN | 是 | 学生当前状态引用 |
| event_sequence_refs | array | GRAPH | 是 | 学习事件序列引用 |
| knowledge_dependency_ref | string | KB | 否 | 知识点依赖图引用 |

## 4. 预测逻辑边界

### 4.1 支持的风险类型

| 风险类型 | 触发信号 | MVP 支持 |
|----------|----------|----------|
| 学习稳定性风险 | 成绩波动、作业质量不稳定 | ✅ |
| 知识掌握风险 | 重复错误、知识点退化 | ✅ |
| 进度偏离风险 | 当前进度落后教学计划 | ✅ |
| 节奏中断风险 | 学习频率下降、延迟提交增多 | ✅ |
| 干预滞后风险 | 风险信号出现但无干预记录 | ✅ |

### 4.2 不支持的风险类型

| 风险类型 | 原因 |
|----------|------|
| 人格特质风险 | 超出教育场景范围 |
| 心理诊断风险 | 需要专业资质 |
| 医疗相关风险 | 超出系统边界 |
| 长期人生走向预测 | 置信度无法保证 |

### 4.3 时间窗口限制

| 窗口 | 说明 | MVP 支持 |
|------|------|----------|
| 短期（1-2 周） | 即时干预、快速响应 | ✅ |
| 中期（3-6 周） | 阶段调整、路径优化 | ✅ |
| 阶段（1-2 月） | 学期规划参考 | ⚠️ 降低输出强度 |
| 长期（>2 月） | 战略决策参考 | ❌ 不支持 |

## 5. 输出契约

### 5.1 风险等级定义

| 等级 | 含义 | 处理方式 |
|------|------|----------|
| low | 低风险 | 正常观察，可进入建议层 |
| medium | 中风险 | 需关注，建议老师端查看 |
| high | 高风险 | 需人工复核，不直接前台输出 |
| pending_confirmation | 待确认 | 证据不足但出现弱信号 |

### 5.2 风险方向定义

| 方向 | 含义 | 使用场景 |
|------|------|----------|
| up | 风险上升 | 需要关注 |
| stable | 风险平稳 | 持续观察 |
| down | 风险下降 | 可适当放松关注 |
| uncertain | 方向不确定 | 需要更多数据 |

### 5.3 输出约束

- 高风险（high）结论必须附加人工复核标记
- 长时间窗口（>4 周）结论必须降低输出强度
- 数据完整性 < 0.50 时，只能输出 pending_confirmation 或"数据不足"

---

## 6. 回退机制

### 6.1 数据不足回退

| 数据完整性评分 | 输出策略 |
|----------------|----------|
| ≥ 0.70 | 正常输出风险等级和方向 |
| 0.50 ~ 0.70 | 输出风险等级，附加边界说明，置信度标记为低 |
| < 0.50 | 输出 pending_confirmation 或"数据不足" |

### 6.2 上游服务不可用回退

| 上游服务 | 不可用时的处理 |
|----------|----------------|
| TWIN | 拒绝风险预测，返回"学生状态不可用" |
| GRAPH | 使用缓存的历史预测结果，或返回"服务暂时不可用" |
| KB | 使用默认知识点依赖，附加边界说明 |

### 6.3 预测失败回退

| 失败场景 | 回退策略 |
|----------|----------|
| 预测超时 | 返回最近一次成功预测结果，标记为"历史参考" |
| 规则冲突 | 选择保守结论，附加冲突说明 |
| 多来源数据冲突 | 使用主数据源，附加冲突说明 |

## 7. 日志追踪要求

### 7.1 关键日志点

| 日志点 | 级别 | 字段 |
|--------|------|------|
| 预测任务创建 | INFO | risk_prediction_id, student_id, time_window |
| 输入加载 | DEBUG | state_ref, event_refs, data_completeness |
| 特征计算 | DEBUG | feature_values, feature_sources |
| 预测执行 | INFO | risk_prediction_id, duration_ms, risk_level |
| 预测完成 | INFO | risk_prediction_id, risk_level, risk_direction, confidence |
| 预测失败 | ERROR | risk_prediction_id, error_type, error_message |

### 7.2 追踪字段

- 每次预测必须包含 `trace_id`
- 所有日志必须包含 `student_id` 和 `risk_prediction_id`
- 关键决策点必须记录 `feature_values` 和 `decision_rule`

## 8. 核心接口

### 8.1 创建风险预测

```
POST /api/v1/risk-predictions
```

**请求体**：
```json
{
  "student_id": "stu_001",
  "time_window": "14d",
  "risk_types": ["knowledge_mastery_risk", "rhythm_interrupt_risk"]
}
```

**响应**：
```json
{
  "risk_prediction_id": "RISK_20260322_001",
  "status": "pending",
  "created_at": "2026-03-22T10:00:00Z"
}
```

### 8.2 查询预测结果

```
GET /api/v1/risk-predictions/{risk_prediction_id}
```

### 8.3 批量预测

```
POST /api/v1/risk-predictions/batch
```
