## 1. 文档目的

本文档用于定义推演引擎（InterventionSimulator）的服务设计，包括模块职责、输入输出、核心接口、依赖关系、错误处理与降级策略。

---

## 2. 模块职责

### 2.1 核心职责

| 职责 | 说明 |
|------|------|
| 干预动作模拟 | 评估不同干预动作在当前学生状态下的潜在效果 |
| 动作比较排序 | 对多个候选动作进行相对优先级排序 |
| 方向性输出 | 输出趋势、方向、可能性区间，而非单点承诺 |
| 后台影子推演 | 支持后台模拟，不直接对前台输出高风险结论 |

### 2.2 不负责

| 非职责 | 负责模块 |
|--------|----------|
| 风险预测 | 风险预测服务（ARCH-033） |
| 动作定义 | 干预动作库（SIM-003） |
| 学生状态计算 | TWIN 模块 |
| 图谱检索 | GRAPH 模块 |

## 3. 输入输出

### 3.1 输入

| 输入项 | 类型 | 来源 | 必需 | 说明 |
|--------|------|------|------|------|
| target_scope | object | 调用方 | 是 | 模拟目标范围（学生/班级） |
| current_state_ref | string | TWIN | 是 | 学生当前状态引用 |
| timeline_memory_refs | array | GRAPH | 是 | 时序记忆引用列表 |
| action_candidates | array | SIM-003 | 是 | 候选干预动作列表 |
| simulation_window | string | 调用方 | 是 | 模拟窗口（7d/14d/30d） |
| rule_constraints | object | RULE/KB | 否 | 规则约束集合 |
| output_policy | object | 调用方 | 否 | 输出策略配置 |

### 3.2 输出

```json
{
  "simulation_id": "SIM_20260322_001",
  "target_scope": {
    "type": "student",
    "ref": "stu_001"
  },
  "simulation_window": "14d",
  "action_results": [
    {
      "action_id": "ACT_PHY_PRACTICE_001",
      "action_name": "受力分析专项训练",
      "expected_direction": "likely_improve",
      "expected_affected_objects": ["PHY_PEP_G1_KP_002", "exercise_accuracy"],
      "confidence_band": "medium",
      "preconditions": ["老师安排专项讲解", "学生完成对应练习"],

  "created_at": "2026-03-22T10:00:00Z",
  "trace_id": "trace_sim_001"

```

## 4. 核心接口

### 4.1 创建推演任务

```
POST /api/v1/simulations
```

**请求体**：
```json
{
  "target_scope": {"type": "student", "ref": "stu_001"},
  "action_candidates": ["ACT_PHY_PRACTICE_001", "ACT_HOMEWORK_TIME_SHIFT"],
  "simulation_window": "14d",
  "output_policy": {"max_results": 5, "min_confidence": "low"}
}
```

**响应**：
```json
{
  "simulation_id": "SIM_20260322_001",
  "status": "pending",
  "created_at": "2026-03-22T10:00:00Z"
}
```

### 4.2 查询推演结果

```
GET /api/v1/simulations/{simulation_id}
```

**响应**：见 3.2 输出示例

### 4.3 取消推演任务

```
DELETE /api/v1/simulations/{simulation_id}
```

## 6. 错误处理与降级策略

### 6.1 错误分类

| 错误类型 | 错误码 | 处理策略 |
|----------|--------|----------|
| 输入校验失败 | 400 | 拒绝请求，返回具体校验错误 |
| 数据不足 | 422 | 返回"数据不足"结论，建议补充输入 |
| 上游服务不可用 | 503 | 降级为简化推演或返回"服务暂时不可用" |
| 推演超时 | 504 | 返回部分结果或"推演超时" |
| 规则冲突 | 409 | 标记冲突，降低输出强度 |

### 6.2 数据不足降级规则

| 数据完整性评分 | 推演策略 | 输出强度 |
|----------------|----------|----------|
| ≥ 0.70 | 正常推演 | 完整输出 |
| 0.50 ~ 0.70 | 降级推演 | 附加强边界说明，置信度标记为 low |
| < 0.50 | 拒绝推演 | 返回"数据不足"，不执行推演 |

### 6.3 上游缺失降级规则

| 上游缺失场景 | 降级策略 |
|--------------|----------|
| TWIN 状态缺失 | 拒绝推演 |
| 时序记忆缺失 | 降低置信度，附加强边界说明 |
| 动作库不可用 | 拒绝推演 |
| 规则约束缺失 | 使用默认规则，附加强边界说明 |

## 8. 日志与追踪

### 8.1 关键日志点

| 日志点 | 级别 | 字段 |
|--------|------|------|
| 任务创建 | INFO | simulation_id, target_scope, action_count |
| 输入加载 | DEBUG | state_ref, memory_refs, data_completeness |
| 推演执行 | INFO | simulation_id, duration_ms, result_count |
| 推演完成 | INFO | simulation_id, status, confidence_distribution |
| 推演失败 | ERROR | simulation_id, error_type, error_message |

### 8.2 追踪字段

每次推演必须包含 `trace_id`，用于全链路追踪。

## 9. 并发与幂等性

### 9.1 并发控制

- 同一学生同时只允许一个推演任务执行
- 新请求到达时，若已有进行中任务，返回 409 Conflict

### 9.2 幂等性保证

- 相同输入参数的推演请求，在 24 小时内返回缓存结果
- 缓存 key 由 target_scope + action_candidates + simulation_window 哈希生成

## 10. MVP 范围

### 10.1 必须支持

- 单学生推演
- 7 天/14 天窗口
- 低风险动作模拟（学习内容类、节奏管理类）
- 方向性输出（likely_improve/slight_improve/uncertain）
- 数据不足降级

### 10.2 后续迭代

- 群体推演
- 30 天窗口
- 高风险动作模拟
- 实时推演（WebSocket）
