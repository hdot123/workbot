## 1. 文档目的

本文档用于给出 GRAPH 首轮骨架开发的最小端到端闭环样例，回答以下问题：

1. 从输入到图谱对象到底怎么跑
2. 降级路径是什么
3. 写入错误时如何回滚或拦截

---

## 2. 样例适用范围

本样例集只适用于以下冻结范围：

- 安徽省
- 高中
- 高一
- 物理
- `PHY_PEP_G1_V1`
- 输入源仅限家长文本、老师反馈文本、单页扫描/OCR

---

## 3. 样例一：正常写入

### 3.1 背景

高一学生一次物理作业结果已由上游标准化，知识引用稳定命中。

### 3.2 输入源

老师反馈文本

### 3.3 事件生成结果

- `event_type = homework_result_event`
- `student_id` 合法
- `knowledge_refs` 稳定命中 `受力分析`
- `chapter_refs` 命中第二章
- `confidence_score = 0.91`

### 3.4 TWIN 输出

- 更新当前关注知识点
- 更新作业结果事实
- 更新行为摘要
- 生成 `state_update_event`

### 3.5 GRAPH 消费情况

- 接收标准学习事件和状态更新事件
- 校验通过
- 进入主写入链

### 3.6 图谱/记忆写入结果

- Event：追加 1 条作业事件
- Fact：替换当前关注知识点、最新作业结果
- Relation：写入 `Student -> has_state -> Snapshot`、`Event -> refers_to -> KnowledgePoint`
- Snapshot：固化当前态
- Retrieval Unit：生成单学生最小检索单元

### 3.7 检索结果

单学生当前态解释查询可召回：

- 当前作业结果事实
- `受力分析` 知识点
- 最近关键作业事件

### 3.8 是否触发回滚/降级

否

### 3.9 验收点

1. 事件成功写入
2. Fact supersede 正确
3. 关系建边正确
4. Snapshot 可追溯
5. Retrieval Unit 可召回

### 3.10 结论

正常输入、稳定引用的场景，GRAPH 应完整跑通主链闭环。

## 4. 样例二：字段/引用缺失时降级

### 4.1 背景

家长文本被上游解析为有效学习反馈，但没有稳定 `knowledge_refs`，仅保留合法 `chapter_refs`。

### 4.2 输入源

家长文本

### 4.3 事件生成结果

- `event_type = parent_feedback_event`
- `student_id` 合法
- `chapter_refs` 合法
- `knowledge_refs` 缺失
- `confidence_score = 0.78`
- `event_status = degraded`

### 4.4 TWIN 输出

- 更新行为摘要
- 不直接更新知识掌握状态
- 生成降级状态更新事件

### 4.5 GRAPH 消费情况

- 校验通过
- 识别为降级写入场景

### 4.6 图谱/记忆写入结果

- Event：追加 1 条家长反馈事件
- Fact：更新行为层相关事实
- Relation：不写强 KnowledgePoint 关系
- Snapshot：可固化当前行为变化
- Retrieval Unit：可保留章节级上下文

### 4.7 检索结果

可召回：

- 家长反馈事件
- 当前行为摘要
- 相关章节上下文

不可召回：

- 强知识点掌握结论

### 4.8 是否触发回滚/降级

触发降级，不触发回滚

### 4.9 验收点

1. 降级原因被记录
2. 未越权写入强知识关系
3. Event / Snapshot 仍可追溯
4. 检索结果没有伪造知识掌握结论

### 4.10 结论

引用不足但输入合法时，GRAPH 应降级保留时间线和行为语义，不强行补造知识结构。

## 5. 样例三：错误写入后回滚或拦截

### 5.1 背景

单页扫描/OCR 事件初次解析把题目错误映射到错误知识点，后续人工复核确认原写入有误。

### 5.2 输入源

单页扫描/OCR

### 5.3 事件生成结果

- 初始事件：`scan_ocr_result_event`
- `confidence_score = 0.73`
- 初始 `knowledge_refs` 命中错误候选
- 人工复核后产生 `reviewed_learning_event`

### 5.4 TWIN 输出

- 初始阶段仅允许有限更新
- 人工复核回写后生成修正状态更新事件

### 5.5 GRAPH 消费情况

第一阶段：

- 初始 OCR 事件只写 Event 与有限 Snapshot
- 不直接建强知识关系

若系统误写了错误 Fact / Relation，则第二阶段：

- 根据 `reviewed_learning_event` 触发对象级回滚

### 5.6 图谱/记忆写入结果

初始阶段：

- Event：保留 OCR 事件
- Fact：不写高确定性知识掌握结论
- Relation：不写强 KnowledgePoint 关系

修正阶段：

- 若存在误写对象，旧版本标记 invalid / superseded
- 写入修正后的 Fact / Relation
- Snapshot 重建
- Retrieval Unit 刷新

### 5.7 检索结果

回滚后查询只应看到修正后的知识关联和状态，不应继续暴露错误旧结论为当前有效结果。

### 5.8 是否触发回滚/降级

先降级；若已误写则触发对象级回滚

### 5.9 验收点

1. 初始低置信 OCR 不得直接强写知识关系
2. 误写时必须能回滚
3. 回滚后版本链不断裂
4. 检索结果只暴露当前有效版本

### 5.10 结论

低置信或误映射场景下，GRAPH 必须优先受控、可退，不能“先写进去再慢慢解释”。
