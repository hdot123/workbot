## 1. 文档目的与边界

本文档用于定义 TWIN 向 OBS 提供输出的最小稳定契约，回答以下问题：

1. OBS 到底消费 TWIN 的哪些状态字段
2. 哪些字段缺失时 OBS 可以继续展示
3. 哪些字段缺失时 OBS 必须拦住或降级
4. TWIN 状态更新时如何通知 OBS
5. 低置信度状态 OBS 应如何处理

本文档只定义模块契约级输出边界，不展开为：

- API 字段协议文档
- 数据库表结构文档
- 前端渲染逻辑文档

---

## 2. 契约适用范围

本契约只适用于以下冻结范围：

| 维度 | 冻结值 |
|------|--------|
| 地区 | 安徽省 |
| 学段 | 高中 |
| 年级 | 高一 |
| 学科 | 物理 |
| 教材版本 | `PHY_PEP_G1_V1` |
| 上游基线 | TWIN 已冻结首轮骨架开发基线（NAV-005） |
| 下游消费方 | OBS 首轮骨架开发 |

超出冻结范围的状态，不属于本契约允许消费范围。

## 3. OBS 可消费的 TWIN 状态字段

### 3.1 必需字段（缺失时 OBS 需降级提示）

| 字段 | 用途 | OBS 使用方式 |
|------|------|--------------|
| `student_id` | 学生唯一标识 | OBS 页面路由与数据关联 |
| `twin_version` | TWIN 当前版本 | 版本追溯与缓存失效 |
| `last_updated_at` | 状态最后更新时间 | 数据新鲜度展示 |
| `region_id` | 地区标识 | 校验是否属于安徽范围 |
| `grade_level` | 年级 | 校验是否属于高一范围 |
| `subject` | 学科 | 校验是否属于物理范围 |
| `curriculum_version_id` | 教材版本 | 校验是否属于 `PHY_PEP_G1_V1` |
| `overall_mastery_level` | 整体掌握等级 | 家长端首页摘要展示 |
| `risk_level` | 整体风险等级 | 家长端首页风险展示 |
| `data_quality_score` | 数据质量评分 | 决定是否展示"数据收集中"提示 |

### 3.2 核心展示字段（优先展示）

| 字段 | 用途 | OBS 使用方式 |
|------|------|--------------|
| `subject_mastery_summary` | 学科掌握摘要 | 家长端首页学科状态展示 |
| `recent_change_summary` | 近期变化摘要 | 家长端首页变化展示 |
| `current_focus_points` | 当前关注点 | 家长端首页关注点展示 |
| `homework_accuracy_trend` | 作业正确率趋势 | 家长周报趋势图表 |
| `exam_accuracy_trend` | 考试正确率趋势 | 家长周报趋势图表 |
| `mastery_gap` | 掌握度差距 | 风险提示展示 |
| `mastery_gap_status` | 掌握度差距状态 | 风险解释展示 |

### 3.3 可选字段（缺失时降级但不阻断）

| 字段 | 用途 | OBS 使用方式 |
|------|------|--------------|
| `knowledge_mastery_snapshot` | 知识点掌握快照 | 老师端详情页知识结构展示 |
| `behavior_tags` | 行为标签 | 老师端行为观察展示 |
| `study_habits_summary` | 学习习惯摘要 | 家长端习惯展示 |
| `teacher_context` | 老师视角上下文 | 老师端专属展示 |
| `parent_context` | 家长视角上下文 | 家长端专属展示 |

## 4. 状态置信度处理规则

### 4.1 置信度分级

| 置信度范围 | 状态 | OBS 处理方式 |
|------------|------|--------------|
| `≥ 0.85` | 高置信 | 直接展示，无需额外说明 |
| `0.70 ~ 0.85` | 中置信 | 展示但标注"基于有限数据" |
| `0.50 ~ 0.70` | 低置信 | 降级展示，明确标注置信度 |
| `< 0.50` | 不可信 | 不展示核心结论，仅展示数据收集状态 |

### 4.2 低置信度状态展示规则

当 `data_quality_score < 0.50` 时，OBS 必须：

1. 不展示高风险结论
2. 展示"数据收集中，暂无法给出可靠判断"
3. 展示已收集数据量和预计达标条件
4. 引导用户补充输入

## 5. 状态更新通知机制

### 5.1 事件类型

TWIN 状态更新后，广播以下事件：

| 事件类型 | 触发条件 | OBS 订阅方式 |
|----------|----------|--------------|
| `twin.state.updated` | TWIN 状态发生实质性变化 | 消息队列订阅 |
| `twin.version.changed` | TWIN 版本变更 | 消息队列订阅 |
| `twin.risk.alert` | 风险等级变化超过阈值 | 消息队列订阅 |

### 5.2 事件负载

```json
{
  "event_type": "twin.state.updated",
  "student_id": "stu_001",
  "twin_version": "v1.2.3",
  "updated_at": "2026-03-21T10:00:00Z",
  "change_summary": "物理学科掌握度从 0.72 提升至 0.78",
  "affected_fields": ["overall_mastery_level", "subject_mastery_summary"],
  "trace_id": "trace_abc123"
}
```

## 6. 观察口径与展示边界

### 6.1 家长端展示口径

| 展示内容 | TWIN 输出字段 | 展示规则 |
|----------|---------------|----------|
| 当前状态摘要 | `overall_mastery_level`、`risk_level` | 直接展示 |
| 近期变化 | `recent_change_summary` | 优先展示正向变化 |
| 关注点 | `current_focus_points` | 限制最多 3 个 |
| 风险提示 | `mastery_gap_status` | 仅展示中高风险 |

### 6.2 老师端展示口径

| 展示内容 | TWIN 输出字段 | 展示规则 |
|----------|---------------|----------|
| 学生详情 | `knowledge_mastery_snapshot` | 完整展示知识结构 |
| 行为观察 | `behavior_tags` | 时间序列展示 |
| 风险对比 | `risk_level`、`mastery_gap` | 班级内对比展示 |

### 6.3 禁止展示的内容

- 不展示原始推断过程
- 不展示模型内部参数
- 不展示置信度低于 0.50 的结论
- 不展示跨学生的直接排名

## 7. 缺失字段降级策略

| 缺失场景 | 是否允许继续展示 | 处理方式 |
|----------|------------------|----------|
| `overall_mastery_level` 缺失 | 允许 | 展示"状态计算中" |
| `recent_change_summary` 缺失 | 允许 | 隐藏变化模块 |
| `knowledge_mastery_snapshot` 缺失 | 允许 | 老师端详情页降级展示 |
| `data_quality_score` 缺失 | 不允许 | 必须拦住，提示数据异常 |
| `student_id` 缺失 | 不允许 | 必须拦住，路由错误 |

## 8. 与上游责任边界

| 模块 | 责任 | 不负责 |
|------|------|--------|
| TWIN | 提供稳定的状态输出、版本追溯、更新通知 | 不负责展示逻辑、不负责降级展示文案 |
| OBS | 消费 TWIN 状态、处理降级展示、管理用户体验 | 不负责状态计算、不负责状态一致性校验 |

结论是：

- TWIN 对"状态是否正确计算"负责
- OBS 对"状态如何展示给用户"负责

---

## 9. 验收要点

本契约的最小验收要点如下：

1. OBS 只消费冻结范围内的 TWIN 状态字段
2. 低置信度状态必须降级展示
3. 缺失关键字段时必须拦住或提示
4. 状态更新事件必须能被 OBS 订阅
5. 版本追溯必须完整
