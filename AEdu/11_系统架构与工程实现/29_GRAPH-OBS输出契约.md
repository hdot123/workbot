## 1. 文档目的与边界

本文档用于定义 GRAPH 向 OBS 提供输出的最小稳定契约，回答以下问题：

1. OBS 到底消费 GRAPH 的哪些检索结果
2. 图谱上下文如何装配成解释依据
3. 检索失败时 OBS 应如何处理
4. 图谱版本如何追溯
5. 低相关度检索结果应如何降级

本文档只定义模块契约级输出边界，不展开为：

- GraphRAG 检索算法文档
- 图数据库查询语句文档
- 上下文装配提示词文档

---

## 2. 契约适用范围

本契约只适用于以下冻结范围：

| 维度 | 冻结值 |
|------|--------|
| 地区 | 安徽省 |
| 学段 | 高中 |
| 年级 | 高一 |
| 学科 | 物理 |
| 教材版本 | `PHY_PEP_G1_V1` |
| 上游基线 | GRAPH 已冻结首轮骨架开发基线（NAV-006） |
| 下游消费方 | OBS 首轮骨架开发 |

超出冻结范围的检索，不属于本契约允许消费范围。

## 3. OBS 可消费的 GRAPH 输出类型

### 3.1 解释依据输出

| 输出类型 | 用途 | OBS 使用方式 |
|----------|------|--------------|
| `student_event_trace` | 学生事件追溯链 | 报告中的"发生了什么"依据 |
| `knowledge_change_trace` | 知识掌握变化追溯 | 报告中的"为什么变化"依据 |
| `risk_factor_trace` | 风险因子追溯 | 报告中的"为什么需要关注"依据 |

### 3.2 上下文装配输出

| 输出类型 | 用途 | OBS 使用方式 |
|----------|------|--------------|
| `observation_context` | 观察上下文装配 | 家长端首页解释依据 |
| `report_context` | 报告上下文装配 | 周报/月报生成依据 |
| `comparison_context` | 对比上下文装配 | 时间对比/班级对比依据 |

## 4. 检索状态处理规则

### 4.1 状态分级

| 状态 | 含义 | OBS 处理方式 |
|------|------|--------------|
| `success` | 检索成功，高相关度 | 直接使用上下文块 |
| `partial_success` | 部分成功，相关度一般 | 使用部分上下文块，标注"部分依据" |
| `low_relevance` | 相关度低 | 降级展示，不使用图谱依据 |
| `failure` | 检索失败 | 不使用图谱依据，仅展示 TWIN 状态 |

### 4.2 检索失败降级规则

当检索状态为 `failure` 或 `low_relevance` 时，OBS 必须：

1. 不展示"基于图谱分析"的结论
2. 仅展示 TWIN 状态层的结论
3. 不提示"系统异常"，避免用户焦虑
4. 记录降级日志供后续排查

## 5. 上下文装配规则

### 5.1 家长端首页上下文装配

| 上下文块类型 | 优先级 | 装配来源 |
|--------------|--------|----------|
| 当前状态摘要 | P0 | TWIN 状态 + 近期事件图谱 |
| 近期变化 | P0 | TWIN 变化记录 + 事件追溯 |
| 关注点解释 | P1 | 风险因子图谱 + 知识关联 |
| 建议行动 | P1 | 知识依赖图谱 + 干预建议模板 |

### 5.2 周报上下文装配

| 上下文块类型 | 优先级 | 装配来源 |
|--------------|--------|----------|
| 本周事件汇总 | P0 | 事件图谱时间切片 |
| 变化趋势 | P0 | TWIN 演化层 + 趋势图谱 |
| 知识点进展 | P1 | 知识掌握图谱 |
| 风险预警 | P1 | 风险因子图谱 |
| 下周建议 | P2 | 干预建议模板 |

### 5.3 老师端详情页上下文装配

| 上下文块类型 | 优先级 | 装配来源 |
|--------------|--------|----------|
| 学生完整画像 | P0 | TWIN 完整状态 + 关系图谱 |
| 知识结构 | P0 | 知识掌握图谱 + 依赖关系 |
| 行为时序 | P1 | 事件时间序列图谱 |
| 班级对比 | P2 | 班级聚合图谱 |

## 6. 图谱版本追溯

### 6.1 版本信息字段

| 字段 | 用途 |
|------|------|
| `graph_version` | 图谱数据版本 |
| `schema_version` | 图谱结构版本 |
| `embedding_version` | 向量嵌入版本 |
| `retrieved_at` | 检索时间 |

### 6.2 版本一致性要求

- 同一学生的多次检索应使用相同版本
- 版本变更时需要刷新 OBS 缓存
- 版本追溯必须记录在 `trace_id` 中

## 7. 检索性能要求

| 检索类型 | 预期响应时间 | 超时处理 |
|----------|--------------|----------|
| 观察上下文装配 | < 500ms | 降级展示 TWIN 状态 |
| 周报上下文装配 | < 2s | 异步生成，推送通知 |
| 详情页上下文装配 | < 1s | 降级展示简化版 |

## 8. 与上游责任边界

| 模块 | 责任 | 不负责 |
|------|------|--------|
| GRAPH | 提供稳定的检索结果、版本追溯、上下文装配 | 不负责最终展示文案、不负责用户体验 |
| OBS | 消费检索结果、处理降级展示、管理用户体验 | 不负责检索算法、不负责图谱一致性 |

结论是：

- GRAPH 对"检索结果是否相关正确"负责
- OBS 对"检索结果如何展示给用户"负责

---

## 9. 验收要点

本契约的最小验收要点如下：

1. OBS 只消费冻结范围内的 GRAPH 检索结果
2. 检索失败时必须降级展示
3. 低相关度结果不得作为核心结论依据
4. 版本追溯必须完整
5. 性能超时必须降级处理
