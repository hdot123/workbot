## 3. 最小可验收集合

### 3.1 推演引擎最小可验收集合

必须至少包含以下功能：

1. 单学生干预推演任务创建
2. 推演结果查询
3. 动作比较排序
4. 方向性输出
5. 置信度区间输出
6. 数据不足降级

### 3.2 风险预测服务最小可验收集合

必须至少包含以下功能：

1. 单学生风险预测任务创建
2. 风险等级输出
3. 风险方向判断
4. 风险因子分析
5. 数据不足降级

### 3.3 影子推演机制最小可验收集合

必须至少包含以下功能：

1. 影子推演任务触发
2. 推演结果内部查看
3. 校准记录生成

---

## 4. 验收维度定义

### 4.1 正确性

输出内容是否与上游 TWIN/GRAPH 输出一致，推演/预测逻辑是否正确。

### 4.2 一致性

同类推演、同类风险判断是否遵循统一规则和口径。

### 4.3 完整性

最小必需功能、必需字段、必需交互是否齐全。

### 4.4 可解释性

推演/预测结果是否附带解释依据、边界说明、风险提示。

### 4.5 安全性

高风险结论是否不直接对家长端输出，是否需要人工复核。

## 10. 首轮骨架开发出场门槛

SIM 从"骨架开发中"进入"正式允许继续开发"前，必须满足以下条件：

- [ ] 推演引擎最小可验收集合全部具备
- [ ] 风险预测服务最小可验收集合全部具备
- [ ] 影子推演机制最小可验收集合全部具备
- [ ] 3 组端到端样例全部跑通并形成记录
- [ ] 与 TWIN/GRAPH 联合消费闭环已验证
- [ ] 数据不足降级链路已验证
- [ ] 高风险控制链路已验证
- [ ] 无 P0 阻断项
- [ ] P1 遗留项已被记录并明确不影响首轮闭环

## 11. 放行规则

| 结论 | 判定标准 |
|------|----------|
| 允许 | Entry Gate 全满足，Exit Gate 全满足，且无 P0 阻断项 |
| 条件允许 | Entry Gate 全满足，但 Exit Gate 仍有少量 P1 缺口，且不破坏主链路 |
| 不允许 | Entry Gate 未满足，或 Exit Gate 存在任一 P0 阻断项 |

---

## 12. 首轮不作为阻断项的内容

以下内容可记录为后续迭代项，但不作为本轮骨架开发阻断项：

- 群体推演
- 志愿填报推演
- 成长型/决策型 Agent
- 长周期推演（>30 天）
- 高风险建议直出
- 性能压测和大规模并发

---

## 13. 结论

SIM 的最小验收，不是要求一开始就把全量推演层做完，而是要求先把可推演、可预测、可降级、可控制的最小闭环做实。

首轮骨架开发阶段，验收必须坚持五条底线：

- 正确性不过关，不放行
- 一致性不收口，不放行
- 完整性不足以支撑闭环，不放行
- 可解释性不满足用户理解，不放行
- 安全性不符合风险控制要求，不放行
