---

## 1. 文档目的

本文档用于定义教育孪生项目中学科建设的优先级规划，包括优先顺序、选择依据、建设阶段、投入策略与扩展节奏。

本文档重点回答以下问题：

1. 为什么不能一开始全学科一起做
2. 应该先建设哪些学科，后建设哪些学科
3. 学科优先级的判断依据是什么
4. 不同学科建设的边界和目标有什么差异
5. 如何用最小资源跑通学科建设闭环

---

## 2. 学科建设优先级定义

学科建设优先级，是指在有限资源下，对不同学科的教材结构化、知识底座建设、事件映射、状态建模和产品输出支持进行先后排序的规划机制。

它的目标不是给学科排高低贵贱，而是决定：

- 先在哪些学科上跑通系统闭环
- 哪些学科更适合做 MVP
- 哪些学科更适合后续扩展
- 如何避免资源被全面铺开后全部变薄

---

## 3. 规划目标

### 3.1 先跑通，再扩张
优先用少数学科跑通完整主链路，而不是同时摊大饼。

### 3.2 优先选择结构可建模学科
优先选择知识结构明确、题型稳定、状态可解释的学科。

### 3.3 优先支持试点价值感知强的学科
优先建设更容易让老师和家长感受到价值的学科。

### 3.4 控制建设成本
不同学科结构化成本不同，优先级必须考虑投入产出比。

---

## 4. 设计原则

### 4.1 闭环优先
优先选能快速形成"教材 -> 事件 -> 状态 -> 报告"闭环的学科。

### 4.2 结构优先
优先选章节、知识点、题型、依赖关系较清晰的学科。

### 4.3 试点优先
优先考虑试点学校真实会用到的学科。

### 4.4 难度分层
不要一开始同时上"高价值 + 高复杂 + 高争议"学科组合。

### 4.5 可复用优先
优先建设其方法论可迁移到其他学科的学科。

---

## 5. 学科优先级判断维度

建议至少从以下六个维度判断优先级：

| 维度 | 说明 |
|------|------|
| 结构化清晰度 | 章节树、知识点结构是否清晰 |
| 事件映射可行性 | 作业、试题、错题是否容易映射 |
| 状态建模可行性 | 是否容易形成结构化状态表达 |
| 家长与老师价值感知度 | 是否容易感受到价值 |
| 试点使用频率 | 在试点学校是否高频出现 |
| 建设成本与复杂度 | 完成首轮底座建设所需投入 |

---

## 6. 结构化清晰度

### 6.1 含义
该学科是否有较清晰的章节树、知识点结构、题型映射和依赖关系。

### 6.2 说明
像物理、数学这类学科通常更适合先做。  
某些高度开放式、解释依赖上下文极强的学科，前期成本和争议会更高。

---

## 7. 事件映射可行性

### 7.1 含义
该学科的作业、试题、错题、反馈是否较容易映射到知识点和能力点。

### 7.2 说明
如果一个学科事件很难稳定结构化，前期就不适合做第一批主学科。

---

## 8. 状态建模可行性

### 8.1 含义
该学科是否容易形成"掌握状态、波动状态、风险状态"这类结构化表达。

### 8.2 说明
能否稳定说出"哪块不稳"，比是否热门更关键。

---

## 9. 价值感知度

### 9.1 含义
该学科建设完成后，家长、老师是否容易感到"这东西有帮助"。

### 9.2 说明
价值感知高的学科更适合试点前期，因为它能帮助建立信任。

---

## 10. 试点使用频率

### 10.1 含义
在目标试点学校中，该学科是否高频出现、是否常被关注。

### 10.2 说明
优先做试点学校最关心的学科，而不是项目组最喜欢的学科。

---

## 11. 建设成本与复杂度

### 11.1 含义
该学科完成首轮底座建设、标准表建设、事件映射和状态解释所需投入。

### 11.2 说明
高价值但高复杂的学科可以做，但不一定适合做第一枪。

---

## 12. 学科优先级建议

结合当前项目目标，建议优先级如下：

### P0：优先首建学科

| 优先级 | 学科 | 说明 |
|--------|------|------|
| P0 | 高中物理 | 首选首建学科 |

### P1：第二批扩展学科

| 优先级 | 学科 | 说明 |
|--------|------|------|
| P1-1 | 高中数学 | 第二批扩展 |
| P1-2 | 高中英语 | 第二批扩展 |

### P2：第三批扩展学科

| 优先级 | 学科 | 说明 |
|--------|------|------|
| P2 | 高中语文 | 第三批扩展 |

### P3：后续扩展学段

| 优先级 | 学段 | 说明 |
|--------|------|------|
| P3-1 | 初中教材 | 学段扩展 |
| P3-2 | 小学教材 | 学段扩展（需更谨慎） |

## 13. 为什么优先做高中物理

建议高中物理作为首个学科，原因包括：

| 原因 | 说明 |
|------|------|
| 知识结构清晰 | 章节、知识点、依赖关系明确 |
| 知识点和题型映射较稳定 | 便于事件映射和状态建模 |
| 依赖关系明显 | 适合推演和路径分析 |
| 能力点较易建模 | 受力分析、图像理解等能力可观察 |
| 风险和状态解释价值高 | 前置缺口影响后续的效果明显 |
| 很适合做样板 | 容易展示"知识点不稳 -> 后续影响"的系统价值 |

物理是那种很适合做"教育孪生项目第一张能说服人的样板图"的学科。

---

## 14. 为什么第二批建议高中数学

数学适合第二批扩展，原因：

| 优势 | 说明 |
|------|------|
| 价值感知高 | 家长和老师都关心 |
| 试点覆盖广 | 几乎所有学校都重视 |
| 题型丰富 | 有利于事件映射验证 |
| 能力点体系明显 | 运算、证明、建模等能力清晰 |

但挑战也存在：

| 挑战 | 说明 |
|------|------|
| 粒度控制容易失衡 | 容易过细或过粗 |
| 结构化成本较高 | 题型和知识点组合复杂 |
| 一些题目依赖过程表达较强 | 过程性评价较难结构化 |

所以适合在物理方法跑顺后进入。

---

## 15. 为什么英语优先于语文

英语通常比语文更容易先做，原因包括：

| 优势 | 说明 |
|------|------|
| 单元结构相对清晰 | 章节和知识点边界明确 |
| 词汇、语法、题型映射较稳定 | 便于结构化 |
| 事件映射相对更标准化 | 作业、测试容易映射 |
| 错题与练习更容易结构化 | 客观题比例较高 |

语文并不是不重要，而是它更复杂，尤其在：

| 复杂点 | 说明 |
|--------|------|
| 阅读理解 | 解释依赖上下文较强 |
| 作文 | 开放性强，标准化难度大 |
| 多重解释层面 | 同一内容可能有多种理解 |

所以前期更容易把系统拖进高复杂区。

---

## 16. 语文学科的定位建议

语文适合在以下条件下再加强建设：

| 条件 | 说明 |
|------|------|
| 报告与解释层已经成熟 | 能处理更复杂的解释逻辑 |
| 老师端反馈机制已顺畅 | 老师能有效校正系统判断 |
| 系统已具备处理更复杂开放文本的能力 | NLP 能力成熟 |
| 试点学校对语文支持有明确需求 | 有真实使用场景 |

---

## 17. 初中与小学的扩展建议

### 17.1 初中

| 建议 | 说明 |
|------|------|
| 适合在高中方法成熟后复制下沉 | 方法论可复用 |
| 知识点和能力点结构类似 | 建设成本相对可控 |

### 17.2 小学

| 建议 | 说明 |
|------|------|
| 需更谨慎 | 学段特殊性高 |
| 家校协作比重更高 | 家长参与度影响大 |
| 状态表达更容易误伤 | 低龄学生状态解释需更谨慎 |
| 行为与节奏解释更敏感 | 行为判断容易引发争议 |

所以小学不建议作为系统主链路第一阶段重点。

## 18. 学科建设阶段规划

建议分为四个阶段：

### 第一阶段：单学科跑通

| 内容 | 说明 |
|------|------|
| 学科 | 高中物理 |
| 目标 | 跑通标准表、事件映射、状态模型、报告输出 |
| 验收标准 | 至少一个试点场景真正用起来 |

### 第二阶段：双学科/三学科扩展

| 内容 | 说明 |
|------|------|
| 学科 | 高中数学、高中英语 |
| 目标 | 复制物理方法论，验证可扩展性 |
| 验收标准 | 多学科并行运行稳定 |

### 第三阶段：复杂学科扩展

| 内容 | 说明 |
|------|------|
| 学科 | 高中语文 |
| 目标 | 处理更复杂开放学科的结构化 |
| 验收标准 | 语文报告和解释被接受 |

### 第四阶段：学段扩展

| 内容 | 说明 |
|------|------|
| 学段 | 初中、小学 |
| 目标 | 下沉学段，验证方法论可复制性 |
| 验收标准 | 低学段场景适用 |

---

## 19. 每个学科的最小建设包

建议每个学科至少包含：

| 内容 | 说明 |
|------|------|
| 原始教材 | 教材 PDF 或文本 |
| 章节树 | 章节层级结构 |
| 知识点表 | 知识点列表及定义 |
| 能力点表 | 能力点列表及定义 |
| 教材锚点表 | 知识点与教材位置映射 |
| 依赖关系表 | 知识点/能力点依赖关系 |
| 常见误区表 | 常见错误理解 |
| 题型映射表 | 知识点与题型关联 |

物理等实验学科额外增加：
- 实验活动表

## 20. 学科优先级动态调整原则

优先级不是永远固定的，可根据以下情况调整：

| 情况 | 说明 |
|------|------|
| 试点学校真实需求变化 | 学校需求优先 |
| 某学科建设阻力异常大 | 可暂缓调整顺序 |
| 某学科价值验证明显更强 | 可提升优先级 |
| 某学科外部资源可得性变化 | 资源到位可加速 |
| 项目阶段目标变化 | 随项目目标调整 |

但调整必须记录原因，不能今天拍脑袋换方向，明天又换回来。

### 21.1 P0 学科资源投入

|------|------|

| 建议 | 说明 |
|------|------|
| 以复制物理方法论为主 | 降低建设成本 |
| 适度调整适配 | 考虑学科差异 |

### 21.3 P2 / P3 学科资源投入

| 建议 | 说明 |
|------|------|
| 更强调分阶段和场景适配 | 不宜一股脑铺开 |
| 根据试点反馈调整 | 灵活响应需求 |

## 22. 成功标准

一门学科进入"已建设可用"状态，建议至少满足：

| 标准 | 说明 |
|------|------|
| 章节树完成 | 章节结构完整 |
| 核心知识点完成 | 核心知识点覆盖 |
| 基础能力点完成 | 基础能力点覆盖 |
| 事件可映射 | 学习事件能映射到知识点 |
| StudentTwinAgent 可输出该学科状态 | 状态建模可用 |
| 家长端/老师端能看到有价值解释 | 产品输出有价值 |
| 至少一个试点场景真正用起来 | 真实使用验证 |

## 25. 风险与边界

### 25.1 不能忽视试点真实需求

| 风险 | 说明 |
|------|------|
| 项目组偏好代替学校需求 | 可能导致学科选择脱离实际 |

### 25.2 不能忽视建设成本

| 风险 | 说明 |
|------|------|
| 低估复杂学科建设成本 | 可能导致资源耗尽 |

### 25.3 不能忽视方法论沉淀

| 风险 | 说明 |
|------|------|
| 每科学科都从零开始 | 建设效率低下 |

### 25.4 不能忽视质量把控

| 风险 | 说明 |
|------|------|
| 为追求速度牺牲质量 | 可能导致学科底座不稳固 |

## 26. 结论

学科建设优先级规划的核心，不是证明哪个学科更重要，而是决定项目应该先在哪些学科上用最小成本跑出真正可用的闭环。

它必须坚持：

- **先跑通** - 用单学科验证主链路
- **再复制** - 方法论复用到其他学科
- **再扩复杂** - 逐步处理更复杂学科
- **最后下沉学段** - 高中成熟后再扩展初中小学

否则系统很容易在"全都想做"的热情里，把每个学科都做成半成品。
