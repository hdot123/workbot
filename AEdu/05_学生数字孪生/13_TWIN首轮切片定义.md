# TWIN 首轮切片定义

> 文档编号：TWIN-013  
> 版本：V1.0  
> 创建日期：2026-03-21  
> 最后更新：2026-03-21  
> 维护人：学生数字孪生负责人  
> 状态：已冻结（首轮骨架开发基线）  
> 用途：支撑 TWIN 首轮骨架开发  
> 变更方式：后续修改需通过新一轮准入评审

---

## 1. 文档目的与定位

本文档用于冻结 TWIN 模块首轮骨架开发的最小切片范围，回答以下问题：

1. TWIN 本轮到底做什么
2. TWIN 本轮明确不做什么
3. 哪些能力看起来相关，但本轮不得放行
4. 在什么边界内，TWIN 可以进入首轮骨架开发

本文档是：

- TWIN 首轮骨架开发的范围冻结文档
- TWIN 模块开发准入前置文档
- TWIN 与 KB、INGEST、GRAPH、OBS、SIM 之间的边界收口文档

本文档不是：

- 数据库字段级设计文档
- API 协议文档
- 报告产品文案文档
- 推演与建议策略文档

---

## 2. 首轮范围冻结值

| 维度 | 冻结值 |
|------|--------|
| 地区 | 广东省深圳市 |
| 学段 | 高中 |
| 年级 | 高一 |
| 学科 | 物理 |
| 教材版本 | `PHY_PEP_G1_V1` |
| 输入源 | 家长文本、老师反馈文本、单页扫描/OCR |
| 模块定位 | StudentTwinAgent 当前态骨架 |
| 放行范围 | 首轮骨架开发 |
| 明确不放行 | 全量实现、全量联调、多地区、多学科、多教材版本、扩展输入源 |

上述冻结值是 TWIN 本轮唯一有效范围。任何超出冻结值的设计、实现、联调或验收需求，均不在本轮准入范围内。

---

## 3. 首轮目标

TWIN 首轮骨架开发只达成以下目标：

1. 在冻结范围内，为单个学生建立可持续更新的 StudentTwinAgent 当前态骨架。
2. 仅消费来自 KB + INGEST 冻结基线的标准学习事件，不直接消费原始输入。
3. 在受控范围内完成状态更新、审计留痕、异常分流和下游状态广播。
4. 为后续 GRAPH、OBS、SIM 提供结构化状态基座，但不提前承担这些模块的职责。

换句话说，本轮目标是：

`标准事件 → TWIN 当前态更新 → 状态变化记录 → review_needed / twin.state.updated`

而不是：

`原始输入 → 全量图谱 → 前台报告 → 后台推演`

---

## 4. 本轮纳入能力

### 4.1 学生对象与基础绑定

本轮纳入：

- 学生对象创建后的基础绑定承接
- `student_id / twin_id / region_id / grade_level / curriculum_version_id` 的稳定承接
- 首轮冻结范围内的单学科绑定

本轮不纳入：

- 多学科并行绑定
- 多教材版本并存切换
- 多地区动态迁移
- 批量导入式学生对象回填

### 4.2 标准事件消费与范围校验

本轮纳入：

- 只消费由 INGEST 输出的标准学习事件
- 对事件范围做深圳 / 高中 / 高一 / 物理 / `PHY_PEP_G1_V1` 校验
- 对事件来源做三类输入源白名单校验

本轮不纳入：

- 原始 OCR 文本直入 TWIN
- 原始家长聊天全文直入 TWIN
- 原始老师聊天全文直入 TWIN
- 范围外事件的“先接进来再说”

### 4.3 首轮五层状态更新

本轮纳入：

- 事实层更新
- 行为层更新
- 演化层基础趋势更新
- 风险层基础风险更新
- 数据质量层更新

本轮不纳入：

- 完整图谱关系建模
- 群体对比状态
- 复杂推演准备包
- 自由生成式解释结果

### 4.4 生命周期最小状态集

本轮只要求跑通以下生命周期状态：

- `pending_binding`
- `cold_start`
- `active`
- `stable`
- `review_needed`

本轮不要求以开发准入为目标强制进入：

- `simulation_ready`
- `archived`

### 4.5 审计与下游广播

本轮纳入：

- `last_event_ref`
- `audit_trace_ref`
- 状态变化记录
- `twin.state.updated` 广播

本轮不纳入：

- 图谱长期记忆主存储
- 前端直接渲染对象
- 高风险建议输出

---

## 5. 本轮不纳入能力

以下能力虽然与 TWIN 有关，但本轮明确不放行：

| 能力 | 不放行原因 |
|------|------------|
| 多地区、多学段、多学科统一 TWIN | 超出冻结范围，会打破 KB + INGEST 已冻结基线 |
| 直接消费原始输入 | 原始输入解析属于 INGEST，不属于 TWIN |
| 完整关系上下文建模 | 班级、学校、家庭复杂关系结构未进入本轮准入范围 |
| GraphRAG 回读依赖 | GRAPH 还不是本轮 TWIN 准入前置 |
| 周报、月报、解释块生成 | 这些属于 OBS，不属于 TWIN |
| 干预动作生成、推演建议 | 这些属于 SIM，不属于 TWIN |
| 群体对比、学校聚合看板 | 这些属于 OBS/学校端，不属于 TWIN |
| 高风险前台直接结论 | 本轮只允许结构化状态，不允许高风险直接对外输出 |
| 批量回放、历史全量补算 | 不属于首轮骨架开发的最小闭环 |
| 学校组织结构大规模变更同步 | 不属于当前冻结范围 |

## 6. 首轮核心对象范围

首轮只冻结以下核心对象：

| 对象 | 定位 | 是否纳入首轮 |
|------|------|---------------|
| StudentTwinAgent 主对象 | 学生个体当前态容器 | 是 |
| 基础绑定对象 | 学生与地区、年级、教材版本的稳定绑定 | 是 |
| 当前事实快照 | 最近作业、最近考试、当前关注点等事实承接 | 是 |
| 行为摘要对象 | 作业节奏、订正跟进、反馈响应等摘要 | 是 |
| 演化摘要对象 | 趋势、波动、变化方向摘要 | 是 |
| 风险摘要对象 | 学习稳定性、知识掌握、False Mastery 等风险 | 是 |
| 数据质量摘要对象 | 完整性、置信度、时效性等质量状态 | 是 |
| 状态变化记录 | 记录每次状态变化的来源和结果 | 是 |
| 审计引用对象 | `last_event_ref / audit_trace_ref` 等追踪引用 | 是 |
| 图谱关系主对象 | 长期记忆与关系主存储 | 否 |
| 报告对象 | 周报、月报、解释结果 | 否 |
| 推演对象 | 干预动作、模拟结果、建议对象 | 否 |

## 7. 首轮状态能力范围

### 7.1 基础绑定状态

必须支持：

- 学生唯一标识与 TWIN 对象绑定
- 地区、年级、学科、教材版本校验
- 范围外事件拦截

### 7.2 事实层状态

必须支持：

- 最近作业结果承接
- 最近考试结果承接
- 当前知识关注点承接
- 当前章节/知识点引用快照承接

### 7.3 行为层状态

必须支持：

- 作业完成节奏摘要
- 订正跟进摘要
- 反馈响应摘要
- 行为异常的结构化留痕

### 7.4 演化层状态

必须支持：

- 学科趋势基础摘要
- 知识点变化基础摘要
- 节奏变化基础摘要
- 风险变化基础摘要

### 7.5 风险层状态

必须支持：

- 学习稳定性风险
- 知识掌握风险
- 进度偏离风险
- 节奏中断风险
- False Mastery 风险

其中 `mastery_gap` 必须能被显式更新，但只允许在有足够事实支撑时更新。

### 7.6 数据质量层状态

必须支持：

- `data_completeness_score`
- `recency_score`
- `event_density_score`
- `state_confidence_score`

### 7.7 本轮限制性说明

本轮允许保留最小关系上下文摘要，例如：

- `parent_context_summary`
- `teacher_context_summary`

但本轮不把“关系上下文层”当成独立大型能力建设，也不把“推演准备层”当成首轮准入对象。

## 8. 首轮事件消费范围

本轮只允许消费以下逻辑事件类型：

| 逻辑事件类型 | 典型来源 | 是否纳入首轮 | 说明 |
|--------------|----------|--------------|------|
| 作业结果事件 | 老师反馈文本、扫描/OCR | 是 | 承接题目完成情况、结果与知识引用 |
| 订正跟进行为事件 | 老师反馈文本、家长文本 | 是 | 承接订正、复做、跟进动作 |
| 老师观察反馈事件 | 老师反馈文本 | 是 | 承接老师对物理学习行为与状态的观察 |
| 家长观察反馈事件 | 家长文本 | 是 | 承接家庭场景下的学习节奏与反馈 |
| 扫描件结果事件 | 单页扫描/OCR | 是 | 承接纸面结果与引用候选 |
| 人工复核回写事件 | 人工复核结果 | 是 | 用于修正 review_needed 事件后的状态更新 |

本轮不允许消费：

- 钉钉组织接入事件
- 授权硬件连续上传事件
- 批量导入事件
- 范围外学科事件
- 不带教材版本约束的泛化事件

---

## 9. 首轮输出范围

TWIN 首轮只允许输出以下结果：

- StudentTwinAgent 当前态快照
- 状态变化记录
- 结构化风险摘要
- 数据质量摘要
- `review_needed` 标记与原因
- `twin.state.updated` 广播

TWIN 首轮明确不输出：

- 家长端完整报告正文
- 老师端完整解释文案
- 学校端聚合看板
- 干预建议
- 决策结论
- 反事实推演结果

---

## 10. 与 KB + INGEST 的依赖关系

### 10.1 TWIN 对 KB 的依赖

TWIN 不直接维护知识底座主数据，必须依赖 KB 提供：

- `curriculum_version_id`
- `chapter_refs`
- `knowledge_refs`
- `anchor_refs`
- 引用有效性边界

### 10.2 TWIN 对 INGEST 的依赖

TWIN 不直接解析原始输入，必须依赖 INGEST 提供：

- 标准学习事件
- 事件类型判定结果
- 事件质量与置信度结果
- 成功 / 降级 / review_needed / reject 的事件状态

### 10.3 TWIN 的消费边界

TWIN 只消费“已被上游处理过的标准事件”，并做以下动作：

- 校验是否在冻结范围内
- 校验是否满足最小消费条件
- 决定进入直接更新、降级更新、review_needed、hold 或 reject
- 形成当前态与状态变化记录

TWIN 不负责修补上游主数据，也不负责替上游重做解析。

---

## 11. 进入开发的最小边界

TWIN 进入首轮骨架开发，至少应满足以下边界：

1. 首轮冻结范围已明确为深圳 / 高中 / 高一 / 物理 / `PHY_PEP_G1_V1`。
2. TWIN 只消费 KB + INGEST 冻结基线中的标准事件。
3. 五层状态更新边界已明确。
4. review_needed、hold、reject 已明确区分。
5. 至少存在 3 组端到端样例可支撑开发与验收。
6. 最小验收矩阵已冻结。
7. 试点范围与责任边界已冻结。

满足以上边界后，允许 TWIN 进入首轮骨架开发。

但该“允许”仅指：

- TWIN 当前态骨架开发
- TWIN 事件消费与状态更新开发
- TWIN 审计留痕与状态广播开发

不指：

- GRAPH、OBS、SIM 的完整联调开发
- 全量试点实现
- 多范围扩展实现
