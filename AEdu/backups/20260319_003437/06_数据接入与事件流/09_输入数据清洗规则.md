# 输入数据清洗规则

> 文档编号：INGEST-009  
> 版本：V1.0  
> 创建日期：2024  
> 最后更新：待定  
> 维护人：数据接入与事件流负责人

---

## 1. 文档目的

本文档用于定义教育孪生项目中各类输入数据的清洗规则，包括文本输入、图片/扫描件输入、批量表格输入、老师反馈输入等在进入学习事件生成层之前的标准化处理方式。

本文档重点回答以下问题：

1. 输入数据为什么必须先清洗
2. 哪些输入问题需要自动处理，哪些需要人工介入
3. 清洗的目标到底是什么
4. 清洗后数据应具备什么最低质量
5. 如何避免"原始输入一进系统就污染后续状态和图谱"

---

## 2. 输入数据清洗定义

输入数据清洗，是指对来自微信、钉钉、扫描件、OCR、学校批量导入、老师手工录入、家长反馈等原始输入进行统一规范化、纠错、补全、去重、格式转换和风险筛查的过程。

清洗层不负责做最终教学判断，它的核心任务是：

- 把脏、乱、散的原始输入
- 转成可以被事件生成层稳定处理的中间结构

也就是说，清洗层不是聪明层，而是防脏层。

---

## 3. 清洗目标

### 3.1 提升输入可用性
让原始输入尽量变成系统能读、能解析、能映射的结构。

### 3.2 控制脏数据进入主链路
避免错误时间、错误对象、错误内容直接影响 StudentTwinAgent。

### 3.3 降低输入差异
不同来源的表达风格、格式和字段结构应尽可能统一。

### 3.4 保留原始痕迹
清洗不是覆盖原始输入，必须保留原始记录用于追溯。

### 3.5 为人工复核提供条件
自动清洗无法确认时，应输出可复核的中间态，而不是瞎猜。

---

## 4. 适用范围

本文规则适用于以下输入类型：

| 输入类型 | 说明 |
|----------|------|
| 家长聊天文本输入 | 微信/钉钉聊天窗口输入 |
| 老师反馈文本输入 | 老师端提交的反馈文本 |
| 图片/扫描件/OCR 输入 | 作业、试卷、练习册照片 |
| 批量表格导入 | 学校导出的成绩/作业表格 |
| 学校系统导出数据 | 教务系统导出的结构化数据 |
| 家长或老师上传的附件 | PDF、Word、图片等附件资料 |

## 5. 设计原则

### 5.1 原始输入保留
任何清洗动作都不应覆盖原始记录。

### 5.2 清洗与判断分离
清洗层不做高阶教学结论，不把"猜测"包装成"清洗结果"。

### 5.3 优先做低风险规范化
先处理格式、字段、日期、名称、去重等低风险问题。

### 5.4 高风险场景谨慎处理
遇到对象识别不准、时间不明、内容歧义大时，优先进入待复核，而不是强行入库。

### 5.5 全链路可追溯
清洗前、清洗后、清洗规则、异常原因都应留痕。

## 6. 清洗对象分类

建议将输入清洗分为五类：

| 清洗类型 | 说明 |
|----------|------|
| 格式清洗 | 统一基本格式，使其可被后续模块稳定消费 |
| 对象清洗 | 确认输入中的主体对象是谁 |
| 内容清洗 | 提取输入中真正与学习事件相关的核心内容 |
| 时间清洗 | 把原始输入中的时间表达标准化 |
| 质量与风险清洗 | 对输入质量进行分级，识别是否适合进入主链路 |

---

## 7. 格式清洗

### 7.1 目标
统一输入的基本格式，使其可被后续模块稳定消费。

### 7.2 典型处理

| 处理项 | 说明 |
|--------|------|
| 去除非法字符 | 过滤不可打印字符、控制字符 |
| 统一空格与换行 | 统一空格、制表符、换行符 |
| 统一日期格式 | 统一为 ISO 8601 格式 (YYYY-MM-DD) |
| 统一数字格式 | 统一小数点、千分位等 |
| 统一布尔值表达 | 是/否、true/false、1/0统一 |
| 统一文件扩展名和 MIME 标记 | 文件类型标准化 |

### 7.3 说明
格式问题最不起眼，但最容易让后续流程悄悄出错。

---

## 8. 对象清洗

### 8.1 目标
确认输入中的主体对象是谁。

### 8.2 核心对象

| 对象 | 说明 |
|------|------|
| 学生 | 输入归属的学生主体 |
| 家长 | 输入提交的家长身份 |
| 老师 | 输入提交的老师身份 |
| 班级 | 学生所属班级 |
| 学校 | 学生所属学校 |
| 学科 | 输入涉及的学科 |
| 教材版本 | 如可识别的教材版本信息 |

### 8.3 典型问题

| 问题 | 说明 |
|------|------|
| 同名学生混淆 | 同一班级有多个同名学生 |
| 家长昵称替代真实身份 | 微信昵称无法对应真实家长 |
| 班级写法不统一 | "高一 (1) 班"vs"高一一班" |
| 学科缩写混乱 | "物"vs"物理"vs"PHY" |

### 8.4 处理原则

| 原则 | 说明 |
|------|------|
| 能通过稳定映射表识别的，自动标准化 | 使用标准映射表 |
| 不能稳定识别的，标记待确认 | 进入待复核状态 |
| 不允许对象不明直接写入学生主状态链路 | 阻断高风险输入 |

---

## 9. 内容清洗

### 9.1 目标
提取输入中真正与学习事件相关的核心内容。

### 9.2 典型处理

| 处理项 | 说明 |
|--------|------|
| 去除无关寒暄、表情、重复文本 | 保留核心内容 |
| 分离学习内容和情绪描述 | 学习内容进入事件，情绪作为附加上下文 |
| 识别作业/考试/错题/反馈等事件类型候选 | 初步事件分类 |
| 去除 OCR 明显乱码 | 过滤识别错误 |
| 修正常见错别字（低风险场景） | 如"物理"写成"物力" |

### 9.3 原则
清洗层做"去噪"和"抽取准备"，不做深结论。

---

## 10. 时间清洗

### 10.1 目标
把原始输入中的时间表达标准化。

### 10.2 典型处理

| 处理项 | 说明 |
|--------|------|
| "昨天""上周三"转标准日期 | 相对时间转绝对时间 |
| OCR 中模糊日期格式统一 | 识别并标准化 |
| 批量导入中的 Excel 日期序列转换 | Excel 序列号转日期 |
| 缺少年份时按上下文补全候选 | 根据上下文推断 |

### 10.3 风险规则

| 规则 | 说明 |
|------|------|
| 时间可稳定解析则自动标准化 | 高置信度自动处理 |
| 时间存在多个候选且影响事件判断时，标记待复核 | 多候选时人工确认 |
| 时间缺失但可接受用"录入时间"作为弱代理时，必须标注来源类型 | 明确标注时间来源 |

---

## 11. 质量与风险清洗

### 11.1 目标
对输入质量进行分级，识别是否适合进入主链路。

### 11.2 核心维度

| 维度 | 说明 |
|------|------|
| 完整性 | 必填字段是否齐全 |
| 清晰度 | 内容是否清晰可读 |
| 可解析性 | 能否被后续模块解析 |
| 对象明确度 | 主体对象是否明确 |
| 时间明确度 | 时间是否明确 |
| 内容可信度 | 内容是否可信 |

### 11.3 输出建议

| 标签 | 说明 |
|------|------|
| clean_pass | 清洗通过，可进入主链路 |
| clean_pass_with_notes | 清洗通过但带备注 |
| pending_review | 待人工复核 |
| rejected | 拒绝进入主链路 |

---

## 12. 文本输入清洗规则

### 12.1 家长/老师文本

重点处理：

| 处理项 | 说明 |
|--------|------|
| 去表情/口水词干扰 | 过滤表情符号、口水词 |
| 切分多事件长文本 | 一条文本含多个事件时切分 |
| 保留关键原句 | 保留关键表述原文 |
| 识别事件候选和情绪补充信息 | 事件与情绪分离 |

### 12.2 示例

**原始输入**：
> 今天物理作业写到很晚，受力那块还是不太会，错了好几题 😓

**清洗后中间态**：

| 字段 | 值 |
|------|------|
| 事件候选 | 作业相关 |
| 学科 | 物理 |
| 主题候选 | 受力分析 |
| 现象 | 错题较多 |
| 附加情绪 | 家长焦虑/担忧（仅附加，不直接进入事实层） |

---

## 13. 图片与扫描件清洗规则

### 13.1 目标
提升 OCR 和后续文档理解的可用性。

### 13.2 典型处理

| 处理项 | 说明 |
|--------|------|
| 图片方向纠正 | 自动旋转至正确方向 |
| 页面裁边 | 去除无关边框 |
| 去阴影/去倾斜 | 图像预处理 |
| 多页拆分 | 多页文档拆分处理 |
| 模糊页标记 | 质量差的页面单独标记 |
| OCR 失败页单独标识 | OCR 失败时明确标识 |

### 13.3 原则
图片质量不够时，应优先输出质量标签，而不是假装 OCR 识别得很好。

### 13.4 质量标签建议

| 标签 | 说明 |
|------|------|
| ocr_good | OCR 质量良好，可信任 |
| ocr_medium | OCR 质量一般，需部分人工核对 |
| ocr_bad | OCR 质量差，需人工处理 |

---

## 14. 批量表格导入清洗规则

### 14.1 目标
统一学校或机构导出的表格结构。

### 14.2 典型处理

| 处理项 | 说明 |
|--------|------|
| 列名映射标准化 | 不同列名映射到标准字段 |
| 空行和重复行剔除 | 清理无效数据行 |
| 班级/学生字段格式统一 | 统一命名格式 |
| 日期、分数、排名字段格式化 | 数值字段标准化 |
| 必填字段缺失检查 | 必填字段校验 |

### 14.3 原则
批量导入最容易表面整齐、内里混乱，必须优先做字段级校验。

---

## 15. 去重规则

### 15.1 目标
识别重复输入、重复附件、重复导入记录。

### 15.2 去重维度建议

| 维度 | 说明 |
|------|------|
| 同一学生 | 学生 ID 相同 |
| 同一时间窗口 | 时间相近 |
| 同一来源 | 输入来源相同 |
| 同一事件类型 | 事件类型相同 |
| 同一附件 checksum / 文本近似 | 内容相似度 |

### 15.3 原则

| 原则 | 说明 |
|------|------|
| 完全重复可合并 | 相同内容只保留一份 |
| 高相似但不完全相同的记录应谨慎标记，不自动吞并 | 保留差异记录 |
| 去重结果必须可追溯 | 记录去重依据 |

---

## 16. 标准化规则建议

建议至少标准化以下字段：

| 字段 | 说明 |
|------|------|
| student_id | 学生唯一标识 |
| school_id | 学校唯一标识 |
| class_id | 班级唯一标识 |
| subject | 学科（统一编码） |
| event_time | 事件时间（ISO 8601） |
| source_type | 来源类型 |
| input_channel | 输入渠道 |
| attachment_ref | 附件引用 |
| raw_text_ref | 原始文本引用 |
| cleaned_text | 清洗后文本 |
| quality_label | 质量标签 |

## 17. 质量标签建议

建议给清洗后的输入打以下标签：

| 标签类别 | 标签值 | 说明 |
|----------|--------|------|
| 对象置信度 | object_confident | 对象识别置信度高 |
| 对象置信度 | object_uncertain | 对象识别不确定 |
| 时间置信度 | time_confident | 时间识别置信度高 |
| 时间置信度 | time_uncertain | 时间识别不确定 |
| 内容清晰度 | content_clear | 内容清晰 |
| 内容清晰度 | content_noisy | 内容噪声较多 |
| OCR 质量 | ocr_good | OCR 质量好 |
| OCR 质量 | ocr_medium | OCR 质量一般 |
| OCR 质量 | ocr_bad | OCR 质量差 |
| 来源验证 | source_verified | 来源已验证 |
| 来源验证 | source_unverified | 来源未验证 |

这些标签后续可直接用于事件生成和人工复核优先级排序。

## 18. 清洗流程建议

```text
接收原始输入
    ↓
保存原始记录
    ↓
做格式清洗
    ↓
做对象识别与标准化
    ↓
做内容提取与去噪
    ↓
做时间标准化
    ↓
做去重与质量评估
    ↓
输出清洗结果与质量标签
    ↓
进入事件生成或人工复核
```

### 18.1 各阶段说明

| 阶段 | 工作内容 | 输出 |
|------|----------|------|
| 接收原始输入 | 接收各渠道原始输入 | 原始输入记录 |
| 保存原始记录 | 保存原始输入不变 | raw_input_id |
| 格式清洗 | 格式标准化 | 格式化数据 |
| 对象识别与标准化 | 识别学生、学科等对象 | 对象标识 |
| 内容提取与去噪 | 提取核心内容 | cleaned_text |
| 时间标准化 | 时间标准化 | event_time |
| 去重与质量评估 | 去重和质量打分 | quality_labels |
| 输出清洗结果 | 输出清洗结果 | 清洗后中间态 |
| 进入事件生成或人工复核 | 根据质量标签分流 | 事件或复核队列 |

## 19. 与学习事件生成标准的关系

输入清洗规则服务于 `INGEST-007 学习事件生成标准`。

关系如下：

```text
原始输入
    ↓
清洗与标准化
    ↓
事件候选
    ↓
结构化学习事件
```

如果清洗层不稳，事件层一定会漂。

---

## 20. 与异常兜底流程的关系

清洗层无法稳定处理的输入，应进入 `INGEST-010 异常输入与人工兜底流程`。

建议触发条件包括：

| 触发条件 | 说明 |
|----------|------|
| 对象不明 | 无法确认学生/老师身份 |
| 时间严重歧义 | 时间无法稳定解析 |
| OCR 质量差 | ocr_bad 标签 |
| 内容无法切分 | 多事件混杂无法分离 |
| 与既有记录强冲突 | 与已有记录矛盾 |

## 21. 质量要求

### 21.1 可追溯性

| 要求 | 说明 |
|------|------|
| 原始输入必须保留引用 | raw_input_id 必须保存 |
| 清洗规则必须记录 | 使用的清洗规则可追溯 |
| 清洗人/清洗时间必须记录 | 人工清洗时记录处理信息 |

### 21.2 可解释性

| 要求 | 说明 |
|------|------|
| 清洗规则和清洗结果应能解释 | 可说明清洗依据 |
| 质量标签应有明确判断依据 | 标签可解释 |

### 21.3 稳定性

| 要求 | 说明 |
|------|------|
| 同类输入经过相同规则后结果应稳定 | 相同输入相同输出 |
| 规则变更应有版本记录 | 规则可追溯 |

### 21.4 保守性

| 要求 | 说明 |
|------|------|
| 不确定时应降级 | 不强行确定 |
| 高风险问题应进入待复核 | 不直接进入主链路 |

### 21.5 可复核性

| 要求 | 说明 |
|------|------|
| 人工能够看懂清洗前后发生了什么 | 清洗对比可查看 |
| 人工可修正清洗结果 | 支持人工修正 |

## 26. 风险与边界

### 26.1 不能忽视原始记录保留

| 风险 | 说明 |
|------|------|
| 原始输入被覆盖 | 无法追溯和复核 |

### 26.2 不能忽视清洗与判断分离

| 风险 | 说明 |
|------|------|
| 清洗层做教学判断 | 清洗结果带有主观倾向，影响后续处理 |

### 26.3 不能忽视高风险输入拦截

| 风险 | 说明 |
|------|------|
| 对象不明输入进入主链路 | 污染学生状态数据 |

### 26.4 不能忽视质量标签

| 风险 | 说明 |
|------|------|
| 不标注质量标签 | 后续无法判断输入可信度 |

### 26.5 不能忽视规则版本管理

| 风险 | 说明 |
|------|------|
| 清洗规则无版本记录 | 无法追溯历史清洗依据 |

## 27. 结论

输入数据清洗规则的核心价值，不是把原始输入"修饰得更好看"，而是把它变成足够可靠、足够稳定、足够可追溯的中间结构，避免脏数据直接冲进系统主链路。

它必须坚持：

| 原则 | 说明 |
|------|------|
| **原始保留** | 原始输入必须保留引用 |
| **低风险清洗优先** | 先处理格式、去重等低风险问题 |
| **高风险问题降级** | 不确定时进入待复核 |
| **清洗与判断分离** | 清洗层不做教学判断 |
| **结果可追溯** | 清洗过程可追溯 |

否则系统后面所有看起来很智能的东西，都会建立在一层不太干净的地板上。
