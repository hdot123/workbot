## 1. 文档目的

本文档用于定义教育孪生项目中面向家长的使用引导，包括家长端目标说明、使用路径、阅读方式、反馈方式、常见误区与沟通建议。

本文档重点回答以下问题：

1. 家长第一次接触系统时应该知道什么
2. 家长端应该怎么用最有效
3. 家长应该怎么看周报、月报和观察页
4. 家长应该如何反馈
5. 如何减少误解、焦虑和错误期待

---

## 2. 家长使用引导定义

家长使用引导，是指在试点和正式使用过程中，为家长提供的系统说明、查看路径、阅读建议和反馈指引。

它的目标不是把系统功能讲全，而是帮助家长尽快形成正确预期：

- 这是什么
- 这不是什么
- 我能看什么
- 我可以做什么
- 我不需要因为什么紧张

家长一旦预期错了，后面很容易把观察工具用成情绪放大器。

---

## 3. 引导目标

### 3.1 帮助家长快速上手
让家长知道从哪里看、先看什么。

### 3.2 帮助家长正确理解系统定位
系统是辅助观察工具，不是命运预言器。

### 3.3 帮助家长形成合理使用习惯
看变化、看重点、看建议，不是每次都盯着找"孩子是不是出问题了"。

### 3.4 帮助家长参与反馈
让家长知道补充信息和纠正误差同样重要。

### 3.5 降低误解和焦虑
避免家长把低强度提示理解成严重结论。

## 4. 家长端定位说明

建议在家长首次进入时明确说明：

### 4.1 这套系统能做什么

| 功能 | 说明 |
|------|------|
| 帮助了解孩子最近学习状态 | 持续追踪学习情况 |
| 帮助识别近期变化和重点关注项 | 及时发现问题苗头 |
| 帮助看到家庭可以配合的方向 | 提供可执行建议 |
| 帮助形成持续观察 | 不是只看一次考试 |

### 4.2 这套系统不做什么

| 不做 | 说明 |
|------|------|
| 不替代老师判断 | 老师是专业教育者 |
| 不保证未来结果 | 不是预测未来 |
| 不直接给高风险决定性结论 | 避免误导 |
| 不用来给孩子贴人格标签 | 不是评价孩子 |

## 5. 家长使用最小路径

建议家长端最小使用路径为：

```
进入首页
    ↓
查看当前状态摘要
    ↓
查看本周/本月变化
    ↓
查看当前关注点
    ↓
查看家庭可配合事项
    ↓
如有需要提交反馈
```

**说明**：家长最重要的不是"把所有页面都点一遍"，而是形成稳定、低负担的阅读习惯。

## 6. 家长端核心页面引导

### 6.1 首页

建议告诉家长：首页重点看三件事：

| 看什么 | 说明 |
|--------|------|
| 当前整体状态 | 孩子最近整体学习情况 |
| 最近变化 | 和之前比有什么变化 |
| 当前关注点 | 当前最值得注意的 1-3 个点 |

### 6.2 周报/月报

建议告诉家长：

| 报告类型 | 说明 |
|----------|------|
| 周报看最近一周 | 看短期变化 |
| 月报看阶段趋势 | 看长期走向 |
| 重点看变化、解释和建议 | 不是只看结论 |

### 6.3 学科页

建议告诉家长：

| 建议 | 说明 |
|------|------|
| 不必每次都看所有学科 | 避免信息过载 |
| 优先看当前提示需要关注的学科 | 聚焦重点 |

### 6.4 反馈页

建议告诉家长：

| 说明 |
|------|
| 系统看不见家庭里的全部真实情况 |
| 家长补充信息会帮助系统更准确 |

## 7. 家长阅读建议

建议引导家长按以下顺序阅读：

```
1. 先看摘要 → 2. 再看变化 → 3. 再看关注点 → 4. 最后看建议
```

### 7.1 不建议的阅读方式

| 方式 | 问题 |
|------|------|
| 一上来只盯着风险提示看 | 容易放大焦虑 |
| 看到"波动"就自动脑补为"严重下滑" | 过度解读 |
| 把每一句系统解释都当成最终定论 | 忽视边界 |

---

## 8. 如何理解"当前状态"

可向家长说明：

| 要点 | 说明 |
|------|------|
| 当前状态是基于最近一段时间的数据和事件得出的观察结果 | 有数据支撑 |
| 它反映的是"当前阶段的情况" | 阶段性判断 |
| 不代表长期固定标签 | 不是永久评价 |
| 后续有新数据时，状态会更新 | 动态更新 |

**这点很关键**。状态是阶段性的，不是给孩子额头上贴的永久条码。

---

## 9. 如何理解"变化趋势"

可向家长说明：

| 要点 | 说明 |
|------|------|
| 趋势比单次结果更重要 | 看方向不看单点 |
| "有波动"不等于"变差" | 波动是正常的 |
| "有改善"也不等于"已经完全稳定" | 避免过度乐观 |
| 趋势帮助看到最近发生了什么 | 不是预测未来 |

---

## 10. 如何理解"关注点"

可向家长说明：

| 要点 | 说明 |
|------|------|
| 关注点是当前最值得注意的几个地方 | 优先级提示 |
| 不代表孩子整体都不好 | 避免以偏概全 |
| 不需要一次解决所有问题 | 聚焦重点 |
| 优先处理最重要的 1 到 3 个点即可 |  manageable |

---

## 11. 如何理解"建议"

可向家长说明：

| 要点 | 说明 |
|------|------|
| 建议主要是家庭可配合的低风险动作 | 可执行 |
| 建议不是必须全部照做 | 结合实际情况 |
| 建议应结合老师意见和家庭实际情况理解 | 综合判断 |
| 有不适合的建议，可以反馈 | 反馈有价值 |

---

## 12. 家长反馈引导

建议明确告诉家长可反馈以下内容：

### 12.1 可反馈内容

| 反馈类型 | 说明 |
|----------|------|
| 家庭实际情况补充 | 系统看不到的家庭情况 |
| 某条观察是否准确 | 判断准确性反馈 |
| 某条建议是否可执行 | 建议可行性反馈 |
| 近期发生但系统尚未反映的重要情况 | 补充重要信息 |
| 对报告内容的疑问 | 理解问题反馈 |

### 12.2 反馈方式

```
1. 进入反馈页面
2. 选择反馈类型
3. 填写简要说明（不必太长）
4. 选择关联内容（学生/报告/建议）
5. 提交
```

### 12.3 反馈原则

| 原则 | 说明 |
|------|------|
| 短而真实 | 不必写很长 |
| 不求写很长 | 关键是补充信息 |
| 关键是补充系统没看到的信息 | 反馈的价值 |

## 13. 家长常见误区说明

### 13.1 误区一：把系统提示当最终结论

| 误区 | 纠正 |
|------|------|
| 系统输出是最终裁定 | 系统输出是观察和辅助 |

### 13.2 误区二：只盯着风险项看

| 误区 | 纠正 |
|------|------|
| 只看风险提示 | 应同时看摘要、变化和建议 |

### 13.3 误区三：把短期波动理解为长期趋势

| 误区 | 纠正 |
|------|------|
| 短期波动=长期变差 | 短期波动很常见，需结合时间窗口理解 |

### 13.4 误区四：觉得不反馈也没关系

| 误区 | 纠正 |
|------|------|
| 反馈是可选的 | 家长反馈是系统看见真实家庭情境的重要来源 |

### 13.5 误区五：希望系统直接给出唯一答案

| 误区 | 纠正 |
|------|------|
| 系统应给出确定答案 | 当前阶段系统更擅长帮助看清问题和变化 |

## 15. 家长端常见问答建议

### 15.1 为什么有些地方只写"需要关注"，没写"严重问题"

**回答**：因为系统会尽量按实际证据强度表达，避免过度下结论。

### 15.2 为什么系统和我感觉不完全一样

**回答**：系统主要基于已有数据和事件，家长掌握的生活细节更多，反馈能帮助系统更准确。

### 15.3 为什么建议有时候比较保守

**回答**：因为当前阶段更强调低风险、可执行和可验证。

### 15.4 为什么不是每天都给很多新变化

**回答**：不是每天都需要强输出，系统更重视有依据的连续变化。

### 15.5 孩子状态不好是不是说明他有问题

**回答**：状态反映的是当前阶段情况，不代表孩子本身有问题。状态会随着新数据更新。

### 15.6 反馈了会有用吗

**回答**：有用。家长反馈会进入系统，帮助校正判断和补充情境信息。

## 18. 风险与边界

### 18.1 不应制造焦虑型使用习惯

| 原则 | 说明 |
|------|------|
| 家长端不能被设计成每天来找"坏消息"的地方 | 避免焦虑 |

### 18.2 不应让家长误以为系统能看穿一切

| 原则 | 说明 |
|------|------|
| 系统有边界，有数据盲区 | 诚实说明 |

### 18.3 不应让家长把系统结论拿去压孩子

| 原则 | 说明 |
|------|------|
| 这是辅助观察工具，不是家庭权威武器 | 正确使用 |

### 18.4 不应让家长因术语看不懂而放弃使用

| 原则 | 说明 |
|------|------|
| 表达必须持续克制和可读 | 易懂优先 |

## 20. 家长端使用快速参考卡

### 20.1 每周快速检查清单

```
□ 打开首页查看当前状态

□ 查看当前关注点（1-3 个）
```

## 24. 结论

家长使用引导的核心，不是把系统说得多强，而是让家长形成一种稳定、合理、不过度焦虑的使用方式。

它必须做到：

| 做到 | 说明 |
|------|------|
| 知道怎么看 | 掌握基本阅读方法 |
| 知道怎么看重点 | 聚焦核心信息 |
| 知道什么时候反馈 | 适时参与 |
| 知道系统有边界 | 理解系统限制 |

做到这几点，家长端才不会从帮助工具滑成家庭误解制造器。
