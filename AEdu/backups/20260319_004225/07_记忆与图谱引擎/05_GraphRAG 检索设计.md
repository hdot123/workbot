## 1. 文档目的

本文档用于定义教育孪生项目中的 GraphRAG 检索设计，包括检索目标、检索对象、检索流程、上下文装配规则、排序原则与质量要求。

本文档重点回答以下问题：

1. 为什么本项目需要 GraphRAG，而不是只做普通向量检索
2. GraphRAG 检索的对象有哪些
3. 检索时如何兼顾当前事实、关系结构和历史事件
4. 检索结果如何装配为可供观察层、报告层和推演层使用的上下文
5. 如何避免检索出一堆"都相关但没用"的上下文噪音

---

## 2. GraphRAG 定义

GraphRAG，是指结合图谱结构、事实关系事件对象、文本片段索引和上下文装配逻辑的一种增强检索机制。

在本项目中，GraphRAG 的目标不是"找最像的文本"，而是：

- 先找到与当前问题最相关的对象
- 再沿关系网络扩展上下文
- 再结合时间和角色口径做筛选
- 最后装配出可解释、可使用的上下文包

它比传统 RAG 多了一层"结构理解"。  
教育孪生这种多层对象系统，不加结构层，很容易检索得很热闹，回答却很漂。

---

## 3. 建设目标

### 3.1 支撑当前状态解释
帮助系统回答学生当前为什么呈现某种状态。

### 3.2 支撑报告与观察输出
前台和报告层需要"可解释、可追溯"的上下文。

### 3.3 支撑推演层输入
后台推演需要稳定、相关、低噪音的上下文集合。

### 3.4 控制历史噪音
避免旧事件、弱相关文本、低质量片段抢占上下文。

### 3.5 支撑多角色输出
家长、老师、学校、后台推演使用的上下文粒度不同。

---

## 4. 设计原则

### 4.1 当前态优先
当前有效事实优先于旧事件片段。

### 4.2 结构优先于语义相似
先确定对象和关系，再补文本解释。

### 4.3 时间敏感
越新的关键事件和当前有效关系，优先级越高。

### 4.4 角色敏感
不同角色可见的上下文范围与表达深度不同。

### 4.5 可解释优先
检索结果必须能回答"为什么选了这些"，不是神秘拼接。

## 5. 检索对象范围

GraphRAG 建议至少检索以下对象：

| 对象类型 | 说明 | 优先级 |
|----------|------|--------|
| 当前事实对象 | 学生当前状态、风险等级等 | 高 |
| 稳定关系对象 | 知识点依赖、学生 - 班级关系等 | 高 |
| 关键事件对象 | 最近作业、考试、反馈等 | 中 |
| 知识底座对象 | 知识点定义、能力点、误区等 | 中 |
| 规则与约束对象 | 地区规则、考试口径等 | 视场景而定 |
| 解释文本片段 | 知识点说明、题型解释等 | 中 |
| 报告与反馈片段 | 历史报告摘要、反馈记录 | 低 |

---

## 6. 当前事实对象

### 6.1 作用
提供"现在是什么"的稳定基础。

### 6.2 典型内容
- 当前风险等级
- 当前重点知识点
- 当前学科状态
- 当前教材绑定
- 当前班级与年级信息

### 6.3 原则
当前事实通常应作为检索入口之一，而不是被埋在一堆历史文本后面。

---

## 7. 稳定关系对象

### 7.1 作用
提供"这些对象之间如何相连"的结构背景。

### 7.2 典型内容
- 知识点依赖关系
- 知识点与能力点支撑关系
- 学生与班级/学校关系
- 风险对象与知识点关系

### 7.3 原则
关系对象提供结构骨架，帮助系统理解对象间的语义连接。

---

## 8. 关键事件对象

### 8.1 作用
提供"最近发生了什么"和"为什么状态变化"。

### 8.2 典型内容
- 最近作业事件
- 最近考试事件
- 关键老师反馈
- 家长补充反馈
- 干预执行事件
- 状态更新事件

### 8.3 原则
事件应按时间窗口和重要性筛选，不能把整条历史流水账都塞进上下文。

---

## 9. 知识底座对象

### 9.1 作用
提供教材、章节、知识点、能力点、题型、误区等结构解释。

### 9.2 典型内容
- 知识点定义
- 能力点定义
- 章节路径
- 误区说明
- 题型特征
- 依赖关系说明

### 9.3 原则
知识底座提供通用解释框架，帮助学生特定情况的可理解性。

---

## 10. 规则与约束对象

### 10.1 作用
为推演层和某些解释场景提供制度边界。

### 10.2 典型内容
- 地区规则
- 教材适配规则
- 考试口径
- 选科要求
- 志愿填报约束（后续）

### 10.3 原则
规则对象确保解释和建议不脱离真实教育环境。

---

## 11. 检索入口设计

建议 GraphRAG 至少支持以下检索入口：

| 入口类型 | 说明 | 典型场景 |
|----------|------|----------|
| 学生当前状态入口 | 以学生当前状态为起点 | 家长端观察摘要、老师端学生详情 |
| 学科/知识点入口 | 以学科或知识点为起点 | 分析某知识点为何不稳 |
| 风险与问题入口 | 以风险信号为起点 | 风险解释、干预动作前上下文整理 |
| 报告生成入口 | 以报告周期为起点 | 周报、月报生成 |
| 推演任务入口 | 以推演任务为起点 | 风险预测、干预模拟 |

---

## 12. 学生当前状态入口

### 12.1 场景
- 生成家长端观察摘要
- 生成老师端学生详情
- 解释当前风险

### 12.2 优先对象
- 当前事实
- 最近关键事件
- 重点知识点关系
- 当前误区与题型解释

### 12.3 检索流程
```text
读取学生当前态事实
    ↓
沿关系网络扩展相关知识点
    ↓
补充近窗口关键事件
    ↓
召回相关解释文本
    ↓
按角色过滤装配
```

---

## 13. 学科/知识点入口

### 13.1 场景
- 分析某知识点为何不稳
- 查看某学科最近变化

### 13.2 优先对象
- 相关知识点
- 依赖链
- 能力点映射
- 题型与误区
- 近窗口相关事件

### 13.3 检索流程
```text
定位目标知识点
    ↓
召回知识点定义与依赖关系
    ↓
检索关联能力点与题型
    ↓
补充该知识点相关事件
    ↓
装配上下文包
```

---

## 14. 风险与问题入口

### 14.1 场景
- 风险解释
- 干预动作前上下文整理
- 后台风险复核

### 14.2 优先对象
- 当前风险事实
- 风险相关事件链
- 支撑关系
- 受影响知识点和能力点
- 数据完整性与边界说明

### 14.3 检索流程
```text
定位风险信号
    ↓
追溯风险来源事件
    ↓
扩展受影响知识点
    ↓
召回相关解释与规则
    ↓
装配风险解释上下文
```

---

## 15. 报告生成入口

### 15.1 场景
- 周报
- 月报
- 老师端阶段总结
- 家长端重点提示

### 15.2 优先对象
- 当前态快照
- 时间窗口内关键变化事件
- 对应知识点和解释块
- 低风险建议相关上下文

### 15.3 检索流程
```text
确定报告周期
    ↓
召回周期内关键事件
    ↓
聚合状态变化趋势
    ↓
补充知识点解释
    → 装配报告上下文
```

---

## 16. 推演任务入口

### 16.1 场景
- 风险预测
- 干预模拟
- 志愿推演（后续）

### 16.2 优先对象
- 当前事实与状态快照
- 历史事件趋势
- 关键依赖关系
- 规则约束
- 干预历史记录

### 16.3 检索流程
```text
确定推演场景
    ↓
召回当前态与历史趋势
    ↓
加载规则约束
    → 装配推演输入上下文
```

## 17. 检索流程建议

建议标准流程如下：

```text
接收查询意图
    ↓
确定角色与场景
    ↓
选择主检索入口对象
    ↓
召回当前事实
    ↓
沿关系图扩展相关对象
    ↓
补充时间窗口内关键事件
    ↓
召回相关解释文本片段
    ↓
按规则过滤与排序
    ↓
装配上下文包
```

## 18. 查询意图拆解建议

检索前应先拆解问题：

### 18.1 谁在问

| 角色 | 说明 |
|------|------|
| 家长 | 需要易懂、可操作的上下文 |
| 老师 | 需要结构化、可教学的上下文 |
| 学校管理者 | 需要聚合、趋势性上下文 |
| 后台推演模块 | 需要完整、低噪音的结构化上下文 |

### 18.2 在问什么

| 问题类型 | 说明 |
|----------|------|
| 当前状态 | 现在是什么情况 |
| 变化原因 | 为什么变成这样 |
| 风险来源 | 风险来自哪里 |
| 干预依据 | 为什么要这样干预 |
| 决策约束 | 有哪些边界条件 |

### 18.3 针对谁

| 对象 | 说明 |
|------|------|
| 单学生 | 个体详细上下文 |
| 班级 | 班级聚合上下文 |
| 学科 | 学科维度上下文 |
| 某知识点 | 知识点详细上下文 |
| 某推演任务 | 任务特定上下文 |

---

## 19. 召回策略建议

建议采用"三段式召回"：

### 19.1 第一段：结构召回

优先召回：

| 内容 | 说明 |
|------|------|
| 当前事实 | 学生当前状态 |
| 核心关系 | 关键结构连接 |
| 关键对象 | 核心知识点、能力点等 |

### 19.2 第二段：时序召回

补充：

| 内容 | 说明 |
|------|------|
| 近窗口关键事件 | 最近发生了什么 |
| 历史变化链 | 怎么变成这样的 |
| 干预记录 | 做过什么干预 |

### 19.3 第三段：文本召回

补充：

| 内容 | 说明 |
|------|------|
| 知识定义 | 知识点、能力点定义 |
| 误区说明 | 常见误区解释 |
| 题型特征 | 题型说明 |
| 规则说明 | 规则与约束 |
| 历史反馈摘要 | 老师/家长反馈摘要 |

---

## 20. 排序原则建议

检索结果建议按以下因素综合排序：

| 排序因素 | 权重建议 | 说明 |
|----------|----------|------|
| 当前有效性 | 高 | 当前有效事实优先 |
| 与查询对象距离 | 高 | 关系网络距离近优先 |
| 时间新鲜度 | 中 | 越新越优先 |
| 关系强度 | 中 | 强关系优先 |
| 来源可信度 | 中 | 来源可信优先 |
| 角色适配度 | 视角色而定 | 符合角色口径优先 |
| 解释价值 | 视场景而定 | 能帮助解释优先 |

**说明**：
"最像查询文本"不应成为唯一排序规则，这对教育孪生来说太单薄了。

---

## 21. 时间窗口规则

### 21.1 默认窗口

建议按场景设置默认窗口：

| 场景 | 默认窗口 | 说明 |
|------|----------|------|
| 周报 | 7 天 | 最近一周变化 |
| 月报 | 30 天 | 最近一月变化 |
| 风险解释 | 14 天优先 | 必要时向前扩展 |
| 推演 | 30-90 天 | 视任务而定 |

### 21.2 原则

旧事件可补充背景，但不能轻易盖过当前事实和近窗口变化。

---

## 22. 上下文装配规则

建议最终上下文包至少分成以下四层：

| 层级 | 内容 | 作用 |
|------|------|------|
| 当前态摘要层 | 当前主要状态、重点问题、主要风险 | 回答"现在是什么" |
| 结构解释层 | 知识点定义、依赖关系、能力点支撑 | 回答"为什么" |
| 关键事件层 | 最近高价值事件、关键变化节点 | 回答"怎么变成这样" |
| 边界与约束层 | 数据完整性说明、判断边界、不确定性提示 | 回答"有哪些限制" |

---

## 23. 当前态摘要层

内容包括：

| 内容 | 说明 |
|------|------|
| 当前主要状态 | 学生当前整体状态 |
| 当前重点问题 | 当前最需要注意的问题 |
| 当前主要风险 | 当前风险等级和来源 |
| 当前学科/知识点焦点 | 当前重点学科和知识点 |

这是上下文的"现在层"。

---

## 24. 结构解释层

内容包括：

| 内容 | 说明 |
|------|------|
| 相关知识点定义 | 知识点是什么 |
| 依赖关系 | 前置知识点有哪些 |
| 能力点支撑 | 支撑哪些能力 |
| 题型与误区说明 | 常见题型和误区 |

这是上下文的"为什么层"。

---

## 25. 关键事件层

内容包括：

| 内容 | 说明 |
|------|------|
| 最近高价值事件 | 最近发生了什么重要事情 |
| 关键变化节点 | 状态变化的关键时间点 |
| 干预前后事件对比 | 干预前后的变化 |
| 老师/家长反馈摘要 | 关键反馈内容 |

这是上下文的"怎么变成这样层"。

---

## 26. 边界与约束层

内容包括：

| 内容 | 说明 |
|------|------|
| 数据完整性说明 | 数据覆盖情况 |
| 当前判断边界 | 当前判断的适用范围 |
| 规则约束 | 相关规则限制 |
| 不确定性提示 | 不确定因素说明 |

这是上下文的"别说太满层"。

---

## 27. 角色化过滤规则

### 27.1 家长端

| 优先 | 内容 |
|------|------|
| ✅ | 当前状态 |
| ✅ | 变化趋势 |
| ✅ | 可理解解释 |
| ✅ | 低风险建议 |

| 过滤 | 内容 |
|------|------|
| ❌ | 后台中间变量 |
| ❌ | 高敏感未校准推演 |
| ❌ | 过多技术指标 |

### 27.2 老师端

| 优先 | 内容 |
|------|------|
| ✅ | 知识点、能力点、题型、误区 |
| ✅ | 事件依据 |
| ✅ | 结构解释 |

### 27.3 学校端

| 优先 | 内容 |
|------|------|
| ✅ | 聚合分布 |
| ✅ | 趋势 |
| ✅ | 重点问题群体 |
| ✅ | 规则边界摘要 |

### 27.4 推演层

| 优先 | 内容 |
|------|------|
| ✅ | 完整结构对象 |
| ✅ | 当前态与事件链 |
| ✅ | 规则与约束 |
| ✅ | 置信和边界标签 |

---

## 28. 质量要求

| 要求 | 说明 |
|------|------|
| 低噪音 | 上下文必须够相关，不可堆砌 |
| 可解释 | 每块上下文都应能说明为什么被选中 |
| 可追溯 | 上下文对象应能追到事实、事件、关系和来源 |
| 新鲜度可控 | 当前态和近窗口事件应优先 |
| 角色适配 | 不同角色使用的上下文必须不同 |

---

## 29. 常见错误

| 错误 | 后果 |
|------|------|
| 只做向量相似 | 会召回大量像但没用的内容 |
| 不区分当前态和历史事件 | 容易拿旧问题解释当前状态 |
| 不做角色过滤 | 会把后台信息直接泄到前台口径里 |
| 关系扩展过度 | 一扩就扩成全图漫游，结果上下文噪音暴增 |

---

## 30. MVP 建议范围

MVP 阶段建议优先支持：

| 能力 | 说明 |
|------|------|
| 单学生 GraphRAG 检索 | 支持单学生上下文检索 |
| 当前态 + 关键事件 + 知识点解释的三层装配 | 基础上下文装配 |
| 家长端与老师端两套角色化过滤 | 基础角色适配 |
| 近 7/30 天窗口的事件召回 | 基础时间窗口控制 |
| 知识点/能力点/误区/题型的基础结构扩展 | 基础结构扩展 |

## 34. 风险与边界

| 风险 | 说明 |
|------|------|
| 不能忽视结构优先 | 只做向量相似会召回大量无用内容 |
| 不能忽视当前态优先 | 旧事件不能盖过当前事实 |
| 不能忽视角色过滤 | 不同角色上下文必须不同 |
| 不能忽视关系扩展控制 | 过度扩展会导致噪音暴增 |
| 不能忽视可解释性 | 检索结果必须能说明为什么被选中 |

## 35. 结论

GraphRAG 检索设计的核心价值，不是让系统"检得更多"，而是让系统在面对复杂教育上下文时，能够稳定地拿到真正有解释力、能支撑当前任务的上下文包。

它必须坚持：

| 原则 | 说明 |
|------|------|
| 先结构 | 先确定对象和关系 |
| 再时序 | 再补充时间窗口内事件 |
| 后文本 | 最后补文本解释 |
| 强调当前态 | 当前事实优先 |
| 控制噪音 | 过滤无关内容 |

否则系统看起来像懂得很多，实际上只是把相关内容胡乱堆在一起。
