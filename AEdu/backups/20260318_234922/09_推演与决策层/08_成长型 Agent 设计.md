# 成长型 Agent 设计

> 文档编号：SIM-008  
> 版本：V1.0  
> 创建日期：2024  
> 最后更新：待定  
> 维护人：推演与决策层负责人

---

## 1. 文档目的

本文档用于定义教育孪生项目中"成长型 Agent"的定位、能力边界、输入输出、运行方式和与 StudentTwinAgent 的关系。

本文档重点回答以下问题：

1. 什么是成长型 Agent
2. 成长型 Agent 和 StudentTwinAgent 是什么关系
3. 成长型 Agent 主要解决什么问题
4. 它与决策型 Agent 的边界在哪里
5. 为什么项目应优先把成长型 Agent 做稳

---

## 2. 定义

成长型 Agent，是以 StudentTwinAgent 为个体状态底座，面向学生长期学习过程中的观察、解释、提醒和陪伴支持而构建的一类低风险、持续性、过程导向型智能体。

它关注的是：

- 当前状态怎么理解
- 最近发生了什么变化
- 哪些地方值得关注
- 如何帮助老师和家长持续看懂学生成长过程

它不是：
- 高风险决策代理
- 升学最终建议系统
- 替代老师判断的自动引擎

---

## 3. 核心定位

成长型 Agent 的一句话定位：

**围绕学生长期成长过程，提供连续观察、变化解释和低风险支持。**

---

## 4. 建设目标

### 4.1 提供持续观察能力
让系统不只是阶段性出报告，而是能持续看见变化。

### 4.2 提供低风险支持
输出重点关注、解释说明、轻建议和提醒。

### 4.3 建立使用信任
让家长和老师先相信系统"看得懂过程"，再逐步接受更复杂能力。

### 4.4 为决策型能力打地基
成长型 Agent 越稳，后面的推演和决策型 Agent 越不容易漂。

---

## 5. 设计原则

### 5.1 过程优先
成长型 Agent 更看重过程变化，而不是单次结果。

### 5.2 解释优先
输出要说明"为什么值得关注"。

### 5.3 低风险优先
优先做家长和老师可接受、可验证的低风险输出。

### 5.4 持续更新
每次新事件进入后，应能更新理解和输出。

### 5.5 与当前态强绑定
成长型 Agent 不应脱离 StudentTwinAgent 当前态自由发挥。

---

## 6. 主要能力范围

建议成长型 Agent 主要承担以下能力：

1. 当前状态解释
2. 近期变化总结
3. 风险前兆提醒
4. 周报/月报摘要生成
5. 关注点提炼
6. 低风险行动提示
7. 反馈理解与闭环

---

## 7. 当前状态解释能力

成长型 Agent 应能解释：

- 当前为什么是重点关注
- 当前问题落在哪些知识点/行为环节
- 当前变化来自哪些近期事件
- 当前判断的边界是什么

---

## 8. 近期变化总结能力

例如：

- 哪个学科最近更稳定
- 哪类知识点近期波动更明显
- 作业节奏有没有变化
- 风险是上升、下降还是平稳

这类能力是成长型 Agent 的核心。

---

## 9. 风险前兆提醒能力

成长型 Agent 可在低风险范围内提醒：

- 某问题在重复出现
- 某类波动值得留意
- 某段时间输入减少导致判断不稳
- 某知识链路近期问题集中

### 原则
提醒是"值得关注"，不是"危险结论"。

---

## 10. 周报/月报摘要能力

成长型 Agent 应协同报告层，完成：

- 周报摘要
- 月报摘要
- 变化亮点提炼
- 关注项排序
- 可理解解释

---

## 11. 低风险行动提示能力

可提供：

- 优先复习某章节
- 建议关注某类错题
- 建议近期保持固定反馈节奏
- 建议老师关注某步骤表达问题

### 原则
动作提示低风险、可解释、可撤回。

---

## 12. 成长型 Agent 的输入

建议主要输入包括：

- StudentTwinAgent 当前状态
- 时间演化层摘要
- 行为执行层摘要
- 关系上下文层摘要
- GraphRAG 上下文包
- 报告模板与角色约束
- 低风险建议规则

---

## 13. 成长型 Agent 的输出

建议主要输出包括：

- 当前观察摘要
- 本周变化摘要
- 重点关注项
- 解释性说明
- 低风险行动提示
- 边界说明

---

## 14. 面向角色建议

### 家长端
偏：
- 变化解释
- 可理解描述
- 家庭可配合提示

### 老师端
偏：
- 知识点级问题
- 班级相对位置
- 近期变化原因
- 跟进抓手

### 学校端
偏：
- 聚合观察
- 风险群体变化
- 试点运行摘要

---

## 15. 与 StudentTwinAgent 的关系

两者关系建议明确如下：

### StudentTwinAgent
是学生当前态和结构化个体模型核心。

### 成长型 Agent
是基于 StudentTwinAgent 当前态和长期记忆运行的解释型、观察型上层 Agent。

也就是说：

- StudentTwinAgent 更像内核
- 成长型 Agent 更像面向用户的理解层

---

## 16. 与决策型 Agent 的区别

### 成长型 Agent
- 关注现在和最近变化
- 强调理解与陪伴
- 低风险高频
- 适合前台持续使用

### 决策型 Agent
- 关注动作比较与未来路径
- 强调模拟与选择
- 高风险低频
- 更适合后台与受控开放

---

## 17. 触发方式建议

成长型 Agent 主要可由以下方式触发：

- 页面查看触发
- 周报/月报任务触发
- 新事件进入后摘要刷新触发
- 老师/家长反馈后解释刷新触发

---

## 18. 风险控制建议

成长型 Agent 不应：

- 直接输出高风险决策建议
- 将推断包装成事实
- 对未来结果做强承诺
- 越权替代老师/家长判断

### 原则
成长型 Agent 的强项是看懂过程，不是直接决定未来。

---

## 19. 工程实现建议

建议成长型 Agent 调用至少带入：

- `task_type`
- `role_type`
- `student_state_ref`
- `context_bundle_ref`
- `output_constraints`
- `risk_policy`

---

## 20. 质量指标建议

建议监控：

- 输出可解释率
- 用户阅读完成率
- 老师/家长反馈采纳率
- 误解投诉率
- 边界说明覆盖率
- 周报可读性反馈

---

## 21. MVP 建议范围

MVP 阶段成长型 Agent 优先支持：

- 当前状态解释
- 周报/月报摘要
- 重点关注项提炼
- 低风险提醒
- 家长端与老师端基础解释能力

## 24. 结论

成长型 Agent 的核心价值，不是让系统更像一个会说话的助手，而是让系统在学生成长过程中持续提供低风险、高解释性、可验证的支持。

它必须坚持：

- 过程优先
- 解释优先
- 当前态优先
- 低风险优先
- 持续更新

只有成长型 Agent 先做稳，教育孪生项目后面的推演与决策能力才有可信基础。
