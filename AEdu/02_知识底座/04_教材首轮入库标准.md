## 一、文档目的

本文档用于定义教育孪生项目中教材首轮入库的标准流程、质量要求、验收标准与版本管理规范。

教材首轮入库是固定知识底座建设的起点，直接决定后续学生数字孪生、图谱记忆、观察推演的准确性与可靠性。

本文档回答以下问题：
- 首轮入库的范围边界是什么
- 入库流程包含哪些步骤
- 每个步骤的质量标准是什么
- 如何验收入库成果
- 如何处理版本变更与回滚

## 二、适用范围

### 2.1 适用对象
- 教材结构化入库项目
- 知识点提取与标注项目
- 能力点映射项目
- 教材锚点定位项目

### 2.2 不适用范围

- 动态学习事件处理

### 2.3 首轮优先范围
| 维度 | 首轮范围 |
|------|----------|
| 学段 | 高中 |
| 学科 | 物理（优先）、数学、语文、英语 |
| 教材版本 | 人教版必修一（优先） |
| 地区 | 试点地区（待定） |

## 三、入库前准备

### 3.1 原始资料收集
入库前必须完成以下资料收集：

| 资料类型 | 要求 | 存放位置 |
|----------|------|----------|
| 教材 PDF | 清晰完整版，含目录、正文、习题、实验 | `13_原始资料库/教材 PDF/` |
| 教师用书 | 含教学目标、重难点说明 | `13_原始资料库/教师用书/` |
| 课程标准 | 教育部或地区课程标准 | `13_原始资料库/课程标准/` |
| 地区考纲 | 试点地区考试大纲 | `13_原始资料库/地区考纲/` |

### 3.2 资料质量检查
- PDF 可复制性检查（非图片型 PDF 优先）
- 目录完整性检查
- 页码连续性检查
- 章节清晰度检查

### 3.3 工具准备
- OCR 工具（备用）
- 标注工具（待选型）
- 版本管理工具（Git）
- 质量校验脚本（待开发）

### 3.4 人员准备
| 角色 | 职责 | 人数 |
|------|------|------|
| 学科专家 | 知识点提取与审核 | 1-2 人 |
| 结构化专员 | 表格填写与数据录入 | 1-2 人 |
| 质量审核员 | 质量校验与问题反馈 | 1 人 |
| 项目负责人 | 进度管理与版本发布 | 1 人 |

## 四、入库流程

教材首轮入库流程分为六个阶段：

```
资料收集 → 章节树拆解 → 知识点提取 → 能力点映射 → 锚点定位 → 质量校验 → 版本发布
```

### 4.1 阶段一：资料收集
**输入**：教材 PDF、教师用书、课程标准  
**输出**：原始资料归档记录  
**验收标准**：
- [ ] 所有资料已存放到指定目录
- [ ] 资料清单已记录
- [ ] 资料版本号已标记

### 4.2 阶段二：章节树拆解
**输入**：教材 PDF 目录  
**输出**：章节树表（初稿）  
**验收标准**：
- [ ] 章、节、小节三级结构完整
- [ ] 章节名称与教材一致
- [ ] 章节编号规则统一
- [ ] 章节页码范围已标记

**章节树表字段要求**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| chapter_id | string | 是 | 章节唯一标识 |
| book_id | string | 是 | 教材版本标识 |
| level | int | 是 | 层级（1=章，2=节，3=小节） |
| chapter_name | string | 是 | 章节名称 |
| parent_id | string | 否 | 父章节 ID |
| start_page | int | 是 | 起始页码 |
| end_page | int | 是 | 结束页码 |
| sort_order | int | 是 | 同级排序 |

### 4.3 阶段三：知识点提取
**输入**：章节树、教材正文  
**输出**：知识点表（初稿）  
**验收标准**：
- [ ] 每个章节下的知识点完整覆盖
- [ ] 知识点命名规范统一
- [ ] 知识点难度等级已标注
- [ ] 知识点与章节映射关系正确

**知识点表字段要求**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| kp_id | string | 是 | 知识点唯一标识 |
| chapter_id | string | 是 | 所属章节 ID |
| kp_name | string | 是 | 知识点名称 |
| kp_code | string | 是 | 知识点编码 |
| difficulty | string | 是 | 难度等级（L1/L2/L3） |
| importance | string | 是 | 重要程度（核心/重点/一般） |
| description | text | 否 | 知识点描述 |
| prerequisites | string[] | 否 | 前置知识点 ID 列表 |

### 4.4 阶段四：能力点映射
**输入**：知识点表、课程标准  
**输出**：能力点表、知识能力映射表  
**验收标准**：
- [ ] 能力点定义符合课程标准
- [ ] 每个知识点至少映射一个能力点
- [ ] 能力点层级清晰

**能力点表字段要求**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ap_id | string | 是 | 能力点唯一标识 |
| ap_name | string | 是 | 能力点名称 |
| ap_code | string | 是 | 能力点编码 |
| ap_level | string | 是 | 能力层级（识记/理解/应用/分析/综合/评价） |
| ap_description | text | 否 | 能力点描述 |

**知识能力映射表字段要求**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| map_id | string | 是 | 映射 ID |
| kp_id | string | 是 | 知识点 ID |
| ap_id | string | 是 | 能力点 ID |
| weight | float | 是 | 权重（0-1） |

### 4.5 阶段五：教材锚点定位
**输入**：知识点表、教材 PDF  
**输出**：教材锚点表  
**验收标准**：
- [ ] 每个知识点至少有一个锚点
- [ ] 锚点页码准确
- [ ] 锚点类型标注清晰

**教材锚点表字段要求**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| anchor_id | string | 是 | 锚点唯一标识 |
| kp_id | string | 是 | 知识点 ID |
| book_id | string | 是 | 教材版本 ID |
| page_num | int | 是 | 页码 |
| anchor_type | string | 是 | 锚点类型（正文/例题/习题/实验/图表） |
| position_desc | string | 否 | 位置描述（如"第 3 段"、"图 2-1"） |

### 4.6 阶段六：依赖关系标注
**输入**：知识点表、学科专家意见  
**输出**：依赖关系表  
**验收标准**：
- [ ] 前置依赖关系完整
- [ ] 后置依赖关系可追溯
- [ ] 跨章节依赖已标注

**依赖关系表字段要求**：
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| dep_id | string | 是 | 依赖关系 ID |
| kp_id | string | 是 | 当前知识点 ID |
| prerequisite_kp_id | string | 是 | 前置知识点 ID |
| dep_type | string | 是 | 依赖类型（强依赖/弱依赖/建议前置） |
| dep_reason | string | 否 | 依赖原因说明 |

## 五、质量标准

### 5.1 完整性标准
| 检查项 | 要求 | 合格线 |
|--------|------|--------|
| 章节覆盖率 | 所有章节已拆解 | 100% |
| 知识点覆盖率 | 核心知识点无遗漏 | ≥95% |
| 能力点映射率 | 知识点映射能力点 | ≥90% |
| 锚点覆盖率 | 知识点有锚点定位 | ≥90% |
| 依赖关系完整率 | 核心依赖已标注 | ≥80% |

### 5.2 一致性标准
- 同一学科内知识点命名风格统一
- 章节编号规则统一
- 难度等级标注标准统一
- 编码规则统一

### 5.3 准确性标准
- 知识点名称与教材表述一致率 ≥95%
- 锚点页码准确率 ≥98%
- 依赖关系学科审核通过率 ≥90%

### 5.4 可追溯性标准
- 每个知识点可回溯到教材位置
- 每个能力点可回溯到课程标准
- 每次变更有记录可查

## 六、质量校验流程

### 6.1 自检
结构化专员完成入库后，按检查清单自检：
- [ ] 表格字段完整
- [ ] 命名规范检查
- [ ] 编码唯一性检查
- [ ] 映射关系检查

### 6.2 互检
不同专员之间交叉检查：
- [ ] 随机抽查 20% 知识点
- [ ] 核对锚点页码
- [ ] 检查依赖关系逻辑

### 6.3 学科审核
学科专家进行专业审核：
- [ ] 知识点提取准确性
- [ ] 能力点映射合理性
- [ ] 依赖关系正确性

### 6.4 系统校验
运行自动化校验脚本：
```bash
# 示例校验命令
python validate_textbook.py --book_id=PEP_PHYSICS必修一 --check=all
```

校验项目：
- [ ] 外键完整性
- [ ] 编码唯一性
- [ ] 循环依赖检测
- [ ] 孤立节点检测

### 6.5 问题修复
发现问题后：
1. 记录问题到问题跟踪表
2. 分配修复责任人
3. 修复后重新校验
4. 问题关闭前需审核确认

## 七、验收标准

### 7.1 文档验收
| 文档 | 要求 | 状态 |
|------|------|------|
| 章节树表 | 完整、规范 | [ ] |
| 知识点表 | 完整、准确 | [ ] |
| 能力点表 | 符合课标 | [ ] |
| 教材锚点表 | 页码准确 | [ ] |
| 依赖关系表 | 逻辑正确 | [ ] |
| 质量校验报告 | 问题已修复 | [ ] |

### 7.2 数据验收
- [ ] 所有表格已入库
- [ ] 数据格式符合标准
- [ ] 外键关系完整
- [ ] 无孤儿数据

### 7.3 业务验收
- [ ] 学科专家签字确认
- [ ] 项目负责人签字确认
- [ ] 试点学校确认可用（如有）

### 7.4 验收报告
验收完成后输出《教材入库验收报告》，包含：
- 入库范围说明
- 质量指标达成情况
- 遗留问题清单
- 版本发布建议

## 八、版本管理

### 8.1 版本编号规则
```
V[主版本].[次版本].[修订版本]
```

| 版本类型 | 触发条件 | 示例 |
|----------|----------|------|
| 主版本 | 教材改版、结构重大调整 | V2.0.0 |
| 次版本 | 新增章节/知识点、映射关系调整 | V1.1.0 |
| 修订版本 | 纠错、文案修正、锚点微调 | V1.0.1 |

### 8.2 版本发布流程
1. 完成入库与质量校验
2. 输出验收报告
3. 版本号递增
4. 更新版本变更记录
5. 发布通知相关方
6. 归档旧版本（如有）

### 8.3 版本变更记录
每次版本变更必须记录：
| 字段 | 说明 |
|------|------|
| 版本号 | 变更后的版本号 |
| 变更日期 | YYYY-MM-DD |
| 变更人 | 负责人姓名 |
| 变更类型 | 主版本/次版本/修订版本 |
| 变更内容 | 具体变更描述 |
| 影响范围 | 影响的知识点/章节列表 |
| 回滚方案 | 如需回滚的操作步骤 |

### 8.4 版本回滚
当新版本发现严重问题时，可执行回滚：
1. 确认回滚原因与影响
2. 审批回滚申请
3. 恢复上一稳定版本数据
4. 通知相关方
5. 记录回滚原因与时间

## 九、异常处理

### 9.1 常见问题与处理
| 问题类型 | 处理方式 |
|----------|----------|
| 教材版本不清晰 | 联系试点学校确认 |
| 知识点边界模糊 | 学科专家裁定 |
| 锚点定位困难 | 人工核对 + 多人确认 |
| 依赖关系争议 | 组织学科讨论会 |
| OCR 识别错误 | 人工校对修正 |

### 9.2 问题升级机制
- 一级问题：结构化专员自行处理
- 二级问题：上报质量审核员
- 三级问题：上报项目负责人
- 四级问题：组织学科专家会审

### 9.3 问题跟踪表
维护《教材入库问题跟踪表》，记录：
- 问题编号
- 问题描述
- 发现日期
- 责任人
- 处理状态
- 解决方案
- 关闭日期
