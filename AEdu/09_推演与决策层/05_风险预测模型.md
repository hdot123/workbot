## 1. 文档目的

本文档用于定义教育孪生项目中的风险预测模型，包括风险定义、风险类型、输入输出、预测逻辑、结果分级、边界条件与使用方式。

本文档重点回答以下问题：

1. 系统中的"风险"到底指什么
2. 风险模型预测哪些风险，不预测哪些风险
3. 风险预测依赖什么输入
4. 风险结果如何表达与使用
5. 如何防止风险预测变成过度诊断工具

---

## 2. 风险预测模型定义

风险预测模型，是后台推演层中用于识别学生当前状态在未来一定时间窗口内可能演化为不良学习结果或不稳定状态的结构化预测能力。

它关注的是：

- 学习稳定性风险
- 知识掌握风险
- 进度偏离风险
- 节奏中断风险
- 干预滞后风险

它不是医疗诊断模型，不是人格判断模型，也不是"预测学生命运"的装置。

---

## 3. 风险预测模型的目标

### 3.1 提前识别问题苗头
在显性结果出现前识别趋势性风险。

### 3.2 支持干预优先级排序
帮助后台判断哪些学生、哪些知识点、哪些链路更值得先关注。

### 3.3 支持解释与复盘
风险不能只报红灯，必须能解释风险来自哪里。

### 3.4 支持推演层
风险结果可作为 InterventionSimulator 和后台决策层的重要输入。

### 3.5 控制误用
风险输出必须说明边界条件和不确定性。

---

## 4. 风险定义范围

本项目中的风险，优先指向教育系统中的结构性学习风险，而不是泛化到所有个人属性。

建议风险范围包括：

1. 学习稳定性风险
2. 知识掌握风险
3. 进度偏离风险
4. 节奏中断风险
5. 干预滞后风险

---

## 5. 风险类型定义

### 5.1 学习稳定性风险
指学习表现、作业完成、知识点状态等出现持续波动或恶化趋势。

**触发信号**：
- 近期成绩波动幅度增大
- 作业完成质量不稳定
- 知识点状态频繁变化

### 5.2 知识掌握风险
指某知识点或知识链条存在不稳定、退化或前置缺口扩大的风险。

**触发信号**：
- 重复错误率上升
- 知识点状态从"掌握"降级
- 前置知识缺口影响后续学习

### 5.3 进度偏离风险
指当前学习状态与教学进度、教材路径或阶段要求出现偏离。

**触发信号**：
- 当前进度明显落后教学计划
- 关键章节未掌握但已进入后续内容
- 阶段目标达成率持续偏低

### 5.4 节奏中断风险
指学习活动频率下降、延迟提交增多、学习连续性被打断的风险。

**触发信号**：
- 学习频率持续下降
- 作业延迟率上升
- 出现连续中断天数

### 5.5 干预滞后风险
指当前已经出现问题，但未及时采取动作，可能导致风险放大的情况。

**触发信号**：
- 风险信号已出现但无干预记录
- 同类问题历史干预效果良好但本次未执行
- 风险等级持续上升但无动作

---

## 6. 模型不预测的内容

风险预测模型原则上不直接预测：

- 人格特质
- 心理诊断结论
- 医疗相关结论
- 长期人生走向
- 与当前教育场景无直接关系的家庭隐私判断

这条边界必须钉死，否则系统会开始装作自己懂一切。

---

## 7. 风险预测时间窗口

建议风险预测按时间窗口输出：

| 窗口 | 含义 | 适用场景 |
|------|------|----------|
| 短期风险 | 1-2 周内 | 即时干预、快速响应 |
| 中期风险 | 3-6 周内 | 阶段调整、路径优化 |
| 阶段风险 | 1-2 月内 | 学期规划、重大决策参考 |

原则：
- MVP 优先支持短期和中期
- 长周期风险应更克制，且更强调条件限制

---

## 8. 风险预测输入

建议风险预测至少接收以下输入：

### 8.1 StudentTwinAgent 当前状态
- 当前总体学习状态
- 学科状态摘要
- 知识点掌握快照
- 行为节奏字段
- 当前关注点
- 数据完整性评分
- 推演准备字段
- 当前风险标签历史

### 8.2 学习事件时间序列
- 作业完成/延迟事件
- 错题事件
- 考试事件
- 教师反馈事件
- 家长反馈事件
- 干预执行事件
- 进度事件

### 8.3 图谱记忆上下文
- 历史事件链
- 历史状态变化
- 知识点依赖关系
- 干预前后变化链
- 风险因子上下文

### 8.4 知识底座与依赖关系
- 章节结构
- 知识点依赖
- 能力点映射
- 教材进度要求

### 8.5 干预历史记录
- 历史干预动作
- 干预执行结果
- 类似学生干预效果参考

### 8.6 外部规则约束
- 学段约束
- 教学进度要求
- 考试节点
- 角色权限约束

## 9. 风险特征建议

建议风险模型至少关注以下特征组：

### 9.1 学习表现特征

| 特征 | 说明 |
|------|------|
| 知识点掌握波动 | 近期知识点状态变化频率 |
| 学科稳定性变化 | 学科状态的标准差/变化幅度 |
| 错题重复率 | 同类错误重复出现比例 |
| 成绩波动幅度 | 考试成绩的变化范围 |

### 9.2 行为节奏特征

| 特征 | 说明 |
|------|------|
| 作业完成率 | 按时完成作业比例 |
| 延迟率 | 逾期提交比例 |
| 学习频率变化 | 学习活动频次的变化趋势 |
| 连续中断情况 | 学习活动的连续中断天数 |

### 9.3 结构依赖特征

| 特征 | 说明 |
|------|------|
| 前置知识缺口 | 前置知识点掌握不足 |
| 知识链阻塞点 | 依赖链中的薄弱环节 |
| 进度与掌握错位 | 当前进度与实际掌握不匹配 |

### 9.4 干预反馈特征

| 特征 | 说明 |
|------|------|
| 干预执行状态 | 是否有动作执行 |
| 干预后改善情况 | 动作执行后是否出现改善 |
| 历史类似动作效果 | 历史相似动作是否有效 |

### 9.5 数据质量特征

| 特征 | 说明 |
|------|------|
| 数据完整性 | 数据覆盖充分程度 |
| 数据新鲜度 | 最近更新时间 |
| 多来源覆盖度 | 输入来源多样性 |
| 冲突程度 | 多来源信息冲突程度 |

## 10. 风险输出结构

建议风险预测输出至少包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| risk_prediction_id | string | 风险预测任务标识 |
| student_id | string | 学生标识 |
| risk_type | string | 风险类型 |
| target_scope | object | 风险作用范围 |
| risk_level | string | 风险等级 |
| risk_direction | string | 风险变化方向 |
| time_window | string | 风险时间窗口 |
| confidence_score | float | 置信度 |
| main_factors | array[string] | 主要风险因子 |
| boundary_notes | string | 适用边界 |
| caution_notes | string | 风险提示 |
| created_at | datetime | 生成时间 |

## 11. 风险等级设计

建议风险等级采用四级：

| 等级 | 含义 | 处理方式 |
|------|------|----------|
| low | 低风险 | 正常观察，可进入建议层 |
| medium | 中风险 | 需关注，建议老师端查看 |
| high | 高风险 | 需人工复核，不直接前台输出 |
| pending_confirmation | 待确认 | 证据不足但出现弱信号 |

说明：
- `pending_confirmation` 很重要，专门用于证据不足但出现弱信号的情况
- 这能防止系统强行把模糊信号涂成红色警报

---

## 12. 风险变化方向设计

建议输出方向包括：

| 方向 | 含义 |
|------|------|
| up | 风险上升 |
| stable | 风险平稳 |
| down | 风险下降 |
| uncertain | 方向不确定 |

说明：
- 风险等级和风险方向不是同一个东西
- 一个风险等级中等的学生，也可能是风险正在下降
- 一个当前低风险学生，也可能有短期上升趋势

---

## 13. 风险对象范围

风险可作用于不同粒度对象：

### 13.1 知识点级风险

例：
- 受力分析掌握退化风险
- 牛顿定律前置知识缺口风险

### 13.2 章节级风险

例：
- 当前章节推进后续理解偏离风险
- 章节核心知识点掌握不足风险

### 13.3 学科级风险

例：
- 物理学科稳定性下降风险
- 数学学习节奏波动风险

### 13.4 个体总体级风险

例：
- 近期整体学习节奏中断风险
- 阶段性学习目标偏离风险

---

## 14. 风险预测方式建议

MVP 阶段建议采用"规则模型 + 风险特征加权 + 人工校准"的方式。

### 14.1 规则模型
用于识别明确阈值型风险。

示例规则：
- 连续 3 次作业延迟 → 节奏中断风险上升
- 知识点重复错误率 > 50% → 知识掌握风险中
- 学习频率连续 7 天下降 → 节奏中断风险中

### 14.2 特征加权
用于综合多个弱信号形成风险判断。

示例：
- 多个弱信号同时出现 → 风险等级提升
- 信号来源多样 → 置信度提升
- 信号方向一致 → 风险方向明确

### 14.3 人工校准
用于处理高风险、边界模糊或规则敏感场景。

---

## 15. 风险判断逻辑建议

建议逻辑流程如下：

```
读取当前状态
    ↓
读取时间序列事件
    ↓
计算风险特征
    ↓
检查规则阈值
    ↓
综合生成风险等级
    ↓
判断风险变化方向
    ↓
附加置信度与边界说明
    ↓
输出风险结果
```

---

## 16. 风险结果使用规则

### 16.1 后台推演层
可直接消费完整风险结果。

### 16.2 老师端
可展示经过解释和筛选的风险摘要。

### 16.3 家长端
仅展示低到中风险、经过解释且适合沟通的结果。

### 16.4 学校端
优先展示聚合分布，不优先展示高敏感个体风险细节。

---

## 17. 风险与解释绑定规则

风险不能孤立展示，必须绑定以下解释要素：

1. 风险对象
2. 风险时间窗口
3. 主要风险因子
4. 主要依据来源
5. 适用边界
6. 可行动建议（如适用）

---

## 18. 风险与干预动作库的关系

风险预测模型输出后，可进入干预动作库筛选流程：

```
风险结果
    ↓
匹配可适用动作
    ↓
过滤不适用动作
    ↓
形成候选动作集
    ↓
进入 InterventionSimulator 比较
```

这一步很关键。
没有动作承接的风险提示，很容易沦为高质量焦虑生产器。

---

## 19. 风险与图谱记忆层的关系

图谱记忆层为风险预测提供：

- 历史事件链
- 历史状态变化
- 知识点依赖关系
- 干预前后变化链
- 风险因子上下文

同时，风险结果也可回写图谱形成：

- Student -> has_risk_signal -> RiskSignal
- RiskSignal -> refers_to -> KnowledgePoint / Subject
- RiskSignal -> implied_by -> LearningEvent

---

## 20. 风险与数据完整性关系

### 20.1 原则

数据完整性不足时，风险模型必须降低输出强度。

### 20.2 建议规则

| 完整性等级 | 输出策略 |
|------------|----------|
| 高 | 可输出明确等级和方向 |
| 中 | 可输出等级，但需附加边界说明 |
| 低 | 优先输出 pending_confirmation 或信息不足 |

---

## 21. 风险与人工复核机制

以下情形建议进入人工复核：

- 高风险且置信度不足
- 多来源冲突明显
- 涉及高敏感建议
- 与老师当前明确判断冲突
- 长时间窗口风险结论过强

## 25. 结论

风险预测模型的任务，不是替系统扮演命运裁判，而是帮助系统更早看见可能的问题方向，并为后续解释、干预和推演提供依据。

它必须坚持：

- 先识别苗头
- 再说明依据
- 再控制输出强度
- 最后才进入动作与推演链路

否则风险模型会从"预警工具"滑成"焦虑制造引擎"。
