# 决策型 Agent 设计

> 文档编号：SIM-009  
> 版本：V1.0  
> 创建日期：2024  
> 最后更新：待定  
> 维护人：推演与决策层负责人

---

## 1. 文档目的

本文档用于定义教育孪生项目中"决策型 Agent"的定位、职责、输入输出、开放边界与风险控制方式。

本文档重点回答以下问题：

1. 什么是决策型 Agent
2. 决策型 Agent 与成长型 Agent 有何不同
3. 决策型 Agent 适合处理哪些问题
4. 它的输入、输出与约束分别是什么
5. 如何避免决策型 Agent 过早前台化

---

## 2. 定义

决策型 Agent，是以 StudentTwinAgent、规则约束、动作库、时序记忆与推演机制为基础，面向动作比较、路径选择、受控模拟和决策支持场景而构建的一类高约束、高风险、低频使用型智能体。

它关注的是：

- 当前有哪些可选路径
- 哪些动作更适配当前问题
- 不同动作之间的差异是什么
- 哪些场景需要人工复核
- 当前结论的边界和不确定性是什么

---

## 3. 核心定位

决策型 Agent 的一句话定位：

**在明确边界和规则约束下，为教育场景中的复杂选择提供受控决策支持。**

---

## 4. 建设目标

### 4.1 帮助比较动作与路径
不只是"看状态"，而是支持"下一步怎么做"的比较。

### 4.2 让高风险能力在后台先成熟
先在后台影子环境中运行，积累校准依据。

### 4.3 提高推演与选择的结构化程度
避免纯经验式、纯文本式建议。

### 4.4 与成长型 Agent 分层协作
成长型 Agent 负责日常理解，决策型 Agent 负责复杂比较。

---

## 5. 设计原则

### 5.1 高约束运行
决策型 Agent 必须受规则包、角色权限和边界说明约束。

### 5.2 比较优先
优先比较候选动作，而不是直接给唯一答案。

### 5.3 高风险后置
高风险能力先后台运行，再逐步受控开放。

### 5.4 不确定性显式表达
决策型输出必须说明置信范围和适用条件。

### 5.5 不直接改写现实状态
模拟和建议不应直接写回 StudentTwinAgent 当前现实状态。

---

## 6. 主要能力范围

建议决策型 Agent 主要承担：

1. 候选动作比较
2. 干预路径排序
3. 风险变化模拟
4. 规则边界判断
5. 受控决策支持
6. 人工复核触发建议

---

## 7. 候选动作比较能力

这是决策型 Agent 的核心能力之一。  
它要回答：

- 当前动作 A 和动作 B 哪个更适合
- 哪个动作成本更低
- 哪个动作适合当前角色执行
- 哪个动作更值得先试

---

## 8. 干预路径排序能力

当问题不是单点动作，而是路径时，决策型 Agent 可尝试排序：

- 先补前置知识
- 再强化专项练习
- 再进入阶段复盘

### 原则
排序是支持结构，不是"最终命令"。

---

## 9. 风险变化模拟能力

在明确边界下，决策型 Agent 可辅助判断：

- 当前风险若不处理，可能如何变化
- 某类动作对风险变化方向可能有什么影响
- 当前问题更适合立即介入还是继续观察

---

## 10. 规则边界判断能力

决策型 Agent 不只是"会模拟"，还应判断：

- 当前场景是否允许输出建议
- 当前角色是否有权限看到完整结果
- 当前是否必须人工复核
- 当前结论是否超出系统能力边界

---

## 11. 适用场景

建议决策型 Agent 优先用于：

1. 个体干预动作比较
2. 风险变化方向判断
3. 学习路径修复顺序判断
4. 选科/志愿等受控模拟场景
5. 复杂建议的后台校准场景

---

## 12. 当前不建议前台直接开放的场景

- 高确定性升学推荐
- 过强个体命运式判断
- 心理、家庭、人格深层推断
- 无足够数据支持的路径选择
- 对家长端直接开放的高风险比较结论

---

## 13. 决策型 Agent 的输入

建议主要输入包括：

- StudentTwinAgent 当前状态
- 时间演化摘要
- 关键时序记忆
- 关系上下文
- 推演准备层输出
- 干预动作库
- 规则包
- 外部数据与制度边界
- 当前角色权限

---

## 14. 决策型 Agent 的输出

建议主要输出包括：

- 候选动作对比
- 路径排序建议
- 边界说明
- 风险说明
- 置信带
- 是否需人工复核
- 是否可对外开放

---

## 15. 与成长型 Agent 的差异

### 成长型 Agent
- 高频
- 低风险
- 偏观察、解释、提醒

### 决策型 Agent
- 低频
- 高风险
- 偏比较、筛选、模拟、受控建议

---

## 16. 与 InterventionSimulator 的关系

决策型 Agent 不是模拟器本身，但通常会调用：

- `SIM-002 InterventionSimulator`

可理解为：

- 模拟器负责跑动作比较
- 决策型 Agent 负责组织输入、调用模拟、整理输出、附加边界与角色控制

---

## 17. 与后台影子推演机制的关系

决策型 Agent 在项目当前阶段，建议主要运行在：

- `SIM-006 后台影子推演机制`

原因：

- 高风险
- 需要校准
- 需要复盘
- 不适合一开始大范围前台化

---

## 18. 输出风险控制建议

决策型 Agent 输出必须至少带：

- 不确定性说明
- 数据充分度说明
- 动作可执行性说明
- 角色边界说明
- 是否需人工复核标记

### 原则
不能只给"结论"，必须同时给"边界"。

---

## 19. 人工复核触发规则

建议以下情况必须或优先进入人工复核：

- 高风险学生路径选择
- 规则边界复杂
- 数据不充分
- 推演动作差异不明显
- 推演结果波动过大
- 涉及升学关键场景

---

## 20. 工程实现建议

建议决策型 Agent 调用时统一带入：

- `task_id`
- `scenario_type`
- `target_scope`
- `candidate_actions`
- `current_state_ref`
- `context_bundle_ref`
- `rule_pack_ref`
- `output_risk_policy`

---

## 21. 质量指标建议

建议监控：

- 决策型任务触发量
- 人工复核比例
- 动作比较一致率
- 真实执行覆盖率
- 建议撤回率
- 决策误导投诉率
- 后台到前台开放转化率

---

## 22. MVP 建议范围

MVP 阶段建议决策型 Agent 仅支持：

- 后台单学生动作比较
- 低到中风险场景
- 影子推演输出整理
- 不确定性说明
- 人工复核建议
- 不直接前台裸开放

---

## 25. 结论

决策型 Agent 的核心价值，不是让系统更像一个"替人拍板的智能体"，而是让系统在复杂教育选择场景中，能够更结构化、更克制、更可审计地支持动作比较和路径判断。

它必须坚持：

- 高约束
- 比较优先
- 风险显式
- 人工复核优先
- 先后台成熟再前台开放

否则决策型 Agent 很容易在能力尚未成熟时，变成高风险输出放大器。
