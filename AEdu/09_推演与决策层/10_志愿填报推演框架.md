## 1. 文档目的

本文档用于定义教育孪生项目中"志愿填报推演框架"的整体结构，包括目标、适用阶段、输入输出、规则边界、推演逻辑与风险控制机制。

本文档重点回答以下问题：

1. 志愿填报推演在教育孪生项目中处于什么位置
2. 它依赖哪些输入
3. 它输出什么，不输出什么
4. 它如何与 StudentTwinAgent、规则数据、后台推演层衔接
5. 如何避免把志愿推演做成自信满满但边界模糊的高风险工具

---

## 2. 志愿填报推演定义

志愿填报推演，是后台推演层中面向升学决策场景的专门推演框架，用于在明确地区规则、考试制度、学校专业数据和学生阶段状态的前提下，对不同志愿方案的适配度、风险层级和选择空间进行结构化评估。

它不是自动替代家长、学生和老师完成志愿填报，而是一个：

- 以规则为底
- 以学生状态与阶段表现为参照
- 以学校与专业数据为目标空间
- 以方案比较与风险提示为输出

的决策支持框架。

---

## 3. 目标

### 3.1 形成志愿方案比较能力
支持对多个志愿方案进行结构化对比，而不是只输出单一答案。

### 3.2 显示规则约束
让系统在不同地区、不同批次、不同选科要求下有明确边界。

### 3.3 支持风险提示
帮助识别"过冲""扎堆""条件不满足""匹配度偏低"等风险。

### 3.4 支持阶段式使用
既可用于正式填报阶段，也可用于提前规划与路径讨论。

### 3.5 保持解释能力
推演结果必须能说明为什么这样评估，而不是神秘打分。

---

## 4. 适用阶段

志愿填报推演建议分阶段建设：

### 4.1 提前规划阶段
用于高一/高二阶段的方向性规划。

**重点**：
- 学科与方向匹配
- 选科要求认知
- 长期路径约束理解

### 4.2 模拟填报阶段
用于高三阶段的模拟方案比较。

**重点**：
- 方案空间
- 风险比较
- 学校/专业层级分布

### 4.3 正式填报辅助阶段
用于临近正式填报时的支持性评估。

**重点**：
- 批次与规则校验
- 冲稳保分层
- 风险提示与解释

---

## 5. 志愿推演不做什么

本框架原则上不直接做以下事情：

- 不直接替代最终人工填报决策
- 不承诺"必录取"
- 不输出脱离规则边界的推荐
- 不输出高风险"唯一正确答案"
- 不把学生长期人生目标压成一个模板化分数

否则它会很快从"辅助系统"滑成"背锅系统"。

---

## 6. 核心输入

志愿填报推演至少依赖以下六类输入：

1. 学生阶段状态输入
2. 考试与成绩输入
3. 地区规则输入
4. 学校与专业数据输入
5. 志愿方案输入
6. 风险控制规则输入

---

## 7. 学生阶段状态输入

来自 StudentTwinAgent 和阶段性记录：

| 输入项 | 说明 |
|--------|------|
| 当前学段与年级 | 高一/高二/高三 |
| 学科状态摘要 | 各学科当前水平 |
| 选科信息 | 已选/意向选科组合 |
| 阶段性稳定性 | 成绩波动情况 |
| 当前风险因素 | 影响升学的风险项 |
| 数据完整性评分 | 判断参考可信度 |

说明：
- 志愿推演不一定直接吃全套学生模型
- 但至少要基于真实阶段状态，不是凭空飘着推

---

## 8. 考试与成绩输入

建议包括：

| 输入项 | 说明 |
|--------|------|
| 阶段考试结果 | 模考、月考、联考成绩 |
| 排名信息 | 校排名/区排名/市排名 |
| 历史波动情况 | 成绩变化趋势 |
| 学科稳定性 | 各学科波动程度 |
| 成绩变化方向 | 上升/平稳/下降 |

说明：
- 不能只看一次分数
- 趋势与波动同样重要

---

## 9. 地区规则输入

志愿推演必须强依赖地区规则层。

建议至少包括：

| 规则项 | 说明 |
|--------|------|
| 地区高考规则 | 高考模式、批次设置 |
| 批次规则 | 各批次录取顺序与规则 |
| 选科要求 | 不同专业对选科的要求 |
| 专业限制规则 | 身体条件、单科成绩等限制 |
| 志愿填报口径 | 可填志愿数量、平行志愿规则 |
| 录取规则说明 | 投档比例、专业分配规则 |

说明：
没有规则输入的志愿推演，基本等于开盲盒。

---

## 10. 学校与专业数据输入

建议包括：

| 数据项 | 说明 |
|--------|------|
| 学校基本信息 | 学校名称、层次、地域 |
| 专业基本信息 | 专业名称、类别、方向 |
| 历史录取数据 | 近年录取分数、位次 |
| 专业选科要求 | 各专业的选科限制 |
| 学校层级与标签 | 985/211/双一流等 |
| 专业方向标签 | 学科门类、就业方向 |
| 近年变化说明 | 招生计划变化、新增/撤销专业 |

---

## 11. 志愿方案输入

志愿填报推演应支持对"方案集"进行比较，而不只接受一个目标学校。

建议志愿方案输入包括：

| 输入项 | 说明 |
|--------|------|
| 学校列表 | 目标学校集合 |
| 专业列表 | 目标专业集合 |
| 顺序结构 | 志愿排序 |
| 冲稳保标签 | 冲刺/稳定/保底分类 |
| 用户偏好说明 | 地域/专业/学校优先级 |

---

志愿填报推演建议输出以下结构：

| 字段 | 说明 |

## 13. 方案评估维度

建议至少从以下几个维度评估志愿方案：

1. **规则匹配度** - 是否符合地区规则和选科要求
2. **条件满足度** - 成绩/位次是否达到参考线
3. **风险等级** - 冲刺/稳定/保底的合理性
4. **稳定性参考** - 历史数据参考价值
5. **结构平衡度** - 冲稳保分布是否合理
6. **选择空间合理性** - 是否有足够备选空间

---

## 14. 规则匹配度

### 14.1 定义
方案是否符合当前地区、批次、选科和专业规则。

### 14.2 典型问题
- 选科要求不满足
- 批次不匹配
- 学校/专业规则不适配
- 方案存在无效项

### 14.3 原则
规则不满足的项应明确标红或剔除，不要假装它们仍然可选。

---

## 15. 风险等级评估

建议志愿方案风险至少分为：

| 等级 | 含义 | 处理方式 |
|------|------|----------|
| low | 低风险/保底 | 录取概率高，建议保留 |
| medium | 中风险/稳定 | 录取概率中等，可作为主力 |
| high | 高风险/冲刺 | 录取概率低，需谨慎配置 |
| invalid | 规则无效 | 不满足基本条件，应剔除 |

### 15.1 风险示例
- **过冲风险** - 成绩/位次远低于历史录取线
- **扎堆风险** - 多所热门学校集中，竞争激烈
- **结构失衡风险** - 冲稳保比例不合理
- **规则无效风险** - 选科/条件不满足

---

## 16. 结构平衡度评估

志愿方案不应只看单点，应看整体组合结构。

建议考虑：

| 维度 | 说明 |
|------|------|
| 冲稳保分布 | 冲刺/稳定/保底的比例是否合理 |
| 学校层级分布 | 不同层次学校的分布是否均衡 |
| 专业方向分布 | 专业方向是否过于集中或分散 |
| 地域分布 | 地域选择是否合理 |
| 风险集中度 | 风险是否过度集中在某一批次 |

---

## 17. 解释机制

志愿推演必须提供解释，而不是只给打分。

建议解释使用以下结构：

1. **规则说明** - 哪些规则被满足/不满足
2. **方案问题点** - 方案存在哪些具体问题
3. **风险来源** - 风险来自哪里
4. **可调整方向** - 如何优化方案

### 17.1 示例

> 当前方案中有 2 个专业存在选科要求不满足，因此不建议继续保留。
> 
> 整体方案偏向高风险冲刺，稳定层和保底层较弱，建议补充中间梯度方案。
> 
> 建议将冲刺院校控制在 2-3 所，增加稳定层院校数量。

## 18. 与 StudentTwinAgent 的关系

StudentTwinAgent 在志愿推演中主要提供：

- 学生当前阶段状态
- 学科结构特征
- 稳定性与波动信息
- 当前约束因素
- 数据完整性和可信度参考

说明：
志愿填报推演不是 StudentTwinAgent 的直接延伸输出，而是建立在其状态基础上的专业场景推演。

---

## 19. 与后台推演层的关系

志愿填报推演是后台推演层中的专门子框架。

与一般干预推演不同，它更强调：

- 外部规则（地区政策、录取规则）
- 学校专业空间（目标范围）
- 方案组合（多方案比较）
- 风险分层（冲稳保结构）
- 人工复核（最终决策需人工确认）

---

## 20. 与规则与外部数据层的关系

本框架强依赖 `10_规则与外部数据` 中的文档：

- 地区考试规则
- 高考历史数据
- 学校与专业数据
- 政策与口径说明

这意味着：
**志愿推演框架再聪明，没有这些数据也只是空心木偶。**

---

## 21. 风险控制机制

### 21.1 规则优先
规则冲突先处理，再谈匹配和风险。

### 21.2 数据完整性提示
若历史数据或规则数据不完整，应明确提示。

### 21.3 人工复核优先
高风险、正式填报阶段、关键结果必须支持人工复核。

### 21.4 不直接面向家长裸输出高风险结果
前台只可消费经解释和筛选后的结果。

### 21.5 输出免责声明
所有志愿推演结果应附带说明：
- 仅供参考，不替代最终决策
- 不承诺录取结果
- 需结合当年实际招生政策

```
读取学生阶段状态
    ↓
读取地区规则与批次设置
    ↓
读取学校与专业数据
    ↓
读取志愿方案输入
    ↓
规则匹配校验
    ↓
成绩/位次匹配评估
    ↓
风险等级判定
    ↓
结构平衡度评估
    ↓
生成方案评估结果
    ↓
输出解释与风险提示
    ↓
写入推演记录
```

MVP 阶段建议先不做完整正式志愿填报系统，而优先做：

### 23.1 必须支持

| 功能 | 说明 |
|------|------|
| 规则匹配校验 | 选科/批次/条件校验 |
| 方案结构对比 | 多方案结构化比较 |
| 风险分层提示 | 冲稳保风险等级 |
| 基础学校/专业数据引用 | 基础数据支撑 |

### 23.2 可以后置

| 功能 | 说明 |
|------|------|
| 完整自动推荐 | 智能推荐算法 |
| 多轮智能问答式填报 | 交互式填报引导 |
| 复杂偏好优化 | 多维度偏好匹配 |

## 26. 结论

志愿填报推演框架的核心价值，不是替学生做选择，而是让不同方案在规则、风险和结构层面被更清楚地比较出来。

它必须坚持：

- **先规则** - 规则不满足的方案直接剔除
- **再结构** - 冲稳保分布是否合理
- **再风险** - 各方案风险等级是否清晰
- **最后才是建议** - 基于以上分析给出优化建议

否则它会从辅助框架迅速滑成高风险意见制造机。
