> 文档编号：STD-008  
> 版本：V1.0  
> 创建日期：2024  
> 最后更新：待定  
> 维护人：教材标准表负责人

---

## 1. 文档目的

本文档用于定义教育孪生项目中的"实验活动表"标准，包括实验活动的定义、分类、字段结构、与知识点/能力点/锚点的映射方式及其在系统中的使用场景。

本文档重点回答以下问题：

1. 什么叫实验活动
2. 实验活动为什么要单独建表
3. 实验活动如何与教材、知识点、能力点和学生状态连接
4. 实验活动如何服务前台解释、学习事件理解和推演层
5. 如何避免把实验活动表做成"实验名称清单"而没有结构价值

---

## 2. 实验活动定义

实验活动，是指教材中围绕某一知识主题组织的、具有明确观察对象、操作过程、实验现象、结论目标或探究任务的结构化教学活动单元。

实验活动既包括狭义实验，也包括：

- 演示实验
- 学生分组实验
- 探究活动
- 观察活动
- 数据记录与分析活动
- 基于教材设置的操作性任务

实验活动不是单一知识点，也不是单一题型，而是连接知识理解、能力形成和行为观察的重要中间层。

---

## 3. 建设目标

### 3.1 支撑实验类学科的结构化表达
让物理、化学、生物等学科中的实验内容能被系统识别与引用。

### 3.2 支撑能力点建模
实验活动通常直接关联观察、分析、建模、误差判断等能力。

### 3.3 支撑学习事件理解
学生在实验活动中的表现、反馈、记录和错误，应能映射到具体实验活动。

### 3.4 支撑解释层与报告层
前台和报告层可引用"实验活动"作为更贴近教学实际的解释单位。

### 3.5 支撑后续推演与干预
系统可基于实验活动判断：学生的问题更偏知识理解、实验操作还是分析能力。

---

## 4. 设计原则

### 4.1 活动应具备独立性
一条实验活动记录应能独立描述一个完整活动单元，而不是零散步骤片段。

### 4.2 活动应可映射
实验活动必须可映射到知识点、能力点、教材锚点和题型。

### 4.3 活动应可观察
实验活动应支持通过作业、实验记录、老师反馈、课堂表现等观察学生表现。

### 4.4 活动应可解释
老师看到一条实验活动记录，应能明白它到底做什么、考什么、训练什么。

### 4.5 粒度适中
不能粗到整章只有一个"实验"，也不能细到每一个操作动作都独立成表项。

---

## 5. 实验活动分类建议

建议至少分为以下五类：

| 类型 | 编码 | 说明 |
|------|------|------|
| 演示实验 | demo | 由老师主导演示、学生观察理解 |
| 学生操作实验 | hands_on | 学生直接参与操作、记录和处理结果 |
| 探究活动 | inquiry | 围绕问题提出、假设、验证的探究式活动 |
| 观察记录活动 | observation | 以观察、描述、记录现象为主的活动 |
| 数据分析活动 | data_analysis | 以读取实验结果、分析数据、判断误差为主的活动 |

---

## 6. 演示实验

### 6.1 定义
由老师主导演示、学生观察和理解的实验活动。

### 6.2 特点
- 更偏现象理解
- 学生操作参与度可能较低
- 更适合观察概念建立和现象解释

### 6.3 物理示例
| 实验名称 | 说明 |
|----------|------|
| 演示小车受力后的运动变化 | 观察力与运动关系 |
| 演示电流磁场效应 | 观察电流产生磁场现象 |
| 演示光的折射现象 | 观察光线折射规律 |

---

## 7. 学生操作实验

### 7.1 定义
学生需要直接参与操作、记录和处理结果的实验活动。

### 7.2 特点
- 操作行为更多
- 更容易暴露步骤错误和实验能力短板
- 可形成更丰富的事件记录

### 7.3 物理示例
| 实验名称 | 说明 |
|----------|------|
| 测量加速度实验 | 使用打点计时器测量加速度 |
| 验证牛顿第二定律 | 探究力、质量与加速度关系 |
| 测定某物理量的变化关系 | 记录数据并分析变化规律 |

---

## 8. 探究活动

### 8.1 定义
围绕问题提出、假设、设计、验证和结论形成的探究式活动。

### 8.2 特点
- 更强调过程和思维
- 常涉及建模和分析能力
- 比常规实验更开放

### 8.3 物理示例
| 实验名称 | 说明 |
|----------|------|
| 探究影响摩擦力大小的因素 | 设计实验验证影响因素 |
| 探究电阻大小规律 | 探究电阻与材料、长度等关系 |
| 探究某类运动与受力关系 | 建立运动与力的关系理解 |

---

## 9. 观察记录活动

### 9.1 定义
以观察、描述、记录现象为主的活动。

### 9.2 特点
- 对实验设计要求较低
- 对观察与描述能力要求较高
- 常见于实验前置或简化场景

### 9.3 示例
| 活动名称 | 说明 |
|----------|------|
| 观察物体运动轨迹 | 记录运动路径和特征 |
| 记录实验现象变化 | 描述实验过程中的现象 |

---

## 10. 数据分析活动

### 10.1 定义
以读取实验结果、分析数据、判断误差和形成结论为主的活动。

### 10.2 特点
- 更靠近能力与题型训练
- 往往与实验分析题强关联
- 是连接实验活动和考试表现的重要桥梁

### 10.3 示例
| 活动名称 | 说明 |
|----------|------|
| 实验数据处理与分析 | 对实验数据进行处理和解释 |
| 误差分析与结论判断 | 分析误差来源并得出结论 |

---

## 11. 标准字段定义

实验活动表建议至少包含以下字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| experiment_id | string | 是 | 实验活动唯一标识 |
| subject | string | 是 | 学科 |
| curriculum_version_id | string | 否 | 教材版本 |
| experiment_name | string | 是 | 实验活动名称 |
| experiment_type | string | 是 | 实验活动类型 |
| chapter_refs | array[string] | 否 | 关联章节 |
| knowledge_refs | array[string] | 否 | 关联知识点 |
| ability_refs | array[string] | 否 | 关联能力点 |
| anchor_refs | array[string] | 否 | 关联教材锚点 |
| objective | string | 是 | 活动目标 |
| setup_summary | string/null | 否 | 实验装置/环境摘要 |
| procedure_summary | string/null | 否 | 操作过程摘要 |
| expected_observations | array[string] | 否 | 预期现象 |
| expected_outputs | array[string] | 否 | 预期结果/结论 |
| common_errors | array[string] | 否 | 常见错误 |
| misconception_refs | array[string] | 否 | 关联误区 |
| exercise_type_refs | array[string] | 否 | 关联题型 |
| observable_signals | array[string] | 否 | 可观察信号 |
| importance_level | string | 是 | 重要程度 |
| status | string | 是 | active/deprecated |
| source_ref | string/null | 否 | 来源引用 |
| version | string | 是 | 版本号 |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 更新时间 |

---

## 12. 字段说明

### 12.1 experiment_id
实验活动唯一主键。

建议格式：`PHY_EXP_001`

编码结构：`学科_EXP_顺序号`

### 12.2 experiment_type
实验活动类型编码。

建议值：
- demo（演示实验）
- hands_on（学生操作实验）
- inquiry（探究活动）
- observation（观察记录活动）
- data_analysis（数据分析活动）

### 12.3 objective
用于说明该实验活动希望学生理解或完成什么。

示例：
- "通过实验探究加速度与作用力、质量之间的关系"
- "观察并记录物体自由落体运动的特征"

### 12.4 expected_observations
说明实验中应看到的关键现象。

示例：
- "质量一定时，加速度随力增大而增大"
- "力一定时，加速度随质量增大而减小"

### 12.5 expected_outputs
说明实验后应形成的结果、判断或结论。

示例：
- "建立加速度与力、质量的关系理解"
- "形成实验数据与规律对应关系"

### 12.6 common_errors
用于记录典型实验性错误，不替代误区表，但可作快速摘要。

示例：
- "未控制变量"
- "数据记录不规范"
- "结论与数据不一致"

### 12.7 observable_signals
用于说明系统如何观察学生在该实验活动中的表现。

示例：
- "变量控制混乱"
- "实验数据解读不稳定"
- "结论表达逻辑断裂"

---

## 13. 命名规范

### 13.1 命名原则
- 与教材表述尽量一致
- 能体现活动核心目标
- 简明清楚
- 不用口号化命名

### 13.2 推荐写法
| 推荐 | 说明 |
|------|------|
| 探究加速度与力、质量的关系 | 清楚说明探究内容 |
| 受力与运动变化演示实验 | 清楚说明实验类型 |
| 实验数据误差分析活动 | 清楚说明活动目标 |

### 13.3 不推荐写法
| 不推荐 | 问题 |
|--------|------|
| 物理实验一 | 太笼统，无法识别 |
| 课堂活动 A | 无实际意义 |
| 重点实验 | 主观判断，无指导价值 |
| 综合实践任务 | 太宽泛，无法定位 |

## 14. 抽取流程建议

```text
读取教材实验与活动内容
    ↓
识别实验活动边界
    ↓
抽取活动目标
    ↓
抽取过程与现象摘要
    ↓
挂载章节/知识点/能力点
    ↓
挂载锚点/题型/误区
    ↓
补充可观察信号
    ↓
质量校验
```

### 14.1 各阶段说明

| 阶段 | 工作内容 | 输出 |
|------|----------|------|
| 读取教材内容 | 识别教材中的实验和活动 | 实验活动清单 |
| 识别边界 | 确定每个实验活动的范围 | 活动边界定义 |
| 抽取目标 | 提取活动目标 | objective 字段 |
| 抽取摘要 | 提取过程和现象摘要 | setup/procedure 字段 |
| 挂载映射 | 关联知识点、能力点等 | 映射关系 |
| 补充信号 | 补充可观察信号 | observable_signals |
| 质量校验 | 按质量要求校验 | 校验报告 |

## 15. 与知识点表的关系

实验活动通常不止映射一个知识点。

建议规则：

| 规则 | 说明 |
|------|------|
| 一个实验活动可关联多个知识点 | 实验活动通常涉及多个知识点 |
| 每个知识点可关联多个实验活动 | 同一知识点可通过多个实验活动训练 |
| 高价值知识点应尽量有实验或活动支撑引用 | 增强知识点的可观察性 |

---

## 16. 与能力点表的关系

实验活动与能力点关系通常更强。

常见关联包括：

| 实验活动 | 能力点 |
|----------|--------|
| 实验操作类 | 实验操作能力 |
| 数据分析类 | 实验分析能力 |
| 观察记录类 | 观察与记录能力 |
| 探究活动类 | 建模能力、探究能力 |

因此实验活动表是能力点表的重要落地层。

---

## 17. 与教材锚点表的关系

实验活动必须尽量引用教材锚点，以支持：

| 支持项 | 说明 |
|--------|------|
| 报告层引用教材出处 | 可追溯实验在教材中的位置 |
| 老师快速定位教材位置 | 方便教学参考 |
| OCR 回贴 | 扫描件与实验活动关联 |
| 人工校验 | 便于人工核对 |

建议：
- 每个高价值实验活动至少有一个主锚点

---

## 18. 与题型映射表的关系

实验活动可映射到实验题型或数据分析题型。

| 实验活动 | 题型映射 |
|----------|----------|
| 验证牛顿第二定律实验 | 实验数据处理题 |
| 图像实验观察活动 | 实验图像判断题 |
| 误差分析活动 | 实验误差分析题 |

这样可打通：
```
实验活动 → 题型 → 学生表现 → 状态解释
```

---

## 19. 与学生状态模型的关系

实验活动后续可支撑以下状态：

| 状态类型 | 说明 |
|----------|------|
| 实验相关能力状态 | 实验能力掌握情况 |
| 某实验活动表现不稳定 | 特定实验活动的问题 |
| 数据分析型问题集中 | 数据分析能力短板 |
| 实验误区高频出现 | 实验类错误模式 |

它能让 StudentTwinAgent 对实验型学科的理解更接近真实课堂场景。

## 20. 质量要求

### 20.1 完整性

| 要求 | 说明 |
|------|------|
| 活动名称、目标、类型和主要映射关系应完整 | 核心字段不能缺失 |

### 20.2 一致性

| 要求 | 说明 |
|------|------|
| 类型、命名和摘要风格应统一 | 避免风格混乱 |

### 20.3 可观察性

| 要求 | 说明 |
|------|------|
| 活动应支持通过事件或反馈观察 | 有 observable_signals 支撑 |

### 20.4 可映射性

| 要求 | 说明 |
|------|------|
| 应能挂载知识点、能力点、锚点、题型等对象 | 映射关系完整 |

### 20.5 可解释性

| 要求 | 说明 |
|------|------|
| 老师和系统都应能读懂这项活动的价值 | 目标清楚、说明清晰 |

## 21. 常见错误

### 21.1 只有活动名称没有目标

| 错误 | 后果 |
|------|------|
| 只写实验名称，不写活动目标 | 后面系统很难用 |

### 21.2 把活动步骤全文搬进表里

| 错误 | 后果 |
|------|------|
| 实验活动表变成教案全文库 | 表结构臃肿，难以维护 |

### 21.3 不挂知识点和能力点

| 错误 | 后果 |
|------|------|
| 实验活动成为孤立对象 | 无法参与状态建模和解释 |

### 21.4 把实验活动和题型混为一谈

| 错误 | 后果 |
|------|------|
| 实验活动是教学活动，题型是考查结构 | 概念混淆导致映射错误 |

### 21.5 实验类型标注不准确

| 错误 | 后果 |
|------|------|
| 演示实验标成学生操作实验 | 影响能力点映射和状态解释 |

## 22. MVP 建议范围

MVP 阶段建议优先支持：

### 22.1 学科范围

| 范围 | 说明 |
|------|------|
| 高中物理 | 首建学科 |

### 22.2 内容范围

| 内容 | 说明 |
|------|------|
| 高中物理核心实验活动 | 优先覆盖高价值实验 |
| 与知识点、能力点、锚点的基础映射 | 支撑事件解释 |
| 与实验分析题型的基础关联 | 支撑题型映射 |
| 高价值实验活动的误区与可观察信号 | 支撑状态解释 |

### 22.3 建议优先覆盖

| 实验活动类型 | 具体实验 |
|--------------|----------|
| 受力与运动类实验活动 | 牛顿定律相关实验 |
| 图像与数据分析类实验活动 | 数据处理和图像分析 |
| 探究活动 | 探究类实验活动 |

| 风险 | 说明 |
|------|------|
| 只有名称没有目标 | 实验活动表失去指导价值 |

### 26.2 不能忽视映射关系

| 风险 | 说明 |
|------|------|

### 26.3 不能忽视可观察性

| 风险 | 说明 |
|------|------|

### 26.4 不能忽视学科差异

|------|------|

|------|------|
| 实验类型标注不准确 | 影响能力点映射和状态解释 |

## 27. 结论

实验活动表的核心价值，是把教材中的实验和探究内容从"课本附属部分"变成系统可识别、可解释、可观察、可用于状态建模的重要对象层。

它必须坚持：

| 原则 | 说明 |
|------|------|
| **活动边界清楚** | 每个实验活动是独立完整单元 |
| **目标清楚** | 活动目标明确可理解 |
| **映射清楚** | 与知识点、能力点、锚点映射完整 |
| **观察点清楚** | 有 observable_signals 支撑 |
| **能服务解释和干预** | 支撑学生状态和干预动作设计 |

否则实验活动最终只会在教材里存在，在系统里失踪。
