# 地区教材适配表标准

> 文档编号：STD-011  
> 版本：V1.0  
> 创建日期：2024  
> 最后更新：待定  
> 维护人：教材标准表负责人

---

## 1. 文档目的

本文档用于定义教育孪生项目中的"地区教材适配表"标准，包括地区与教材版本的绑定方式、适配范围、约束信息、例外规则与系统使用场景。

本文档重点回答以下问题：

1. 为什么地区和教材之间必须有一张适配表
2. 地区教材适配关系应如何表达
3. 一个地区使用多版本教材时如何处理
4. 学生初始化和学校接入如何依赖这张表
5. 如何避免地区绑定靠人工口头判断，最后全系统口径漂移

---

## 2. 地区教材适配定义

地区教材适配，是指某个地区在特定学段、年级、学科和时间范围内所采用的教材版本及其约束关系。

这张表的任务是回答：

- 某地区当前某学科该用哪套教材版本
- 同一地区不同年级是否使用不同教材
- 是否存在多个可选版本
- 是否有特殊学校、特殊学段、过渡期情况
- 学生和学校接入时该绑定哪个教材版本

地区教材适配不是教材版本表的附加备注，而是地区规则层与教材资产层之间的正式桥梁。

---

## 3. 建设目标

### 3.1 支撑学生初始化绑定
确保学生创建时能正确绑定到对应教材版本。

### 3.2 支撑学校接入
学校和班级初始化时可自动推荐或校验教材版本。

### 3.3 支撑知识底座适配
让章节树、知识点、锚点等结构能被地区场景正确使用。

### 3.4 支撑版本切换与过渡
教材版本变化时，系统能识别新旧关系。

### 3.5 支撑后续版本映射
为跨地区、跨版本比较和迁移提供前提。

---

## 4. 设计原则

### 4.1 地区粒度明确
必须明确适配是省级、市级、区级，还是学校特殊例外。

### 4.2 时间边界明确
适配关系必须有生效时间和失效时间。

### 4.3 学段和年级明确
同一地区不同年级可能使用不同版本，不能省略。

### 4.4 多版本情况显式标注
不能默认每个地区同学科只有一个教材版本。

### 4.5 例外情况显式处理
特殊学校、过渡期班级、校本替代情况要明确标记。

---

## 5. 适配对象范围

地区教材适配表建议至少覆盖以下维度：

| 维度 | 说明 |
|------|------|
| 地区 | 省/市/区/县 |
| 学段 | 小学/初中/高中 |
| 年级 | 具体年级 |
| 学科 | 物理/数学/语文等 |
| 教材版本 | curriculum_version_id |
| 生效时间 | valid_from / valid_to |
| 适配状态 | standard/optional/transition/special |
| 例外说明 | 特殊学校或过渡期说明 |

---

## 6. 标准字段定义

地区教材适配表建议至少包含以下字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| region_curriculum_mapping_id | string | 是 | 适配记录唯一标识 |
| region_id | string | 是 | 地区标识 |
| region_level | string | 是 | 地区层级 |
| stage_level | string | 是 | 学段 |
| grade_level | string | 是 | 年级 |
| subject | string | 是 | 学科 |
| curriculum_version_id | string | 是 | 教材版本标识 |
| textbook_id | string | 否 | 具体教材资产标识 |
| mapping_type | string | 是 | 适配类型 |
| priority | int | 是 | 优先级 |
| valid_from | date | 否 | 生效开始时间 |
| valid_to | date/null | 否 | 生效结束时间 |
| default_flag | boolean | 是 | 是否默认版本 |
| exception_scope | string/null | 否 | 例外范围说明 |
| source_ref | string | 是 | 来源引用 |
| status | string | 是 | active/deprecated/draft |
| version | string | 是 | 版本号 |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 更新时间 |

---

## 7. 字段说明

### 7.1 region_curriculum_mapping_id

适配记录唯一主键。

建议格式：`RCM_GD_SZ_HS_PHY_001`

编码结构：`RCM_地区_学段_学科_顺序号`

### 7.2 region_id

地区唯一标识。

建议格式：`CN_GD_SZ`（中国 - 广东 - 深圳）

### 7.3 region_level

地区层级。

| 值 | 说明 |
|------|------|
| province | 省级 |
| city | 市级 |
| district | 区县级 |
| school | 学校级（例外情况） |

### 7.4 mapping_type

适配类型。

| 值 | 说明 | 使用场景 |
|------|------|----------|
| standard | 标准适配 | 地区统一使用的默认版本 |
| optional | 可选适配 | 存在多个可选版本 |
| transition | 过渡期适配 | 新旧版本并行过渡期 |
| special_case | 特殊案例 | 特殊学校或特殊区域 |

### 7.5 priority

用于同一区域同学科存在多条记录时的优先级判断。

建议：
- 数字越大优先级越高
- default_flag = true 的记录 priority 应最高

### 7.6 default_flag

用于标记默认推荐版本。

| 值 | 说明 |
|------|------|
| true | 默认版本，学生初始化时优先使用 |
| false | 非默认版本，需特殊指定 |

**原则**：同一地区 - 学段 - 年级 - 学科组合下，default_flag = true 的记录只能有一条。

### 7.7 exception_scope

用于说明特定学校、班级或过渡期例外。

示例：
- "深圳市外国语学校使用人教版，其他学校使用粤教版"
- "2024 级高一使用新教材，2023 级继续使用旧教材"
- "XX 区试点学校使用实验教材"

## 8. 地区粒度规则

### 8.1 省级适配

适用于省级统一口径较强的场景。

| 特点 | 说明 |
|------|------|
| 适用范围 | 全省统一 |
| 优先级 | 低（被市级覆盖） |
| 使用场景 | 省级统考、全省统一教材 |

### 8.2 市级适配

适用于市级存在明确教材差异的场景。

| 特点 | 说明 |
|------|------|
| 适用范围 | 全市统一 |
| 优先级 | 中（覆盖省级，被区级覆盖） |
| 使用场景 | 多数地级市教材统一场景 |

### 8.3 区县级适配

适用于局部特殊区域使用不同版本的情况。

| 特点 | 说明 |
|------|------|
| 适用范围 | 区县范围内 |
| 优先级 | 高（覆盖市级） |
| 使用场景 | 特区、开发区、特殊政策区 |

### 8.4 学校级例外

不建议默认用学校级作为常规适配层，但应支持特殊学校例外记录。

| 特点 | 说明 |
|------|------|
| 适用范围 | 单个学校 |
| 优先级 | 最高 |
| 使用场景 | 外国语学校、国际学校、试点学校 |

---

## 9. 时间边界规则

### 9.1 valid_from

表示该适配关系何时开始有效。

| 要求 | 说明 |
|------|------|
| 必填 | 适配记录必须有生效时间 |
| 格式 | YYYY-MM-DD |
| 示例 | 2024-09-01（秋季学期开始） |

### 9.2 valid_to

表示该适配关系何时结束。

| 要求 | 说明 |
|------|------|
| 可选 | 长期有效的适配可为空 |
| 格式 | YYYY-MM-DD |
| 使用场景 | 过渡期适配、临时适配 |

### 9.3 原则

地区教材适配没有时间边界，后续版本切换就会变成口耳相传的神秘传统。

---

## 10. 多版本适配规则

### 10.1 场景

同一地区同学科可能存在：

| 场景 | 说明 |
|------|------|
| 默认版本 | 大多数学校使用的版本 |
| 特定学校版本 | 特殊学校使用不同版本 |
| 新旧版本并行过渡 | 版本切换过渡期 |
| 选修或补充版本 | 校本课程或拓展教材 |

### 10.2 表达方式

| 字段 | 用途 |
|------|------|
| priority | 控制优先级 |
| default_flag | 标记默认版本 |
| mapping_type | 标记适配类型 |
| exception_scope | 说明差异范围 |

### 10.3 示例场景

**场景一：新旧版本过渡期**

```json
{
  "region_curriculum_mapping_id": "RCM_GD_SZ_HS_PHY_001",
  "region_id": "CN_GD_SZ",
  "region_level": "city",
  "stage_level": "高中",
  "grade_level": "高一",
  "subject": "物理",
  "curriculum_version_id": "PHY_PEP_G1_V2",
  "mapping_type": "transition",
  "priority": 10,
  "valid_from": "2024-09-01",
  "valid_to": "2025-08-31",
  "default_flag": true,
  "exception_scope": "2024 级高一使用新教材 V2，2023 级继续使用 V1"
}
```

**场景二：特殊学校例外**

```json
{
  "region_curriculum_mapping_id": "RCM_GD_SZ_HS_PHY_002",
  "region_id": "CN_GD_SZ",
  "region_level": "city",
  "stage_level": "高中",
  "grade_level": "高一",
  "subject": "物理",
  "curriculum_version_id": "PHY_PEP_G1_V1",
  "mapping_type": "special_case",
  "priority": 5,
  "valid_from": "2024-09-01",
  "valid_to": null,
  "default_flag": false,
  "exception_scope": "深圳市外国语学校使用人教版，其他学校使用粤教版"
}
```

## 11. 学生初始化使用方式

学生创建时建议按以下顺序绑定教材版本：

```text
读取学生地区
    ↓
读取学段/年级/学科
    ↓
查询地区教材适配表
    ↓
优先取 active + default_flag = true 的记录
    ↓
如存在学校级例外，再走特殊适配逻辑
    ↓
绑定 curriculum_version_id
```

### 11.1 查询优先级

| 优先级 | 查询条件 | 说明 |
|--------|----------|------|
| 1 | 学校级例外 | 最高优先级 |
| 2 | 区县级适配 | 覆盖市级 |
| 3 | 市级适配 | 覆盖省级 |
| 4 | 省级适配 | 最低优先级 |

### 11.2 冲突处理

当存在多条匹配记录时：

1. 按 region_level 优先级排序
2. 同级别按 priority 排序
3. priority 相同时 default_flag = true 优先
4. 仍无法确定时进入人工确认流程

## 12. 学校接入使用方式

学校接入时建议：

1. **先读取地区教材适配表生成默认建议**
   - 根据学校所在地区自动推荐教材版本
   
2. **再由学校确认是否存在例外**
   - 学校可确认或修改推荐版本
   
3. **若学校使用特殊版本，必须单独记录 exception_scope**
   - 记录到学校级例外记录中
   - 补充例外说明

这能避免接入时全靠群消息和老师记忆来确认教材版本。

## 13. 与教材元数据表的关系

地区教材适配表引用的是：

| 字段 | 说明 |
|------|------|
| curriculum_version_id | 教材版本标识（必填） |
| textbook_id | 具体教材资产标识（可选） |

因此它必须建立在教材元数据表之上，而不是直接写死教材名字。

### 13.1 引用规则

| 规则 | 说明 |
|------|------|
| curriculum_version_id 必须存在于教材元数据表 | 版本必须已注册 |
| textbook_id 可选，用于指定具体册别 | 细化到具体教材 |
| 版本状态必须为 active | 废弃版本不可用 |

## 14. 与版本映射规则的关系

| 表 | 回答的问题 |
|------|----------|
| 地区教材适配表 | "这个地区此时该用哪套教材" |
| 教材版本映射规则 | "不同教材版本之间如何建立对应" |

两者关系紧密，但不是同一张表。

### 14.1 联动场景

| 场景 | 适配表作用 | 版本映射作用 |
|------|----------|------------|
| 学生转学 | 确定新地区教材版本 | 迁移历史学习状态 |
| 版本切换 | 确定新旧版本关系 | 建立知识点对应 |
| 跨地区比较 | 确定各地区使用版本 | 建立跨版本对照 |

---

## 15. 与知识底座和学科库的关系

地区教材适配表将决定：

| 决定项 | 说明 |
|--------|------|
| 哪套章节树进入某地区主链路 | 章节树选择 |
| 哪套知识点和锚点对该地区学生生效 | 知识点生效范围 |
| 哪些题型和误区解释适用 | 解释适配 |

没有这张表，知识底座会默认整个中国只有一套教材宇宙，这显然过于理想主义。

## 16. 质量要求

### 16.1 一致性

| 要求 | 说明 |
|------|------|
| 地区、学段、年级、学科和教材版本的组合口径必须统一 | 避免口径混乱 |
| region_id 必须与地区规则表一致 | 地区标识统一 |
| curriculum_version_id 必须与教材元数据表一致 | 版本标识统一 |

### 16.2 唯一性

| 要求 | 说明 |
|------|------|
| 同一组合下默认版本不能同时有多个冲突记录 | default_flag = true 只能有一条 |
| 同一学校级例外不能重复记录 | 避免例外冲突 |

### 16.3 时效性

| 要求 | 说明 |
|------|------|
| 必须明确生效时间 | valid_from 必填 |
| 过渡期适配必须有失效时间 | transition 类型 valid_to 必填 |

### 16.4 可追溯性

| 要求 | 说明 |
|------|------|
| 来源和依据必须清楚 | source_ref 必填 |
| 政策文件应可追溯 | 来源可查证 |

### 16.5 可维护性

| 要求 | 说明 |
|------|------|
| 支持新增地区、新增学校例外和过渡版本 | 结构支持扩展 |
| 支持版本状态变更 | active/deprecated 可切换 |

## 17. 常见错误

### 17.1 只写地区不写年级

| 错误 | 后果 |
|------|------|
| 同一地区不同年级用不同教材，但适配表没区分 | 一到跨年级场景就出问题 |

### 17.2 只写教材名不写版本标识

| 错误 | 后果 |
|------|------|
| curriculum_version_id 写成教材名称 | 后续系统无法稳定引用 |

### 17.3 多个默认版本并存

| 错误 | 后果 |
|------|------|
| 同一组合下多条 default_flag = true 记录 | 会直接导致绑定逻辑漂移 |

### 17.4 例外情况不显式记录

| 错误 | 后果 |
|------|------|
| 特殊学校版本靠口头传达 | 最后所有"特殊情况"都会在人脑里流传，系统本身完全不知道 |

### 17.5 时间边界缺失

| 错误 | 后果 |
|------|------|
| transition 类型没有 valid_to | 过渡期变成永久期，后续版本切换困难 |

## 18. MVP 建议范围

MVP 阶段建议优先支持：

### 18.1 地区范围

| 范围 | 说明 |
|------|------|
| 一个试点地区 | 如深圳市 |
| 市级适配为主 | 先覆盖市级统一场景 |
| 少量学校级例外 | 特殊学校单独记录 |

### 18.2 学科范围

| 学科 | 说明 |
|------|------|
| 高中物理 | 首建学科 |
| 高中数学 | 第二批 |
| 高中英语 | 第二批 |

### 18.3 内容范围

| 内容 | 说明 |
|------|------|
| 默认教材版本适配 | standard 类型 |
| 少量特殊学校例外记录 | special_case 类型 |
| 有效时间与来源信息完整 | valid_from 和 source_ref 必填 |

## 21. 地区教材适配表维护责任

### 21.1 角色定义

| 角色 | 职责 |
|------|------|
| 适配表创建人 | 负责创建地区教材适配记录 |
| 适配表校验人 | 负责校验适配关系准确性 |
| 适配表审核人 | 负责审核适配表质量和发布 |

| 周期 | 工作内容 |
|------|----------|

| 年度 | 适配表全面审查、历史版本归档 |
| 政策变更时 | 版本更新、状态同步 |

## 22. 风险与边界

### 22.1 不能忽视地区粒度

| 风险 | 说明 |
|------|------|
| 省级适配覆盖市级特殊情况 | 适配关系不准确 |

### 22.2 不能忽视时间边界

| 风险 | 说明 |
|------|------|
| 过渡期没有失效时间 | 过渡期变成永久期 |

### 22.3 不能忽视例外记录

| 风险 | 说明 |
|------|------|
| 特殊学校版本不记录 | 系统无法识别例外情况 |

### 22.4 不能忽视版本引用

| 风险 | 说明 |
|------|------|
| curriculum_version_id 引用不存在版本 | 绑定失败 |

### 22.5 不能忽视多版本冲突

| 风险 | 说明 |
|------|------|
| 多个 default_flag = true 记录 | 绑定逻辑混乱 |

## 23. 结论

地区教材适配表的核心意义，是把"某地区到底用哪套教材"从口头经验变成系统可计算、可追溯、可维护的正式结构。

它必须坚持：

| 原则 | 说明 |
|------|------|
| **地区清楚** | region_id 和 region_level 明确 |
| **年级清楚** | stage_level 和 grade_level 明确 |
| **版本清楚** | curriculum_version_id 明确且有效 |
| **时间清楚** | valid_from 和 valid_to 明确 |
| **例外清楚** | exception_scope 显式记录特殊情况 |

否则学生初始化、学校接入、知识底座绑定这些基础动作，后面都会带着隐性错误一路滚雪球。
