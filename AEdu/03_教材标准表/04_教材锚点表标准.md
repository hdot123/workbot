## 1. 文档目的

本文档用于定义教育孪生项目中的"教材锚点表"标准，包括锚点定义、锚点粒度、字段结构、定位规则、映射关系与质量要求。

本文档重点回答以下问题：

1. 什么叫教材锚点
2. 为什么知识底座必须有锚点表
3. 锚点应定位到什么粒度
4. 锚点如何与章节、知识点、实验、题型和原始教材建立关系
5. 如何避免锚点做成"页码抄录表"或"定位精度假装很高的半残结构"

---

## 2. 教材锚点定义

教材锚点，是指教材原始内容中的可定位位置单元，用于把结构化对象重新映射回教材中的具体出处。

锚点的核心作用，是回答：

- 某个知识点在教材哪里出现
- 某个实验活动在教材哪一页、哪一块
- 某个章节节点对应教材的哪些位置
- 某个解释和建议能否回到教材依据

教材锚点不是知识点本身，也不是章节树，而是"结构化世界"和"原始教材"之间的定位桥梁。

---

## 3. 教材锚点表的目标

### 3.1 支撑可追溯性
让系统中的知识点、章节、误区、实验、题型都能回到教材出处。

### 3.2 支撑解释层
前台观察层和报告层在必要时可引用教材位置作为依据。

### 3.3 支撑 OCR 与文档理解
扫描件、页码、文本片段可通过锚点与教材结构建立稳定对应。

### 3.4 支撑版本迁移
不同教材版本可通过锚点增强映射精度。

### 3.5 支撑人工校验
当系统映射有疑义时，锚点能帮助人工快速回看原文位置。

---

## 4. 设计原则

### 4.1 原始出处优先
锚点的首要任务是忠实定位原始教材位置，而不是重写内容。

### 4.2 粒度适中
锚点既不能粗到只剩一整章，也不能细到每个标点都想定位。

### 4.3 可映射
锚点必须能和章节树、知识点、实验活动、题型等建立映射关系。

### 4.4 可检索
锚点应支持通过页码、章节路径、对象引用快速查找。

### 4.5 可扩展
MVP 可先做到页级和块级，后续再扩到更细粒度。

---

## 5. 锚点对象范围

教材锚点建议覆盖以下对象：

1. 章节标题锚点
2. 正文知识表达锚点
3. 公式锚点
4. 图示/插图锚点
5. 实验活动锚点
6. 例题锚点
7. 课后习题锚点
8. 阅读/补充说明锚点

---

## 6. 锚点粒度建议

建议锚点粒度分为四层：

### 6.1 页级锚点
锚定到教材页码范围。
- 用途：章节起止页、知识点大致位置、报告引用
- 特点：实现成本低，适合 MVP

### 6.2 区块级锚点
锚定到页内某个段落块、说明块、图文块、实验块、例题块。
- 用途：精准回到某段知识表达、区分同页多个内容对象
- 特点：比页级更细，但仍保持可维护

### 6.3 对象级锚点
锚定到教材中的特定对象，如某个公式、某张图、某道例题、某个实验框。
- 用途：实验活动表映射、题型映射、图像/公式定位、OCR 回贴
- 特点：适合高价值内容

### 6.4 引用级锚点
用于前台或报告中生成引用时使用的轻量引用信息。
- 内容：教材名、册别、页码、章节路径、对象标题
- 用途：更偏前台和解释层

---

## 7. 标准字段定义

教材锚点表建议至少包含以下字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| anchor_id | string | 是 | 锚点唯一标识 |
| curriculum_version_id | string | 是 | 教材版本标识 |
| subject | string | 是 | 学科 |
| grade_level | string | 是 | 年级 |
| volume_no | string | 是 | 册别 |
| anchor_type | string | 是 | 锚点类型 |
| page_no | int | 是 | 所在页码 |
| page_range_start | int/null | 否 | 起始页 |
| page_range_end | int/null | 否 | 结束页 |
| block_code | string/null | 否 | 页内区块编码 |
| object_code | string/null | 否 | 对象编码 |
| chapter_node_id | string/null | 否 | 所属章节节点 |
| knowledge_refs | array[string] | 否 | 关联知识点 |
| ability_refs | array[string] | 否 | 关联能力点 |
| experiment_refs | array[string] | 否 | 关联实验活动 |
| exercise_type_refs | array[string] | 否 | 关联题型 |
| anchor_title | string/null | 否 | 锚点标题 |
| anchor_text_excerpt | string/null | 否 | 摘要文本 |
| source_ref | string | 是 | 来源引用 |
| status | string | 是 | active/deprecated |
| version | string | 是 | 版本号 |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 更新时间 |

---

## 8. 字段说明

### 8.1 anchor_id
锚点唯一主键。

建议格式：`PHY_ANCHOR_001`

### 8.2 anchor_type
建议值：
- chapter_title（章节标题）
- paragraph_block（段落块）
- formula（公式）
- figure（图示）
- example（例题）
- exercise（习题）
- experiment（实验）
- reading_note（阅读说明）

### 8.3 block_code
表示页内区块位置。

示例：`P023_B01`、`P023_B02`

### 8.4 object_code
表示更细粒度对象。

示例：`P023_EX01`、`P023_FIG01`、`P023_FORMULA01`

### 8.5 anchor_text_excerpt
用于保存短摘要，不建议保存长文本全文。

---

## 9. 锚点编码建议

建议锚点编码结构如下：

`学科 + 类型 + 顺序号`

示例：
- `PHY_ANCHOR_001`
- `PHY_ANCHOR_002`

如需更强可读性，可扩展为：`PHY_V1_P023_B01`

但主键建议保持稳定，不要把所有位置信息都绑死在主键里。

---

## 10. 锚点定位规则

### 10.1 页码定位
页码必须与教材 PDF 或标准教材页保持一致。

### 10.2 区块定位
同一页多个内容对象时，优先用 block_code 区分。

### 10.3 对象定位
公式、图、例题、实验等高价值对象优先建立对象级锚点。

### 10.4 路径补充
锚点应尽量补充章节路径，便于上下文理解。

---

## 11. 与章节树表的关系

章节树提供结构路径，锚点表提供原始位置。

建议规则：
- 章节节点可映射多个锚点
- 一个锚点通常归属于一个主章节节点
- 同一章节下的不同知识表达应通过多个锚点区分

---

## 12. 与知识点表的关系

建议规则：
- 每个高价值知识点至少有一个主锚点
- 同一知识点可以有多个锚点
- 主锚点用于解释和展示，辅锚点用于补充和扩展

---

## 13. 与实验活动表的关系

实验类学科建议明确建立：
- 实验活动 -> 实验锚点
- 实验锚点 -> 页码 / 图示 / 步骤块

这样后续观察层和报告层才能更准确引用实验出处。

---

## 14. 与题型映射表的关系

例题、习题、典型练习可建立锚点后，与题型映射表形成：
- 题型 -> 例题锚点
- 题型 -> 习题锚点

这比单纯说"该题型常见于某章"更有解释力。

---

## 15. 抽取流程建议

```text
读取教材原始 PDF
    ↓
识别页码与目录结构
    ↓
标记章节标题位置
    ↓
识别高价值正文区块
    ↓
识别图示/公式/实验/例题
    ↓
生成锚点编码
    ↓
挂载章节与知识点引用
    ↓
质量校验
```

---

## 16. 质量要求

### 16.1 可追溯性
核心知识点和章节必须能回到教材出处。

### 16.2 一致性
页码、区块命名、对象命名规则必须统一。

### 16.3 精度适配
MVP 不要求过细，但要保证"够用"。

### 16.4 映射性
锚点必须能支持知识点、章节、实验、题型等对象的引用。

### 16.5 可维护性
不要把锚点设计得精细到后期没人愿意维护。

---

## 17. 常见错误

### 17.1 只有页码没有对象
这样在同页内容复杂时几乎没法用。

### 17.2 锚点粒度过细
一开始就追求超精细定位，很容易把自己做进维护地狱。

### 17.3 锚点与章节路径脱节
后续很难理解锚点的上下文位置。

### 17.4 保存长文本全文
锚点表不是全文库，不要把它做成文本仓。

---

## 18. MVP 建议范围

MVP 阶段建议优先支持：
- 高中物理
- 页级锚点
- 章节标题锚点
- 核心知识点主锚点
- 例题与实验锚点（关键部分）

后置能力：
- 全量对象级锚点
- 全书高精度区块切分
- 更复杂图像区域定位

## 21. 结论

教材锚点表的核心价值，是让系统在做出解释、映射、报告和推演时，不至于悬在空中，而能回到教材原始出处。

它必须坚持：
- **有定位** - 页码和区块清晰
- **有路径** - 章节路径完整
- **有对象** - 高价值对象有独立锚点
- **有引用价值** - 能支撑报告和解释
- **有维护可行性** - 粒度适中可维护

否则系统就会一直说"依据教材"，但一问教材哪儿，它就开始沉默。
