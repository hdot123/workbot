> 最后更新：待定  
> 维护人：教材标准表负责人

---

## 1. 文档目的

本文档用于定义教育孪生项目中"知识点表"的标准结构、提取原则、字段规范、映射关系与质量要求。

知识点表是固定知识底座中的核心内容表，用于回答：

1. 教材中到底有哪些可建模的知识点
2. 一个知识点应被如何抽象和命名
3. 知识点如何与章节、能力点、错题、事件、推演建立关系
4. 知识点如何被用于学生状态建模与图谱记忆

---

## 2. 知识点定义

知识点，是教材和教学内容中可以被相对明确识别、学习、测量、解释和映射的最小知识单元。

知识点应满足以下特征：

- 可命名
- 可定位
- 可学习
- 可被测评
- 可与题型/误区/能力要求建立映射

知识点不是整章标题，不是随意切碎的一句话，也不是完全抽象的学科理念。

---

## 3. 知识点表的作用

### 3.1 服务学生状态建模
知识点掌握状态是 StudentTwinAgent 的核心状态之一。

### 3.2 服务学习事件映射
错题、作业、考试、教师反馈等事件最终要尽量映射到知识点。

### 3.3 服务图谱建模
知识点是教育图谱中的核心实体之一。

### 3.4 服务报告与解释
家长、老师、学校端的很多解释都应落到知识点层，而不是只说"物理不太稳定"。

---

## 4. 提取原则

### 4.1 以教学可用性为准
知识点抽取不是越细越好，而是要保证教学、测评和解释可用。

### 4.2 以教材为锚
知识点应优先锚定教材和课程表达，不脱离教材胡乱抽象。

### 4.3 避免过粗
"牛顿运动定律"可以过粗，应根据教材结构和教学使用进一步拆解。

### 4.4 避免过细
"牛顿第二定律中 F、m、a 三个字母的顺序"这种颗粒度过细，不适合作为知识点主层。

### 4.5 一点一义
一个知识点应尽量表达单一核心概念或单一核心规律，避免混合多个独立概念。

### 4.6 可映射
知识点应能映射到：
- 章节
- 锚点
- 题型
- 误区
- 能力点
- 学习事件

---

## 5. 知识点粒度标准

建议知识点粒度介于"章节"与"具体题目"之间。

| 粒度层级 | 适用性 | 示例 |
|----------|--------|------|
| 章节级 | 过粗 | 牛顿运动定律 |
| 知识点级 | 推荐 | 牛顿第二定律及其公式关系 |
| 题目级 | 过细 | 某题第 3 问中的受力分解步骤 |

推荐原则：
- 一个知识点通常可支撑 3~10 道题的统一解释
- 一个知识点通常可对应 1~3 个能力点
- 一个知识点通常能形成相对稳定的掌握状态

---

## 6. 标准字段定义

知识点表建议至少包含以下字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| knowledge_id | string | 是 | 知识点唯一标识 |
| curriculum_version_id | string | 是 | 教材版本标识 |
| subject | string | 是 | 学科 |
| grade_level | string | 是 | 年级 |
| chapter_node_id | string | 是 | 主挂载章节节点 |
| secondary_chapter_refs | array[string] | 否 | 次挂载章节 |
| knowledge_name | string | 是 | 知识点名称 |
| knowledge_aliases | array[string] | 否 | 别名/同义表达 |
| knowledge_definition | string | 是 | 知识点定义 |
| knowledge_type | string | 是 | 概念/规律/方法/模型/实验等 |
| granularity_level | string | 是 | 粒度级别 |
| prerequisite_refs | array[string] | 否 | 前置知识点 |
| successor_refs | array[string] | 否 | 后继知识点 |
| ability_refs | array[string] | 否 | 关联能力点 |
| misconception_refs | array[string] | 否 | 关联误区 |
| exercise_type_refs | array[string] | 否 | 关联题型 |
| anchor_refs | array[string] | 否 | 教材锚点引用 |
| measurable_flag | boolean | 是 | 是否可直接测评 |
| importance_level | string | 是 | 核心程度 |
| status | string | 是 | active/deprecated |
| source_ref | string/null | 否 | 来源引用 |
| version | string | 是 | 版本号 |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 更新时间 |

---

## 7. 关键字段说明

### 7.1 knowledge_id
知识点唯一主键。

建议格式：`PHY_PEP_G1_KP_001`

### 7.2 knowledge_name
知识点标准名称，应简洁、稳定、可教学使用。

示例：
- 正确：牛顿第二定律及其公式关系
- 不推荐：关于力和加速度那个公式

### 7.3 knowledge_definition
知识点标准定义，要求：
- 简明
- 准确
- 不写成长篇讲义
- 能支持理解和映射

### 7.4 knowledge_type
建议枚举值：
- concept（概念）
- law（规律）
- method（方法）
- model（模型）
- procedure（程序/步骤）
- experiment（实验）
- representation（表征/图像）

### 7.5 measurable_flag
是否适合通过题目、作业、考试等方式直接观察掌握程度。

---

## 8. 知识点命名规范

### 8.1 命名原则
- 使用标准教学表达
- 避免口语化
- 避免过长
- 避免多义
- 避免把多个独立点混在一起

### 8.2 推荐写法
- 位移与路程的区别
- 匀变速直线运动公式应用
- 受力分析的基本方法
- 牛顿第二定律及其公式关系

### 8.3 不推荐写法
- 运动这部分
- 力学重点 1
- 老师常考那个
- 公式和受力还有图像综合

---

## 9. 知识点提取流程

### 9.1 原始来源
- 教材正文
- 目录结构
- 课后习题
- 教师教学经验补充
- 课程标准（如适用）

### 9.2 提取步骤

```
教材结构阅读
    ↓
章节主旨识别
    ↓
候选知识点抽取
    ↓
去重与合并
    ↓
粒度校正
    ↓
标准命名
    ↓
定义撰写
    ↓
挂载章节
    ↓
建立依赖与映射
    ↓
质量校验
```

## 10. 与章节树表的关系

### 10.1 主挂载关系

每个知识点必须至少有一个主挂载章节节点。

### 10.2 次挂载关系

若知识点跨章节反复出现，可增加次挂载章节。

### 10.3 原则

章节树负责"结构位置"，知识点表负责"内容抽象"。

---

## 11. 与能力点表的关系

知识点和能力点的关系通常是多对多：

- 一个知识点可对应多个能力要求
- 一个能力点可服务多个知识点

示例：
- 知识点：受力分析的基本方法
- 能力点：分析能力、建模能力、图示表达能力

---

## 12. 与误区表的关系

每个知识点应尽量关联常见误区。

示例：
- 知识点：位移与路程的区别
- 常见误区：把路程当位移、忽略方向性

误区表不是知识点定义的一部分，但必须能挂回知识点。

---

## 13. 与题型映射表的关系

知识点表应能关联常见题型，用于：
- 错题解释
- 风险识别
- 干预设计
- 推演模拟

示例：
- 知识点：匀变速直线运动公式应用
- 题型：基础公式代入题 / 图像综合题 / 多阶段运动题

---

## 14. 与学习事件的关系

学习事件映射知识点时，建议遵循以下规则：

1. 高置信题目识别可直接映射知识点
2. 低置信文本输入可先映射章节，再补知识点
3. 教师反馈中提到的教学术语，应尽量对应标准知识点名称
4. 映射失败时可标记待确认，不强行乱挂

---

## 15. 数据质量要求

### 15.1 唯一性

同一版本下，知识点名称和编码不能重复表达同一含义。

### 15.2 一致性

命名、定义、类型、粒度口径要统一。

### 15.3 可测性

大多数知识点应能通过事件或题目得到一定程度观测。

### 15.4 可解释性

知识点定义必须足够清楚，能支撑产品解释。

### 15.5 可映射性

必须支持和章节、能力点、题型、误区、事件之间的映射。

---

## 16. 常见错误

### 16.1 把章节标题直接当知识点

章节标题通常太粗，不能直接代替知识点表。

### 16.2 把题目问法当知识点

题目只是表现形式，不是知识点本体。

### 16.3 粒度失衡

同一张表里同时出现"牛顿运动定律"和"受力图中箭头方向标注规则"这种跨度，会导致后续状态建模混乱。

### 16.4 定义写成讲义

知识点定义应是标准定义，不是课堂教学逐字稿。

---

## 17. MVP 最小标准

MVP 阶段建议知识点表至少满足：

- 高中物理
- 单一教材版本
- 每章均有核心知识点抽取
- 每个知识点有主挂载章节
- 每个知识点有基础定义
- 每个知识点尽量关联至少一个能力点或题型

## 20. 结论

知识点表是教育孪生项目中最核心的内容抽象表之一。

如果没有知识点表：

- 学生状态无法细粒度表达
- 学习事件无法稳定映射
- 风险识别会停留在学科粗粒度
- 图谱建模会缺少核心实体

因此知识点表建设必须坚持：

- 粒度适中
- 命名稳定
- 定义清楚
- 映射充分
- 可用于状态、事件、图谱和推演
