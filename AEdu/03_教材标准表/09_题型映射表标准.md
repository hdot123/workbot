## 1. 文档目的

本文档用于定义教育孪生项目中的"题型映射表"标准，包括题型定义、题型层级、字段结构、与知识点/能力点/误区的映射方式与使用场景。

本文档重点回答以下问题：

1. 什么叫题型
2. 题型为什么要单独建表
3. 题型如何与知识点、能力点和错因建立关系
4. 题型映射如何服务学习事件、学生状态、报告解释和干预动作
5. 如何避免题型表变成一张"题目外号大全"

---

## 2. 题型定义

题型，是指在某学科中具有相对稳定结构、解题路径、考查重点或错误模式的题目类别单元。

题型不是某一道具体题目，也不是整个学科泛泛的"选择题/填空题"粗分类，而是处于"具体题目"和"考试题类大类"之间的结构化类型。

例如在物理中：

- 受力图分析题
- 匀变速直线运动基础计算题
- 多阶段运动综合题
- 图像判断与转换题
- 实验误差分析题

题型关注的是"这类题怎么考、考什么、容易错在哪里"。

---

## 3. 题型映射表的目标

### 3.1 支撑事件解释
学习事件中的错题、练习、作业、考试表现，很多时候最先映射到题型比直接映射知识点更稳。

### 3.2 支撑错因分析
同一个知识点失败，可能是不同题型导致的不同问题。

### 3.3 支撑能力识别
题型能更直接体现某类能力短板。

### 3.4 支撑干预设计
干预动作很多是按题型组织练习，而不是按单一知识点组织。

### 3.5 支撑报告解释
老师和家长更容易理解"在哪类题上总出问题"，而不是只看抽象状态标签。

---

## 4. 设计原则

### 4.1 题型应稳定
题型必须是可以重复出现、可被识别的类型，而不是某道题的临时描述。

### 4.2 题型应可解释
题型名称和定义应支持老师与系统共用。

### 4.3 题型应可映射
题型必须可映射到知识点、能力点、误区和事件。

### 4.4 粒度适中
题型既不能粗到只剩选择题，也不能细到每一道题都是一个题型。

### 4.5 学科特性优先
题型必须体现学科特征，不搞跨学科硬套模板。

---

## 5. 题型层级建议

建议题型至少分为三层：

### 5.1 题类层
学科中较粗粒度的问题类型分类。

**物理示例**：
- 计算类
- 图像类
- 实验类
- 分析类
- 综合应用类

**用途**：适合看板和聚合统计，但不适合作为最细粒度状态单元。

### 5.2 题型层
最主要的标准题型层，是题型映射表的核心层。

**物理示例**：
- 受力图分析题
- 匀变速公式直接应用题
- 牛顿定律综合分析题
- 图像读取与判断题
- 实验数据处理题

**用途**：适合作为事件映射单位、错误归因单位、报告解释单位、干预动作组织单位。

### 5.3 子题型层
在标准题型下进一步细分的常见变化类型。

**示例**：
- 图像判断题 -> 位移 - 时间图像判断
- 图像判断题 -> 速度 - 时间图像判断

**原则**：子题型不是必须全做，只有高价值题型才需要下钻。

## 6. 标准字段定义

题型映射表建议至少包含以下字段：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| exercise_type_id | string | 是 | 题型唯一标识 |
| subject | string | 是 | 学科 |
| curriculum_version_id | string/null | 否 | 教材版本 |
| category_name | string | 是 | 题类名称 |
| exercise_type_name | string | 是 | 题型名称 |
| subtype_name | string/null | 否 | 子题型名称 |
| exercise_type_definition | string | 是 | 题型定义 |
| knowledge_refs | array[string] | 否 | 关联知识点 |
| ability_refs | array[string] | 否 | 关联能力点 |
| misconception_refs | array[string] | 否 | 关联误区 |
| chapter_refs | array[string] | 否 | 关联章节 |
| anchor_refs | array[string] | 否 | 关联例题/习题锚点 |
| typical_features | array[string] | 否 | 典型特征 |
| observable_signals | array[string] | 否 | 可观察信号 |
| importance_level | string | 是 | 重要程度 |
| status | string | 是 | active/deprecated |
| source_ref | string/null | 否 | 来源引用 |
| version | string | 是 | 版本号 |
| created_at | datetime | 是 | 创建时间 |
| updated_at | datetime | 是 | 更新时间 |

---

## 7. 字段说明

### 7.1 exercise_type_id
题型唯一主键。

建议格式：`PHY_EX_004`

### 7.2 category_name
题类名称。

示例：
- 图像类
- 计算类
- 实验类
- 分析类

### 7.3 exercise_type_name
题型名称。

示例：
- 受力图分析题
- 匀变速直线运动基础计算题
- 实验误差分析题

### 7.4 typical_features
用于说明该题型的稳定结构特征。

示例（受力图分析题）：
- 需要先识别研究对象
- 需要绘制受力图
- 需要列出平衡或动力学关系

### 7.5 observable_signals
用于说明系统如何识别和观察该题型问题。

示例：
- 漏画或多画受力
- 力方向判断错误
- 后续列式与受力图不一致

## 8. 题型命名规范

### 8.1 命名原则
- 稳定：题型名称应长期稳定
- 清晰：名称应清晰表达题型特征
- 可教学使用：老师能看懂、能共用
- 尽量直接体现题型结构

### 8.2 推荐写法
- 受力图分析题
- 匀变速直线运动多阶段计算题
- 图像读取与判断题
- 实验数据误差分析题

### 8.3 不推荐写法
- 常考题一
- 经典难题
- 老师重点题
- 容易丢分那种题

## 9. 题型抽取流程

建议流程如下：

```text
收集教材例题与课后习题
    ↓
结合教学经验与试题结构
    ↓
抽取候选题型
    ↓
合并近义题型
    ↓
控制粒度
    ↓
标准命名与定义
    ↓
映射知识点 / 能力点 / 误区
    ↓
质量校验
```

---

## 10. 与知识点表的关系

### 10.1 关系原则

- 一个题型可映射多个知识点
- 一个知识点可对应多个题型
- 题型是知识点应用场景的稳定外显结构

### 10.2 用途

帮助系统区分：

- "不会这个知识点"
- "会知识点但在这种题里用不出来"

---

## 11. 与能力点表的关系

题型与能力点关系极强。

例如：

| 题型 | 能力点 |
|------|--------|
| 受力图分析题 | 受力分析能力 |
| 图像判断题 | 图像理解与转换能力 |
| 实验分析题 | 实验分析能力 |

因此题型映射表是连接"题目表现"和"能力短板"的关键桥。

---

## 12. 与误区表的关系

每个高价值题型建议关联典型误区。

例如：

| 题型 | 典型误区 |
|------|----------|
| 受力图分析题 | 漏画力、多画力、方向判断错误 |
| 图像题 | 把斜率和面积混淆 |
| 实验题 | 误把系统误差当偶然误差 |

---

## 13. 与学习事件的关系

学习事件映射时可采用以下优先级：

```
原始题目 / 作业 / 错题
    ↓
题型识别
    ↓
知识点映射
    ↓
能力点与误区补充
```

**原因**：题型往往是比知识点更容易从题目表层稳定识别的中间层。

---

## 14. 与学生状态模型的关系

题型映射表将直接服务以下状态：

- 高频错误题型分布
- 学科问题聚集结构
- 能力短板定位
- 风险来源解释
- 干预动作推荐

后续老师端和报告层会非常依赖这张表来回答：
"不是哪道题错了，而是总在什么类型的题上出问题。"

---

## 15. 质量要求

### 15.1 稳定性
题型定义不能随便改，否则历史事件映射会抖动。

### 15.2 一致性
命名、粒度、分类口径必须统一。

### 15.3 可映射性
必须能和知识点、能力点、误区、锚点建立联系。

### 15.4 可观察性
要能通过题目和练习识别出来。

### 15.5 可教学性
老师必须看得懂，也愿意承认这分类有用。

---

## 16. 常见错误

### 16.1 粒度过粗
只分"选择题、填空题、解答题"，这对教育孪生帮助不大。

### 16.2 粒度过细
一套题拆出十几个只出现一次的题型，后续完全不可维护。

### 16.3 与知识点混淆
题型是表现结构，不是知识内容本体。

### 16.4 与难度混淆
"难题"不是题型，最多是属性标签。

---

## 17. MVP 建议范围

MVP 阶段建议优先支持：

### 17.1 学科范围
- 高中物理

### 17.2 内容范围
- 高频核心题型
- 与知识点和能力点的基础映射
- 与典型误区的基础映射
- 与例题/习题锚点的基础引用

### 17.3 建议先覆盖
- 受力分析题
- 匀变速基础计算题
- 多阶段运动综合题
- 图像判断题
- 实验分析题

## 21. 风险与边界

### 21.1 不能忽视题型稳定性
题型定义频繁变动会导致历史事件映射失效。

### 21.2 不能忽视学科差异
不同学科题型特点不同，不能强行统一模板。

### 21.3 不能忽视粒度控制
粒度过粗或过细都会影响系统可用性。

### 21.4 不能忽视映射质量
题型与知识点、能力点的映射必须准确。

---

## 22. 结论

题型映射表的核心价值，是让系统从"题目错没错"提升到"总在什么结构的题上出问题，背后更可能是哪类知识和能力短板"。

它必须坚持：

- **类型稳定** - 题型定义长期稳定
- **粒度适中** - 不过粗也不过细
- **能映射知识** - 与知识点建立联系
- **能映射能力** - 与能力点建立联系
- **能服务解释和干预** - 支撑报告和问题定位

否则题型表就会变成一张名词很多、但真正出事时谁也帮不上忙的清单。
