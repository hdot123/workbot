## 1. 文档目的

本文档用于定义教育孪生项目中的图谱记忆层设计，包括实体建模、关系建模、记忆分层、写入规则、检索方式与上下游协同方式。

本文档重点回答以下问题：

1. 为什么教育孪生系统需要图谱记忆层
2. 图谱中存什么，不存什么
3. StudentTwinAgent、学习事件、知识底座如何写入图谱
4. 图谱如何服务观察层、推演层和 GraphRAG
5. 图谱如何控制污染、冲突和版本问题

---

## 2. 图谱记忆层定义

图谱记忆层，是教育孪生系统中用于承载长期事实、关系、事件和时序关联的结构化记忆层。

它不是单纯的知识图谱，也不是单纯的向量检索库，而是：

- 以实体和关系为骨架
- 以事件和时间为驱动
- 以学生个体为中心
- 以检索增强和解释追溯为目标

的长期记忆系统。

---

## 3. 图谱记忆层的目标

### 3.1 形成长期可追溯记忆
学生的学习过程不能只存在当前状态里，图谱层负责沉淀长期演化轨迹。

### 3.2 建立多实体关系网络
学生、家长、老师、班级、学校、教材、知识点、事件之间需要结构化连接。

### 3.3 服务 GraphRAG 检索
观察层和推演层需要的不只是文本检索，而是"有关系、有时间、有上下文"的检索。

### 3.4 支持解释与复盘
系统输出不能凭空而来，应能回溯到事实、事件与关系链。

### 3.5 支持版本化与冲突隔离
图谱应支持事实更新、状态演化、冲突保留和版本回滚。

---

## 4. 设计原则

### 4.1 事实优先
图谱首先记录事实、关系和事件，不优先记录大段结论性话术。

### 4.2 个体中心
图谱以学生个体为中心组织局部子图，不追求一开始构建全宇宙教育总图谱。

### 4.3 时间显式化
所有重要事件和状态变化应显式带时间信息。

### 4.4 关系可解释
每条关键关系都应能说明来源和依据。

### 4.5 写入受控
不是所有解析结果都应直接写图谱，低置信内容必须控制污染。

### 4.6 图谱与当前态分工
StudentTwinAgent 保存当前态摘要，图谱记忆层保存长期关系和演化轨迹。

---

## 5. 图谱记忆层的组成

建议图谱记忆层由四部分组成：

1. 事实图谱
2. 关系图谱
3. 事件记忆
4. 时序轨迹层

---

## 6. 事实图谱

### 6.1 定义
保存相对稳定、可确认的客观事实节点。

### 6.2 典型事实实体
- Student
- School
- Class
- Region
- Subject
- CurriculumVersion
- Chapter
- KnowledgePoint
- AbilityPoint
- Exam
- Homework

### 6.3 特点
- 稳定性高
- 更新频率相对低
- 是关系图谱和事件图谱的基础骨架

---

## 7. 关系图谱

### 7.1 定义
保存实体之间的语义关系。

### 7.2 典型关系
- Student -> belongs_to -> Class
- Student -> studies -> Subject
- Student -> uses -> CurriculumVersion
- KnowledgePoint -> located_in -> Chapter
- KnowledgePoint -> depends_on -> KnowledgePoint
- KnowledgePoint -> maps_to -> AbilityPoint
- Teacher -> teaches -> Class
- Parent -> related_to -> Student

### 7.3 特点
- 强调结构连接
- 服务解释与推演
- 是 GraphRAG 的重要骨架

---

## 8. 事件记忆

### 8.1 定义
保存学习过程中发生的动态事件节点。

### 8.2 典型事件
- 作业提交事件
- 考试成绩事件
- 错题识别事件
- 教师反馈事件
- 家长反馈事件
- 教材进度事件
- 干预执行事件

### 8.3 特点
- 高频写入
- 强时序属性
- 是学生状态更新的重要来源

---

## 9. 时序轨迹层

### 9.1 定义
用于把事件和状态变化串成时间链，形成学生成长过程中的可追溯演化轨迹。

### 9.2 典型内容
- 知识点掌握状态变化
- 风险等级变化
- 学科趋势变化
- 干预前后变化
- 关键事件时间线

### 9.3 特点
- 强时间结构
- 服务观察层解释
- 服务推演层建模

---

## 11. 核心关系定义

建议至少建立以下关系类型：

| 关系 | 含义 |
|------|------|
| belongs_to | 隶属 |
| related_to | 关联 |
| studies | 学习 |
| uses | 使用教材/版本 |
| located_in | 位于章节中 |
| depends_on | 前置依赖 |
| maps_to | 映射到 |
| has_event | 拥有事件 |
| refers_to | 引用知识点/章节 |
| occurred_at | 发生于时间 |
| caused_by | 由...引起 |
| implies | 暗示/关联风险 |
| intervened_by | 被干预 |
| changed_to | 状态变化到 |

## 12. 写入规则

### 12.1 什么应该写入
以下内容应优先写入图谱：
- 高置信事实
- 标准学习事件
- 关键状态变化
- 知识点与能力点关系
- 干预动作及结果关系

### 12.2 什么不应直接写入
以下内容不建议直接写图谱：
- 原始长文本整段复制
- 低置信 OCR 结果
- 未校验的模糊推断
- 临时前端展示文案
- 单次未经验证的情绪判断

### 12.3 写入原则
- 优先写结构化节点和边
- 文本摘要可作为属性，不作为主体
- 所有重要写入需带来源和时间

---

## 13. 冲突与污染控制

### 13.1 冲突来源
- 不同来源对同一事实描述不同
- OCR 结果与人工录入不一致
- 教师反馈与考试结果表面冲突
- 模型推断与已知事实冲突

### 13.2 处理规则
1. 不直接删除旧记录
2. 保留冲突来源和时间
3. 标记当前推荐版本
4. 必要时进入人工复核
5. 低置信信息不直接覆盖高置信事实

### 13.3 污染控制
- 设置写入阈值
- 区分 confirmed / inferred / pending
- 推断关系默认弱写入
- 高风险推断不默认长期固化

---

## 14. 图谱与 StudentTwinAgent 的分工

### 14.1 StudentTwinAgent 负责
- 当前状态摘要
- 当前风险等级
- 当前关注点
- 当前学科状态
- 当前推演准备信息

### 14.2 图谱记忆层负责
- 长期事件链
- 历史状态变化轨迹
- 多实体关系
- 知识点依赖和映射
- 干预动作与结果关系

### 14.3 原则
Agent 是当前态工作区，图谱是长期结构化记忆。两者不是互相替代关系。

---

## 15. 图谱与 GraphRAG 的关系

### 15.1 图谱提供结构骨架
GraphRAG 不应只是"向量检索 + 拼文本"，图谱提供关系和路径约束。

### 15.2 检索目标
GraphRAG 典型检索对象包括：
- 某学生近期关键事件链
- 某知识点相关错题和反馈
- 某风险信号的来源依据
- 某干预动作的历史上下文

### 15.3 上下文装配
建议上下文由三部分构成：
- 当前态摘要
- 图谱关系链
- 相关原始事件/文本片段

---

## 16. 与学习事件的关系

学习事件进入系统后，典型图谱写入路径为：

学习事件生成  
→ Student 节点  
→ LearningEvent 节点  
→ KnowledgePoint / Chapter / AbilityPoint 映射  
→ RiskSignal（如适用）  
→ 时间节点关联  
→ 状态变化边或干预关系边

---

## 17. 与状态模型的关系

图谱记忆层与状态模型的关系如下：

- 状态模型定义"当前怎么表达"
- 图谱记忆层定义"历史如何沉淀"
- 状态变化应能在图谱中找到来源事件和关系依据
- 图谱可为状态重算提供长期上下文支持

---

## 18. MVP 建议范围

MVP 阶段图谱层不宜做得过大，建议优先支持：

### 18.1 必须支持
- Student
- KnowledgePoint
- LearningEvent
- Chapter
- CurriculumVersion
- RiskSignal（基础版）

### 18.2 必须支持的关系
- has_event
- refers_to
- located_in
- uses
- belongs_to
- occurred_at

### 18.3 可以后置
- 复杂社交关系
- 跨学校群体关系
- 高级因果链建模
- 自动生成复杂解释路径

## 21. 结论

图谱记忆层是教育孪生系统中的长期结构化记忆中枢。

如果没有图谱记忆层：

- StudentTwinAgent 只能保存当前态
- 历史关系和演化轨迹无法沉淀
- GraphRAG 会退化成普通文本检索
- 观察与推演缺少可解释依据

因此图谱层建设必须坚持：

- 先个体中心
- 先高价值实体与关系
- 先可追溯
- 再扩复杂网络
- 先服务状态更新和解释
- 再扩展高级推演
