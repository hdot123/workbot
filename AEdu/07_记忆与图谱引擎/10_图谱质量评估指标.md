# 图谱质量评估指标

> 文档编号：GRAPH-010  
> 版本：V1.0  
> 创建日期：2024  
> 最后更新：待定  
> 维护人：记忆与图谱引擎负责人

---

## 1. 文档目的

本文档用于定义教育孪生项目中知识图谱与记忆系统的质量评估指标，包括结构质量、内容质量、时效质量、检索质量、解释质量与运维质量等维度。

本文档重点回答以下问题：

1. 怎么判断当前图谱质量是高还是低
2. 图谱质量应该看哪些指标
3. 哪些指标用于日常监控，哪些指标用于阶段评估
4. 如何避免图谱"越长越大，但质量并不更好"
5. 如何让图谱质量评估真正服务系统演进，而不是做漂亮表格

---

## 2. 图谱质量定义

图谱质量，是指知识图谱与记忆系统在对象完整性、关系正确性、时效更新性、检索可用性、解释支撑力与运行稳定性等方面达到的整体可用水平。

一个图谱"节点很多、边很多"不等于质量高。  
真正高质量的图谱，应满足：

- 对象边界清楚
- 关系结构可信
- 当前态新鲜
- 历史链条可追
- 检索结果有用
- 解释结果稳定
- 更新和回滚都可控

---

## 3. 建设目标

### 3.1 建立统一质量语言
让产品、技术、数据和运营可以用同一套口径讨论图谱质量。

### 3.2 支撑持续优化
通过指标识别图谱薄弱点，指导后续修复和演进。

### 3.3 支撑阶段验收
判断某一阶段图谱是否达到可进入试点主链路的标准。

### 3.4 支撑监控与复盘
高频指标进入监控，阶段指标进入复盘。

### 3.5 控制质量幻觉
防止"对象越来越多"被误认为"图谱越来越好"。

---

## 4. 设计原则

### 4.1 结构与业务并重
图谱既要看结构正确性，也要看业务可用性。

### 4.2 指标分层
区分在线运行指标、离线质量指标和业务价值指标。

### 4.3 少而关键
优先跟踪能真正反映问题的指标，不做指标森林。

### 4.4 可追踪趋势
指标要能看趋势，而不是只看某一天快照。

### 4.5 可行动
指标异常时，应能知道下一步往哪里查。

---

## 5. 评估维度建议

建议至少从以下七个维度评估：

| 维度 | 说明 | 评估频率 |
|------|------|----------|
| 结构质量 | 实体、关系和版本链是否结构健康 | 周/月 |
| 对象质量 | 事实、关系、事件对象本身是否可靠 | 周/月 |
| 更新质量 | 图谱增量更新是否及时、稳定、正确 | 日/周 |
| 检索质量 | GraphRAG 是否能拿到正确、相关、低噪音的上下文 | 日/周 |
| 解释质量 | 图谱是否真正支撑"为什么如此"的解释 | 周/月 |
| 运行质量 | 图谱系统的运行稳定性与可维护性 | 日/周 |
| 业务质量 | 图谱对业务链路的真实支撑价值 | 周/月 |

---

## 6. 结构质量

### 6.1 定义
评估实体、关系和版本链本身是否结构健康。

### 6.2 建议指标

| 指标 | 计算公式 | 告警阈值 |
|------|----------|----------|
| 实体类型覆盖率 | 已定义实体类型数 / 应定义实体类型数 | < 80% |
| 核心关系覆盖率 | 已建立核心关系数 / 应建立核心关系数 | < 70% |
| 非法关系占比 | 不符合规范的关系数 / 总关系数 | > 5% |
| 环路异常率 | 检测到异常环路的对象数 / 总对象数 | > 2% |
| 孤立节点占比 | 无有效关系连接的核心实体数 / 总核心实体数 | > 10% |
| 缺失来源引用占比 | 无来源引用的对象数 / 总对象数 | > 15% |
| 版本链断裂率 | 缺失 previous_version_id 的版本对象数 / 应有版本链对象数 | > 5% |

### 6.3 关注点
图谱结构是否像一张能走通的图，而不是一堆散点加误连边。

---

## 7. 对象质量

### 7.1 定义
评估事实、关系、事件对象本身是否可靠。

### 7.2 建议指标

| 指标 | 计算公式 | 告警阈值 |
|------|----------|----------|
| 对象唯一性冲突率 | 主标识冲突对象数 / 总对象数 | > 1% |
| 重复对象率 | 逻辑重复对象数 / 总对象数 | > 5% |
| 无效对象率 | 标记 invalid/retracted 的对象数 / 总对象数 | > 10% |
| 低置信对象占比 | 置信度低于阈值的对象数 / 总对象数 | > 20% |
| 当前有效事实完整率 | 有效事实字段数 / 应存在事实字段数 | < 85% |
| 学生关键事实覆盖率 | 有关键事实的学生数 / 总学生数 | < 90% |
| 学生关键关系覆盖率 | 有关键关系的学生数 / 总学生数 | < 90% |

---

## 8. 更新质量

### 8.1 定义
评估图谱增量更新是否及时、稳定、正确。

### 8.2 建议指标

| 指标 | 计算公式 | 告警阈值 |
|------|----------|----------|
| 图谱写入成功率 | 写入成功次数 / 总写入次数 | < 95% |
| 事实 supersede 成功率 | supersede 成功次数 / 总 supersede 次数 | < 95% |
| 关系更新成功率 | 关系更新成功次数 / 总关系更新次数 | < 95% |
| 索引刷新成功率 | 索引刷新成功次数 / 总索引刷新次数 | < 95% |
| 更新延迟 (P95) | 95% 更新操作的延迟时间 | > 60s |
| 重试率 | 需要重试的写入次数 / 总写入次数 | > 10% |
| 回滚次数 | 单位时间内回滚操作次数 | > 5 次/天 |
| 回滚恢复成功率 | 回滚后恢复预期状态的对象数 / 回滚对象总数 | < 95% |

---

## 9. 检索质量

### 9.1 定义
评估 GraphRAG 是否能拿到正确、相关、低噪音的上下文。

### 9.2 建议指标

| 指标 | 计算公式 | 告警阈值 |
|------|----------|----------|
| 检索命中率 | 召回相关对象的查询数 / 总查询数 | < 90% |
| 当前态召回率 | 召回当前关键事实的查询数 / 应召回查询数 | < 95% |
| 关键对象召回率 | 召回关键对象的查询数 / 应召回查询数 | < 90% |
| 上下文噪音率 | 人工评估为低相关的内容块数 / 总内容块数 | > 20% |
| 检索延迟 (P95) | 95% 检索查询的响应时间 | > 1000ms |
| 角色化过滤正确率 | 过滤正确的查询数 / 总查询数 | < 95% |
| 上下文装配成功率 | 装配成功的查询数 / 总查询数 | < 95% |

### 9.3 说明
图谱如果检索质量低，再完整也只是冷藏库，不是可用中台。

## 10. 解释质量

### 10.1 定义
评估图谱是否真正支撑"为什么如此"的解释。

### 10.2 建议指标

| 指标 | 计算公式 | 告警阈值 |
|------|----------|----------|
| 可追溯解释占比 | 可回到来源的解释输出数 / 总解释输出数 | < 85% |
| 解释引用成功率 | 引用成功的解释数 / 总解释数 | < 90% |
| 解释链完整率 | 同时具备"当前态 + 结构依据 + 关键事件"的解释数 / 总解释数 | < 80% |
| 老师端解释认可度 | 老师认可的解释数 / 总解释数 | < 80% |
| 人工复核通过率 | 人工复核通过的解释数 / 总复核数 | < 90% |
| 解释冲突率 | 相近时间窗口中互相矛盾的解释数 / 总解释数 | > 5% |

### 10.3 原则
图谱不是为了让数据库好看，而是为了让系统能说清楚。

## 11. 运行质量

### 11.1 定义
评估图谱系统的运行稳定性与可维护性。

### 11.2 建议指标

| 指标 | 计算公式 | 告警阈值 |
|------|----------|----------|
| 图谱查询成功率 | 查询成功次数 / 总查询次数 | < 95% |
| 查询耗时 (P95) | 95% 查询的响应时间 | > 500ms |
| 索引刷新时延 | 从写入完成到索引可用的时间 | > 300s |
| 死信任务数量 | 死信队列中图谱相关任务数 | > 50 |
| 图谱相关任务积压量 | 等待执行的图谱任务数 | > 1000 |
| 写入失败率 | 写入失败次数 / 总写入次数 | > 5% |
| 版本回滚耗时 | 回滚操作平均耗时 | > 300s |

## 12. 业务质量

### 12.1 定义
评估图谱对业务链路的真实支撑价值。

### 12.2 建议指标

| 指标 | 计算公式 | 告警阈值 |
|------|----------|----------|
| 学生详情页解释可用率 | 解释可用的详情页访问数 / 总访问数 | < 90% |
| 报告生成引用图谱成功率 | 成功引用图谱的报告数 / 总报告数 | < 95% |
| 风险解释链图谱支撑率 | 有图谱支撑的风险解释数 / 总风险解释数 | < 85% |
| 干预建议图谱支撑率 | 有图谱支撑的干预建议数 / 总干预建议数 | < 85% |
| 推演任务图谱依赖成功率 | 成功依赖图谱的推演任务数 / 总推演任务数 | < 90% |

## 14. 结构质量指标定义建议

### 14.1 孤立节点占比

**定义**：没有任何有效关系连接的核心实体占比。

**计算公式**：
```
孤立节点占比 = 孤立核心实体数 / 总核心实体数 × 100%
```

**说明**：核心实体指 Student, KnowledgePoint, AbilityPoint 等高价值对象。

### 14.2 非法关系占比

**定义**：不符合实体关系规范的边占比。

**计算公式**：
```
非法关系占比 = 非法关系数 / 总关系数 × 100%
```

**说明**：非法关系指不符合 GRAPH-008 定义的关系类型。

### 14.3 版本链断裂率

**定义**：应存在 previous_version_id 但缺失的版本对象占比。

**计算公式**：
```
版本链断裂率 = 缺失 previous_version_id 的对象数 / 应有版本链对象数 × 100%
```

### 14.4 环路异常率

**定义**：在不应存在循环依赖的关系集合中，检测到环的比例。

**计算公式**：
```
环路异常率 = 检测到异常环路的对象集合数 / 总对象集合数 × 100%
```

---

## 15. 对象质量指标定义建议

### 15.1 当前有效事实完整率

**定义**：单学生关键 fact 字段中有效存在的比例。

**计算公式**：
```
当前有效事实完整率 = 有效存在的关键事实字段数 / 应存在的关键事实字段数 × 100%
```

**关键事实字段**：current_risk_level, current_focus_knowledge_points, data_completeness_score 等。

### 15.2 重复对象率

**定义**：逻辑上同一对象被重复写成多个活跃对象的比例。

**计算公式**：
```
重复对象率 = 重复对象组数 / 总对象组数 × 100%
```

### 15.3 无效对象率

**定义**：已标记 invalid / retracted / deprecated 但仍被错误引用的对象比例。

**计算公式**：
```
无效对象率 = 被错误引用的无效对象数 / 总无效对象数 × 100%
```

---

## 16. 更新质量指标定义建议

### 16.1 更新延迟

**定义**：从标准事件进入到图谱对象可查询之间的时间差。

**计算公式**：
```
更新延迟 = 图谱对象可查询时间 - 标准事件进入时间
```

**监控**：P50, P95, P99 分位数。

### 16.2 supersede 成功率

**定义**：当前态事实替换成功的比例。

**计算公式**：
```
supersede 成功率 = supersede 成功次数 / 总 supersede 次数 × 100%
```

### 16.3 回滚恢复成功率

**定义**：回滚任务后对象恢复为预期状态的比例。

**计算公式**：
```
回滚恢复成功率 = 恢复为预期状态的对象数 / 回滚对象总数 × 100%
```

---

## 17. 检索质量指标定义建议

### 17.1 当前态召回率

**定义**：查询场景下，当前关键事实被正确召回的比例。

**计算公式**：
```
当前态召回率 = 正确召回当前关键事实的查询数 / 应召回当前关键事实的查询数 × 100%
```

### 17.2 上下文噪音率

**定义**：最终上下文包中被人工评估为低相关内容的占比。

**计算公式**：
```
上下文噪音率 = 人工评估为低相关的内容块数 / 总内容块数 × 100%
```

### 17.3 关键对象召回率

**定义**：在老师/家长/推演场景下，关键知识点、误区、关系对象被召回的比例。

**计算公式**：
```
关键对象召回率 = 召回关键对象的查询数 / 应召回关键对象的查询数 × 100%
```

---

## 18. 解释质量指标定义建议

### 18.1 可追溯解释占比

**定义**：可回到事实/关系/事件来源的解释输出占比。

**计算公式**：
```
可追溯解释占比 = 可追溯到来源的解释数 / 总解释数 × 100%
```

### 18.2 解释链完整率

**定义**：解释中同时具备"当前态 + 结构依据 + 关键事件"的比例。

**计算公式**：
```
解释链完整率 = 完整解释链数 / 总解释数 × 100%
```

### 18.3 解释冲突率

**定义**：同一对象在相近时间窗口中出现明显互相矛盾解释的比例。

**计算公式**：
```
解释冲突率 = 存在冲突的解释对象数 / 总解释对象数 × 100%
```

## 19. 指标采集方式建议

建议采用三种采集方式：

### 19.1 在线指标采集

**适合**：
- 成功率
- 延迟
- 队列积压
- 查询耗时

**采集频率**：实时/分钟级

### 19.2 离线质量扫描

**适合**：
- 孤立节点
- 非法关系
- 版本链断裂
- 覆盖率

**采集频率**：日/周级

### 19.3 人工评估与抽检

**适合**：
- 解释质量
- 检索噪音率
- 老师认可度
- 业务可用性

**采集频率**：周/月级

## 20. 质量等级建议

建议将图谱质量分为四档：

| 等级 | 名称 | 说明 | 进入主链路 |
|------|------|------|----------|
| A | 优秀 | 结构、检索、解释和运行均稳定 | ✅ 可稳定进入 |
| B | 良好 | 主要链路可用，但局部质量有待加强 | ⚠️ 可受限使用 |
| C | 合格 | 可做试验，不适合重度依赖 | ❌ 需重点修复 |
| D | 不合格 | 存在明显结构或运行风险 | ❌ 不可进入 |

## 21. 阶段性验收建议

### 21.1 图谱基础可用

**验收标准**：
- 关键实体和关系已成网
- 当前态事实可查询
- 更新链路基本稳定

**核心指标**：
- 关键关系覆盖率 ≥ 70%
- 图谱写入成功率 ≥ 95%
- 当前有效事实完整率 ≥ 85%

### 21.2 图谱业务可用

**验收标准**：
- GraphRAG 可支撑老师端与家长端
- 报告和解释开始稳定使用图谱

**核心指标**：
- GraphRAG 查询成功率 ≥ 95%
- 可追溯解释占比 ≥ 85%
- 学生详情页解释可用率 ≥ 90%

### 21.3 图谱推演可用

**验收标准**：
- 能支撑后台推演和干预模拟的结构上下文

**核心指标**：
- 推演任务图谱依赖成功率 ≥ 90%
- 干预建议图谱支撑率 ≥ 85%
- 解释链完整率 ≥ 80%

## 22. 常见误判

### 22.1 节点多不等于质量高

| 误判 | 真相 |
|------|------|
| 图谱节点数量增长 | 很多节点可能只是噪音堆积 |

### 22.2 边多不等于解释强

| 误判 | 真相 |
|------|------|
| 关系数量增长 | 错误关系越多，解释越危险 |

### 22.3 查询能跑不等于可用

| 误判 | 真相 |
|------|------|
| 查询返回结果 | 若召回的都是弱相关上下文，仍然算低质量 |

### 22.4 更新很快不等于健康

| 误判 | 真相 |
|------|------|
| 更新速度快 | 如果更新快但错误率高，反而更危险 |

MVP 阶段建议至少做到：

| 能力 | 说明 | 优先级 |

| 当前有效事实完整率可测 | 学生当前态事实字段完整性 | P0 |
| 图谱写入成功率可测 | 图谱写入操作成功率 | P0 |
| GraphRAG 查询成功率可测 | 检索查询成功率 | P0 |

## 27. 风险与边界

| 风险 | 说明 | 缓解措施 |
|------|------|----------|
| 不能忽视结构质量 | 结构问题会导致检索和解释失效 | 定期离线扫描 |
| 不能忽视对象质量 | 对象错误会污染整个图谱 | 建立对象校验机制 |
| 不能忽视更新质量 | 更新失败会导致数据不一致 | 监控更新链路 |
| 不能忽视检索质量 | 检索质量低会让图谱变成冷藏库 | 监控检索指标 |
| 不能忽视解释质量 | 解释质量低会让系统失去可信度 | 人工抽检 + 老师反馈 |
| 不能忽视业务质量 | 业务质量才是最终检验标准 | 建立业务指标体系 |

## 28. 结论

图谱质量评估指标的核心价值，不是给图谱打一串漂亮分数，而是让团队真正知道：

- 图谱哪里好
- 图谱哪里虚
- 图谱哪里已经能进主链路
- 图谱哪里还不能乱信

它必须坚持：

| 原则 | 说明 |
|------|------|
| 结构和业务一起看 | 不能只看结构不看业务 |
| 在线和离线一起看 | 实时监控 + 离线扫描 |
| 自动指标和人工抽检一起看 | 自动化 + 人工评估 |
| 指标最终服务修复和演进 | 指标驱动改进行动 |

否则图谱很容易进入一种危险状态：规模越来越像平台，质量越来越像传说。
