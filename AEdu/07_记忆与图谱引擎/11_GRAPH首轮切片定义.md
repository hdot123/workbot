---

## 1. 文档目的与定位

本文档用于冻结 GRAPH 模块首轮骨架开发的最小切片范围，回答以下问题：

1. GRAPH 本轮到底做什么
2. GRAPH 本轮明确不做什么
3. 哪些能力虽然相关，但本轮不得提前放入
4. 在什么边界内，GRAPH 可以从“条件允许”推进到“允许”

本文档是：

- GRAPH 首轮骨架开发的范围冻结文档
- GRAPH 模块开发准入前置文档
- GRAPH 与 KB、INGEST、TWIN、OBS、SIM 的边界收口文档

本文档不是：

- 字段级数据库设计文档
- API 协议文档
- 前台展示或报告文案文档
- 推演策略设计文档

---

## 2. 首轮范围冻结值

| 维度 | 冻结值 |
|------|--------|
| 地区 | 广东省深圳市 |
| 学段 | 高中 |
| 年级 | 高一 |
| 学科 | 物理 |
| 教材版本 | `PHY_PEP_G1_V1` |
| 输入源 | 家长文本、老师反馈文本、单页扫描/OCR |
| 上游边界 | 仅消费 KB + INGEST + TWIN 已冻结基线 |
| 放行范围 | GRAPH 首轮骨架开发 |
| 明确不放行 | 全量实现、全量联调、多地区、多学科、多教材版本、扩展输入源、下游产品化输出 |

上述冻结值是 GRAPH 首轮唯一有效范围。任何超出冻结值的建模、写入、检索、联调或验收需求，均不在本轮准入范围内。

---

## 3. 首轮目标

GRAPH 首轮骨架开发只达成以下目标：

1. 在冻结范围内，为单学生建立可追溯的图谱记忆骨架。
2. 承接来自 TWIN 的受控对象写入，不直接消费原始输入。
3. 在最小范围内形成 Fact / Relation / Event / Snapshot / Retrieval Unit 的写入闭环。
4. 为后续 OBS 与 SIM 提供可检索、可回滚、可审计的结构化记忆基础，但不提前承担这些模块职责。

换句话说，本轮目标是：

`标准事件与 TWIN 当前态 → GRAPH 对象写入 → 版本留痕 → 检索可召回`

而不是：

`原始输入 → 全量知识图谱 → 全量 GraphRAG 产品输出 → 推演结论`

---

## 4. 本轮纳入能力

### 4.1 图谱核心对象建设

本轮纳入：

- Student 相关个体子图骨架
- KnowledgePoint / Chapter / CurriculumVersion 的最小知识骨架引用
- Fact / Relation / Event / Snapshot / Retrieval Unit 的对象定义与落位
- 单学生、单学科、单教材版本的长期记忆沉淀

本轮不纳入：

- 跨学校群体大图
- 多地区统一总图谱
- 多学科统一学生全域图谱
- 家庭、班级、学校复杂社交网络扩建

### 4.2 受控写入与增量更新

本轮纳入：

- 以 TWIN 输出对象为入口的受控写入
- 事件追加、事实替换、关系激活/失效、快照固化
- 单对象级和单学生级的基础版本回滚
- GraphRAG 基础索引增量刷新

本轮不纳入：

- 全库历史回放
- 复杂批量迁移
- 大范围规则切换重算
- 多租户大规模运维编排

### 4.3 基础检索支持

本轮纳入：

- 单学生 GraphRAG 基础检索
- 当前态事实 + 结构关系 + 关键事件的最小上下文装配
- 为老师/内部复核场景提供结构化召回基础

本轮不纳入：

- 完整家长端上下文产品化输出
- 大规模学校聚合检索
- 高级因果推理路径生成
- 推演层专用复杂上下文包

---

## 5. 本轮不纳入能力

以下能力与 GRAPH 相关，但本轮明确不放行：

| 能力 | 不放行原因 |
|------|------------|
| 复杂因果链自动推理 | 超出首轮骨架开发目标，且会把推演职责提前塞入 GRAPH |
| 面向家长/老师的完整可视化解释输出 | 属于 OBS 的职责，不属于 GRAPH 准入范围 |
| 干预建议生成与策略评估 | 属于 SIM 的职责，不属于 GRAPH |
| 多地区、多教材版本共存图谱 | 超出冻结范围，会打破上游已冻结基线 |
| 钉钉、硬件、批量导入等扩展输入源的直接建图 | 超出冻结输入范围 |
| 全量学校级/年级级聚合图 | 不属于首轮单学生骨架闭环 |
| 全库级规则切换与历史重建 | 不属于首轮准入需要的最小实现 |
| 原始长文本直接图谱化 | 会造成结构污染，不符合受控写入原则 |

## 6. 首轮对象范围

首轮只冻结以下对象：

| 对象 | 定位 | 是否纳入首轮 |
|------|------|---------------|
| Student | 单学生图谱根对象 | 是 |
| CurriculumVersion | 教材版本绑定对象 | 是 |
| Chapter | 章节定位对象 | 是 |
| KnowledgePoint | 知识点定位对象 | 是 |
| LearningEvent | 事件对象 | 是 |
| Fact | 当前事实与历史事实对象 | 是 |
| Relation | 结构关系对象 | 是 |
| StateSnapshot | 学生状态快照对象 | 是 |
| Retrieval Unit | 检索装配引用对象 | 是 |
| RiskSignal | 基础风险信号对象 | 是 |
| Class / School / Region | 仅保留最小归属引用 | 条件纳入 |
| ReportArtifact | 报告产物对象 | 否 |
| InterventionAction 的复杂策略链 | 推演相关对象 | 否 |
| 群体聚合对象 | 班级/学校级聚合视图对象 | 否 |

---

## 7. 首轮写入范围

### 7.1 允许写入

本轮允许写入以下对象类型：

- 由 TWIN 当前态和状态变化映射出的 Fact
- 稳定结构关系，如 `uses / belongs_to / contains / depends_on / has_state / has_risk_signal`
- 标准学习事件与状态更新事件
- 周期或关键时点快照
- 服务 GraphRAG 的最小 Retrieval Unit

### 7.2 条件写入

以下对象只在满足条件时写入：

- KnowledgePoint 关系：必须有稳定 `knowledge_refs`
- Chapter 级引用：可在 `knowledge_refs` 缺失时作为降级写入
- RiskSignal：必须有足够事实支撑，不可由单条低质事件直接提升

### 7.3 不允许写入

本轮不允许写入：

- 原始长文本全文
- 低置信 OCR 原始结果直接入图
- 未校验的自由推断关系
- 面向前台的文案对象
- 面向推演层的假设性结果对象

---

## 8. 首轮检索范围

本轮检索只覆盖以下场景：

1. 单学生当前态解释所需的最小结构检索
2. 单知识点相关事件与关系的最小召回
3. 风险解释的最小证据召回
4. 内部复核与后续模块联调用的基础上下文召回

本轮不覆盖：

1. 学校级聚合检索
2. 大规模报告批量生成检索
3. 复杂干预推演输入包
4. 全域多学生关联分析

---

## 9. 与 TWIN / KB / INGEST 的依赖关系

### 9.1 对 KB 的依赖

GRAPH 不维护知识底座主数据，依赖 KB 提供：

- `curriculum_version_id`
- `chapter_refs`
- `knowledge_refs`
- `anchor_refs`
- 知识点依赖与教材定位规则

### 9.2 对 INGEST 的依赖

GRAPH 不解析原始输入，依赖 INGEST 提供：

- 标准学习事件
- 输入质量与置信度判定
- 事件状态与追踪信息

### 9.3 对 TWIN 的依赖

GRAPH 首轮写入入口依赖 TWIN 提供：

- 当前态事实快照
- 状态变化结果
- 风险与质量摘要
- 可追溯的 `trace_id / last_event_ref / raw_input_ref`

### 9.4 GRAPH 的消费边界

GRAPH 只消费“已被上游处理过的结构化对象”，并完成：

- 对象落位
- 版本链建立
- 检索单元生成
- 回滚与审计留痕

GRAPH 不负责重做上游解析，也不负责替下游生成展示或推演结论。

---

## 10. 进入开发的最小边界

GRAPH 进入首轮骨架开发，至少应满足以下边界：

1. 首轮冻结范围已明确为深圳 / 高中 / 高一 / 物理 / `PHY_PEP_G1_V1`。
2. GRAPH 只消费 KB + INGEST + TWIN 已冻结基线的结构化结果。
3. 写入对象范围已明确收口到 Fact / Relation / Event / Snapshot / Retrieval Unit。
4. 检索范围已明确限制在单学生最小解释闭环。
5. 不允许把 OBS 的展示职责和 SIM 的推演职责提前塞进 GRAPH。
6. 至少存在 3 组端到端样例可支撑开发与验收。
7. 最小验收矩阵与试点责任边界已冻结。

满足以上边界后，允许 GRAPH 进入首轮骨架开发。
