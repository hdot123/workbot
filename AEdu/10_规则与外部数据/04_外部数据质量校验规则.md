# 外部数据质量校验规则

> 文档编号：RULE-004  
> 版本：V1.0  
> 创建日期：2024  
> 最后更新：待定  
> 维护人：规则与外部数据负责人

---

## 1. 文档目的

本文档用于定义教育孪生项目中外部数据的质量校验规则，包括校验维度、校验流程、问题分级、修正机制与入库准入标准。

本文档重点回答以下问题：

1. 外部数据在入库前要检查什么
2. 哪些问题是可接受的，哪些问题必须拦截
3. 如何判断一份外部数据是否足够可信
4. 如何处理来源冲突、字段缺失和口径不一致
5. 如何避免把低质量外部数据喂进系统后污染推演与解释

---

## 2. 外部数据定义

本文所指外部数据，主要包括：

| 数据类型 | 说明 |
|----------|------|
| 地区规则数据 | 地区考试规则、选科规则、批次规则等 |
| 高考历史数据 | 历年录取分数线、位次、招生计划等 |
| 学校与专业数据 | 学校信息、专业信息、选科要求等 |
| 教材版本映射数据 | 教材版本与地区适配关系 |
| 政策与口径说明数据 | 政策解读、执行口径说明等 |
| 其他外部参考数据 | 来自外部机构或公开来源的结构化数据 |

外部数据不是系统内部生成数据，因此其最大风险通常不是"没有"，而是"看起来像真的，但其实口径有坑"。

---

## 3. 质量校验目标

| 目标 | 说明 |
|------|------|
| 保证数据可用 | 确保进入系统的数据至少可被正确读取、解释和使用 |
| 保证数据可信 | 尽可能确认来源可靠、口径明确、时间有效 |
| 保证数据一致 | 避免不同来源、不同字段、不同年份的内容相互冲突却同时生效 |
| 保证数据可追溯 | 任何关键数据都应能追到来源文件、来源时间和处理记录 |
| 控制数据污染 | 低质量数据不得直接进入推演和前台解释主链路 |

---

## 4. 设计原则

| 原则 | 说明 |
|------|------|
| 先校验，再入库 | 校验不是入库后的补救动作，而是入库前门槛 |
| 来源优先于数量 | 多一份不可靠的数据，不一定比少一份可靠的数据更有价值 |
| 结构化优先 | 只有结构清楚的数据，才值得被系统消费 |
| 可疑数据不强行使用 | 证据不足时，应标记为待确认，而不是假装它没问题 |
| 高影响数据高标准 | 越会影响推演和决策的数据，校验要求越高 |

---

## 5. 校验对象分类

建议将外部数据按风险和用途分为三类：

### 5.1 规则类数据

| 特征 | 说明 |
|------|------|
| 数据类型 | 地区规则、选科规则、批次规则、政策说明 |
| 影响 | 影响系统边界和推演约束 |
| 错误代价 | 高 |
| 校验强度 | 高强度校验 |

### 5.2 历史参考类数据

| 特征 | 说明 |
|------|------|
| 数据类型 | 高考历史录取数据、学校专业历年数据 |
| 影响 | 用于参考和比较，不直接决定结果 |
| 错误代价 | 中 |
| 校验强度 | 需校验年份、口径和来源 |

### 5.3 映射类数据

| 特征 | 说明 |
|------|------|
| 数据类型 | 教材版本映射、专业名称映射、学校别名映射 |
| 影响 | 影响系统连接关系 |
| 错误代价 | 中（易出现隐蔽错误） |
| 校验强度 | 需重视一致性校验 |

---

## 6. 校验维度

建议外部数据至少按以下七个维度校验：

| 维度 | 说明 |
|------|------|
| 来源校验 | 确认数据来源是否可信、可追溯 |
| 结构校验 | 确认数据是否满足目标表结构要求 |
| 完整性校验 | 确认关键字段是否缺失 |
| 一致性校验 | 确认数据内部及与现有库之间是否一致 |
| 时效性校验 | 确认数据是否仍在有效时间窗口内 |
| 逻辑合理性校验 | 确认数据在业务逻辑上没有明显异常 |
| 影响等级校验 | 根据数据用途决定校验强度 |

---

## 7. 来源校验

### 7.1 校验目标

确认数据来源是否可信、可追溯。

### 7.2 检查项

| 检查项 | 说明 |
|--------|------|
| 是否有来源文件 | 来源文件是否存在 |
| 是否有来源链接或标识 | 来源是否可追溯 |
| 是否有发布日期 | 发布日期是否明确 |
| 是否官方/权威来源 | 来源可信度评估 |
| 是否存在二手转述痕迹 | 是否经过多次转述 |

### 7.3 来源可信度分级

| 等级 | 来源类型 | 说明 |
|------|----------|------|
| A 级 | 官方发布 | 教育考试院、招生办等官方渠道 |
| B 级 | 权威机构 | 教育部门、学校官方等 |
| C 级 | 可验证第三方 | 经多源交叉验证的数据 |
| D 级 | 来源不明 | 无法追溯来源的数据 |

### 7.4 处理原则

| 原则 | 说明 |
|------|------|
| 来源不明的数据 | 不进入主链路 |
| 来源可信但有口径疑点 | 进入待校验池 |
| 高风险场景 | 仅允许 A/B 级来源进入 |

---

## 8. 结构校验

### 8.1 校验目标

确认数据是否满足目标表结构或目标对象结构要求。

### 8.2 检查项

| 检查项 | 说明 |
|--------|------|
| 必填字段是否齐全 | 必填字段不能缺失 |
| 字段类型是否正确 | 字段类型符合定义 |
| 枚举值是否合法 | 枚举值在允许范围内 |
| 日期格式是否统一 | 日期格式符合规范 |
| 数值字段是否可解析 | 数值字段可正确解析 |
| 结构化对象是否符合规范 | 对象结构符合定义 |

### 8.3 典型问题

| 问题 | 说明 |
|------|------|
| 日期写成自然语言 | "2023 年"而非"2023" |
| 分数字段混入文本 | "600 分"而非"600" |
| 批次字段口径混乱 | 不同年份批次名称不统一 |
| 地区字段缺层级信息 | 只有市没有省 |

---

## 9. 完整性校验

### 9.1 校验目标

确认关键字段是否缺失。

### 9.2 核心检查项

| 数据类型 | 核心字段 |
|----------|----------|
| 地区规则 | region_id, rule_type, valid_from |
| 高考历史数据 | exam_year, region_id, batch_code, school_id |
| 学校专业数据 | school_id, major_id, required_subjects |
| 教材版本映射 | curriculum_version_id, region_id, subject |

### 9.3 处理原则

| 情况 | 处理方式 |
|------|----------|
| 核心字段缺失 | 阻断入库 |
| 非核心字段缺失 | 可标记后入库 |
| 推演关键字段缺失 | 禁止进入推演主链路 |

---

## 10. 一致性校验

### 10.1 校验目标

确认数据内部以及与现有库之间是否一致。

### 10.2 检查项

| 检查项 | 说明 |
|--------|------|
| 同一学校名称是否一致 | 学校命名统一 |
| 同一专业名称是否映射一致 | 专业命名统一 |
| 同地区同年份规则是否冲突 | 规则无冲突 |
| 同类字段口径是否统一 | 字段口径一致 |
| 同批次数据是否混杂口径 | 批次口径统一 |

### 10.3 冲突处理原则

| 优先级 | 原则 |
|--------|------|
| 1 | 优先比较来源可信度 |
| 2 | 优先比较生效时间（新规则优先） |
| 3 | 优先比较地区粒度（细粒度优先） |
| 4 | 无法解决时进入人工复核 |

---

## 11. 时效性校验

### 11.1 校验目标

确认数据是否仍在有效时间窗口内。

### 11.2 检查项

| 检查项 | 说明 |
|--------|------|
| 是否有发布日期 | 发布日期明确 |
| 是否有生效时间 | 生效时间明确 |
| 是否有失效时间 | 失效时间明确（如适用） |
| 是否已被更新版本替代 | 检查是否有新版本 |
| 是否处于政策切换期 | 政策过渡期标注 |

### 11.3 处理原则

| 情况 | 处理方式 |
|------|----------|
| 过期规则 | 不可作为当前规则主版本 |
| 历史数据 | 可保留，但必须明确年份和用途边界 |
| 时效不明的关键规则 | 不直接启用 |

---

## 12. 逻辑合理性校验

### 12.1 校验目标

确认数据在业务逻辑上没有明显异常。

### 12.2 典型检查项

| 检查项 | 说明 |
|--------|------|
| 最低分是否高于平均分 | 分数逻辑合理性 |
| 位次是否为负数或明显异常 | 位次合理性 |
| 同一学校同专业同年是否重复多条冲突记录 | 记录唯一性 |
| 教材映射是否出现互相指向错误 | 映射逻辑正确 |
| 规则生效时间是否晚于失效时间 | 时间逻辑正确 |

### 12.3 说明

这一步很像给数据做常识体检。很多数据并不"格式错误"，但它能从常识上把你一拳打醒。

---

## 13. 影响等级校验

### 13.1 校验目标

根据数据用途决定校验强度。

### 13.2 建议分级

| 等级 | 影响范围 | 校验要求 |
|------|----------|----------|
| 高影响 | 直接影响推演、志愿、规则边界 | 最高强度校验 |
| 中影响 | 影响解释与映射 | 标准校验 |
| 低影响 | 影响说明文字、附加信息 | 基础校验 |

### 13.3 原则

高影响数据校验要求最高，不能和普通说明文本用同一套轻飘飘规则。

---

## 14. 数据质量等级建议

建议将外部数据质量分为四级：

### 14.1 A 级：可直接使用

| 条件 | 说明 |
|------|------|
| 来源可信 | A/B 级来源 |
| 结构完整 | 必填字段齐全 |
| 口径一致 | 无口径冲突 |
| 时间有效 | 在有效时间窗口内 |
| 无关键冲突 | 无关键逻辑问题 |

### 14.2 B 级：可受限使用

| 条件 | 说明 |
|------|------|
| 来源可信 | A/B 级来源 |
| 非关键字段缺失 | 少量非核心字段缺失 |
| 有边界说明 | 有明确使用说明 |
| 可进入部分链路 | 限制使用范围 |

### 14.3 C 级：待确认

| 条件 | 说明 |
|------|------|
| 来源或口径存在不确定性 | 需要进一步确认 |
| 可暂存 | 不进入关键链路 |
| 需人工复核 | 需要人工确认 |

### 14.4 D 级：拒绝入库

| 条件 | 说明 |
|------|------|
| 来源不明 | 无法追溯来源 |
| 结构严重缺失 | 核心字段缺失 |
| 逻辑明显异常 | 业务逻辑错误 |
| 关键字段冲突严重 | 无法解决的冲突 |

---

## 15. 校验流程建议

```
接收外部数据
    ↓
来源校验
    ↓
结构校验
    ↓
完整性校验
    ↓
一致性与逻辑校验
    ↓
时效性校验
    ↓
质量等级判定
    ↓
生成校验报告
    ↓
入库 / 待确认 / 拒绝
```

### 15.1 各阶段说明

| 阶段 | 工作内容 | 输出 |
|------|----------|------|
| 接收外部数据 | 接收原始数据文件 | 原始数据集 |
| 来源校验 | 验证来源可信度 | 来源评级 |
| 结构校验 | 验证字段结构 | 结构校验报告 |
| 完整性校验 | 验证必填字段 | 完整性报告 |
| 一致性与逻辑校验 | 验证数据一致性 | 一致性报告 |
| 时效性校验 | 验证时间有效性 | 时效性报告 |
| 质量等级判定 | 综合评估质量等级 | 质量等级 |
| 生成校验报告 | 生成完整校验报告 | 校验报告 |
| 入库/待确认/拒绝 | 根据等级决定处理方式 | 处理结果 |

---

## 16. 校验输出结构建议

建议每次校验输出以下信息：

| 字段 | 类型 | 说明 |
|------|------|------|
| validation_id | string | 校验任务唯一标识 |
| data_object_id | string | 数据对象标识 |
| data_type | string | 数据类型 |
| source_ref | string | 来源引用 |
| validation_status | string | 校验结果（pass/fail/pending） |
| quality_grade | string | 质量等级（A/B/C/D） |
| issue_count | int | 问题数量 |
| critical_issue_count | int | 关键问题数量 |
| issue_list | array[object] | 问题清单 |
| reviewed_by | string | 审核人 |
| reviewed_at | datetime | 审核时间 |
| action_result | string | 处理结果（accepted/restricted/pending/rejected） |

## 17. 问题分级建议

### 17.1 P0：阻断性问题

必须阻断入库，例如：

| 问题 | 说明 |
|------|------|
| 来源不明 | 无法追溯数据来源 |
| 核心字段缺失 | 必填字段缺失 |
| 逻辑明显错误 | 业务逻辑错误 |
| 高影响冲突未解决 | 关键规则冲突 |

### 17.2 P1：高优先级问题

可暂缓使用，例如：

| 问题 | 说明 |
|------|------|
| 关键口径存在歧义 | 口径解释不明确 |
| 规则版本不清 | 版本信息缺失 |
| 专业映射冲突 | 映射关系冲突 |

### 17.3 P2：一般问题

可带说明入库，例如：

| 问题 | 说明 |
|------|------|
| 部分说明字段缺失 | 非核心字段缺失 |
| 非关键备注不完整 | 备注信息不完整 |

## 18. 与系统链路的关系

### 18.1 知识底座层

| 要求 | 说明 |
|------|------|
| 教材映射和地区适配类数据 | 必须先过校验 |
| 校验不通过 | 不能进入知识底座 |

### 18.2 推演层

| 要求 | 说明 |
|------|------|
| 高影响规则与历史参考数据 | 只有通过质量校验后才能进入推演 |
| 低质量数据 | 禁止进入推演主链路 |

### 18.3 前台解释层

| 要求 | 说明 |
|------|------|
| 报告和解释依据 | 必须来自合格数据 |
| 低质量数据 | 不得直接作为解释依据 |

### 18.4 试点实施层

| 要求 | 说明 |
|------|------|
| 接入学校时使用的地区与规则数据 | 必须来自合格版本 |
| 校验不通过 | 不能用于试点学校 |

## 19. 问题处理机制

### 19.1 问题记录

每个问题应记录：

| 字段 | 说明 |
|------|------|
| issue_id | 问题唯一标识 |
| issue_type | 问题类型 |
| issue_level | 问题等级（P0/P1/P2） |
| issue_description | 问题描述 |
| affected_fields | 受影响字段 |
| suggested_action | 建议处理动作 |
| status | 处理状态 |

### 19.2 问题修复流程

```
发现问题
    ↓
记录问题
    ↓
分配责任人
    ↓
修复问题
    ↓
重新校验
    ↓
确认关闭
```

### 19.3 问题升级机制

| 情况 | 处理方式 |
|------|----------|
| P0 问题 | 立即上报，暂停入库 |
| P1 问题累积超过阈值 | 上报审核人 |
| 同一来源多次出现问题 | 降低来源可信度评级 |

## 20. MVP 建议范围

MVP 阶段建议优先校验以下对象：

| 数据类型 | 优先级 |
|----------|--------|
| 地区规则数据 | 高 |
| 高考历史数据 | 高 |
| 学校与专业映射数据 | 高 |
| 教材版本映射数据 | 高 |

重点保证：

| 保证项 | 说明 |
|--------|------|
| 来源可追溯 | 所有数据有来源记录 |
| 核心字段齐全 | 必填字段不缺失 |
| 口径一致 | 同类型数据口径统一 |
| 质量等级清楚 | 每条数据有质量评级 |

## 24. 风险与边界

### 24.1 不能忽视校验结果

校验不通过的数据不得强行入库。

### 24.2 不能忽视来源可信度

来源不明的数据风险极高，应严格拦截。

### 24.3 不能忽视时效性

过期数据不得作为当前规则使用。

### 24.4 不能忽视问题修复闭环

发现的问题必须有修复和确认关闭记录。

### 24.5 不能忽视校验标准一致性

不同人员执行的校验应有统一标准。

## 26. 结论

外部数据质量校验规则的核心，不是增加一道形式化手续，而是防止外部世界的混乱、过时和错误口径直接冲进系统主链路。

它必须坚持：

- **先校验** - 入库前必须完成校验
- **再分级** - 根据质量等级决定使用范围
- **再决定能不能用** - 不合规数据不得进入主链路
- **决定能用到什么程度** - 明确数据使用边界

否则系统会在最依赖现实世界的时候，被现实世界的数据碎片狠狠干一棍子。
