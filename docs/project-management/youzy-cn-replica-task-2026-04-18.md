# youzy.cn 1:1 全站复制 — GitHub Projects 任务草案

> 起草人：pm-bot（via workbot）
> 起草日期：2026-04-18
> 项目目标：1:1 复制 `https://youzy.cn` 网站（前端页面 + API 行为 + 静态资源），实现像素级还原。
> 数据定位：从 replica 网站中查询出来的数据是本项目的**副产物**，非主交付物。
> 范围：整站（首页、志愿填报核心功能、选科、查大学、查专业、看职业、课堂、资讯、帮助中心、用户系统、支付/VIP、社区、机构、直播等全部模块）。

- [ ] **S1.3 静态资源清单**：提取所有 CSS / JS / 图片 / 字体 / 视频资源清单，标注来源 URL、类型、大小、是否 CDN 化（`staticv2.youzy.cn` / `staticv3.youzy.cn` / `staticv3-pre.youzy.cn`）。

## Phase 4 — API Replay 层 & 数据副产物

> 目标：建立 API mock / replay 层，使 replica 站点的所有数据查询请求能回放真实响应。

### 子任务

- [ ] **S4.1 HAR 整理与分类**：将 Phase 1 录制的 HAR 文件按 API 端点分类，提取唯一请求模板。
- [ ] **S4.2 API Mock 服务器**：搭建 mock 服务（WireMock / MSW / 自定义），录入录制到的真实响应数据。
- [ ] **S4.3 动态参数处理**：识别 API 中的动态参数（省份 ID、年份、分数、用户 token），设计参数化 replay 策略。
- [ ] **S4.4 鉴权 API 处理**：登录 / 注册 / VIP 校验等鉴权相关 API 做 mock 或 bypass。
- [ ] **S4.5 数据落盘**：将 replay 得到的真实响应数据持久化到本地 JSON / SQLite，形成可查询的数据副产物。
- [ ] **S4.6 Replay 覆盖率验证**：对 Phase 3 已复刻的页面，逐页面验证 API 请求能否被 mock 服务正确响应。

### 验收清单

| # | 验收项 | 通过标准 |
|---|--------|----------|
| 1 | API Mock 覆盖 | Phase 3 页面所需 API 100% 有 mock 响应 |
| 2 | 参数化 replay | 省份切换、年份切换等核心参数变化能返回正确数据 |
| 3 | 数据副产物 | 大学列表、专业列表、分数线等查询结果已落盘可查 |
| 4 | 鉴权 bypass | 未登录 / 已登录 / VIP 场景均有对应 mock |
| 5 | Replay 验证报告 | 逐页列出 API 调用 → mock 匹配 → 数据渲染全链路验证结果 |

## 风险与约束

- **动态实时数据**：如直播、实时资讯等依赖外部实时数据源，replica 只能做 UI 还原 + mock 数据。
- **真实支付流程**：VIP 开通 / 志愿卡购买的真实支付链路不在复刻范围内，仅做 UI 层面 1:1。
- **后端业务逻辑**：志愿填报推荐算法、录取概率计算等核心业务逻辑不逆向，仅 mock 输入输出行为。
- **第三方服务**：统计 SDK、客服系统、短信服务等外部依赖标记为 mock 或移除。
- **移动端 / APP / 小程序**：本次仅覆盖 Web 端，移动端策略另定。
